Edit sites and names

This part of the Editor is used to edit/manage the details associated with a site, and observer/recorder names.

Sites and Telescopes

Location

Enter here the site location. The longitude and latitude can be entered in either dms, dm.d or d.dd formats - using the relevant radio buttons - although the report will record the coordinates in dms format. The desired level of precision for the three formats are:
formatDesiredAcceptable
dms0.1"1"
dm.m0.002'0.01'
d.ddd0.00003deg0.0001deg

Note: Site coordinates will only be stored at a precision of 0.1". Coordinates at a purported accuracy of 0.01" will be rounded up [unless the check-box 'Coordinates are high-precision (surveyed)' has been checked.]

Select the horizontal datum from the drop-down box (which lists about 80 datums). Preferred datum is WGS84. The datums are presented in 3 groups. If at all possible, specify a datum in the first group. A datum from the 2nd group is acceptable - although WGS84 is to be preferred. Datums in the third group should not be used for new observations. They are included for historical observations only.

IMPORTANT If using a GPS unit, make sure you do not change it to display coordinates on an 'old' datum.

Specify the Altitude. It can be either meters of feet (using the radio button), but is stored in meters. Even when using WGS84 for latitude and longitude, you should report heights with reference to Mean Sea Level (msl). The form includes an option to specify the height as being referenced to the spheroid (which implicitely is the WGS84 spheroid) - but the preferred height is MSL.

IMPORTANT. By international agreement, the 'standard' reference datum for altitude is MSL. Most GPS units display altitudes as MSL - even when the 'horizontal' datum is set to WGS84. Some GPS units allow you to output a spheroid height - but you should set it to output MSL heights. The spheroid height should only be used if the GPS unit only outputs spheroid altitudes.

Telescope

Specify the aperture in cm. Usually the closest cm is sufficient.

Specify the focal length in cm - usually to the closest cm. Remember that the focal length can be found by multiplying the aperture by the f-ratio.

Select the type of telescope from the drop-down box. [Note - you cannot manually enter your own description]. Select the description closest to your telescope. The following is a more complete list:.

  • Refractor - includes Achromatic, Apochromatic, binoculars, and camera lenses
  • Newtonian reflector - includes Dobsonian, Schmidt-Newtonian, Maksutov-Newtonian, Jones-Bird, Schiefspiegler
  • Cassegrain - includes Schmidt-Cassegrain, Maksutov, Ritchy-Chretien, Dall-Kirkham

Select the type of mounting from the drop-down box. [Note - you cannot manually enter your own description]. Dobsonian, and most commercial GoTo telescopes are Altazimuth mounted.

Select whether the telescope was driven or not from the drop-down box. [Note - you cannot manually enter your own description].

ILOC codes

The system of codes used by ILOC for stations, telescopes and observers was effectively discontinued in about 2006. This data item is not included in the IOTA2009 format.

Editing tools

Buttons allow you to perform the following actions:
ButtonFunction
Add as new siteCreates a new Site entry based on the details entered, allocating a site letter to the site.
Replace selectedReplaces the site highlighted in the list of sites with the details entered.
Set selected as defaultStores the details of the site as a Default site. Typically this would be for the observer's usual observing location. The current default value is displayed as a tooltip.
Add default as new siteInserts the Default site as a new site, allocating a site letter to the site. The current default value is displayed as a tooltip.
Delete selectedDeletes the selected site from the list of sites. Note: a site cannot be deleted if it is associated with an event in the editor.
ReNumberThis is used to re-assign site letters to the sites - particularly after sorting, or deleting a site. The site identifications associated with any events in the editor are updated in the process.

Sorting

Buttons are also provided to sort the list of sites by their allocated letter (number), longitude or latitude, or to move an individual site up or down in the list. If the order of the sites is changed, it may be appropriate to ReNumber the sites. This can be particularly useful for graze reports - where data can be entered as reports are received, and subsequently sorted so that the sites are in the same order as the placement of the observers across the graze path.


Checking site locations

If you have GoogleEarth installed on the computer, you can easily check for any data entry errors in site coordinates by plotting the sites in GoogleEarth. To do this, simply click the menu item Display sites in GoogleEarth.


Names of observers

In this part of the form, enter the name of the observers, and their email address. The email address allows reductions to be sent directly to the observer, and allows a graze organiser to send emails from within Occult to individual observers. Email addresses are not archived.

If the name is entered as a single initial plus family name, the sort function can sort on the basis of the whole name, or just the family name.

The system of codes used by ILOC for stations, telescopes and observers was effectively discontinued in about 2006. This data item is not included in the IOTA2009 format. Similarly, the IOTA2009 format does not include 'recorders'.

This topic is for a legacy function that will be removed.

The legacy functionality relied on the occultation elements being held on disk with a fixed-length record structure - with the data for any one event being read from disk. The fixed-length record structure prevented easy modifications to incorporate extra data elements - and as a result became a barrier to the development of improvements to Occult. The replacement approach maintains all the prediction data in memory, allowing for easier access and filtering; it uses variable length .csv files and .xml files for holding relevant data.

Asteroid - Summary of events

The summary display is a convenient way of accessing events that meet the selection criteria set by the user. The summary list enables you to go directly to events of interest and generate the full range of prediction options for those events. Additionally, the summary page provides tools to:

  • plot 'all' the occultation paths against a regional map, and
  • automatically generate prediction files for all events listed on the summary - for inclusion on web pages and the like.

There are two forms of summary pages

  • a Global Summary of events, in which local circumstances are not a consideration - that is, the selection criteria of the observer's longitude/latitude and the minimum distance from the star are ignored, and
  • a Local Summary of events, where local circumstances are considered.

The Global Summary is particularly useful for events with high uncertainty in path location (e.g. TNO predictions) or for occultations by planets - where the path location is large.     Control functions

    Display event

  • You can display any event by double-clicking on the line. If the menu item with Predictions... Auto-display of plot is checked, merely highlighting the line will result in it being displayed.
  • When the event is displayed, you can access all the prediction options available from the right-click menu on the plot.

    Sort

    The list of events can be sorted by any of Date, Diameter (km), Diameter (apparent), duration, Star magnitude, magnitude drop, elongation, distance, probability, name, number or star number. the list as initially displayed is sorted by date.

    The option Exclude events included in FUTURE.DAT excludes from the listing any events included in the downloaded version of the Future.dat file that is on the computer. This is particularly for generating a feed for OccultWatcher - it prevents the feed including events that are already included in Future.dat. [Note - if applied when displaying the Future.dat file, not all events will disappear - due to some different star formats in Future.dat.]

    Stars occulted more than once

    A list of stars that are occulted more than once can be obtained under the menu item Sort List stars occulted more than once. This is most relevant when predictions for a whole year are generated.

    Abbreviated output

    This option (under with Predictions...) truncates the summary line - excluding all data fields after the planet name (but the Rec # field is retained.)

    Regional plot of occultation paths

    You can plot a regional map showing the paths of all occultations in the summary listing that occur in that region - by clicking the with Predictions.., Plot paths on a map. Click here for details. Note For this functionality you can set up to 8 pre-defined plotting regions in the User Settings.

    Auto-generation of prediction files

    The summary list provides functionality to automatically generate prediction files for all events listed on the summary. This function is particularly designed for those creating web pages, including those creating feeds for OccultWatcher.

    IMPORTANT To use this option, you must enable it from the User Settings, in the box Asteroid occultations - Search and display. You must check the box Enable generation of output files, and then check those files that you want to have generated. To include city locations in the Occult map, you need to specify the site file in the User Settings.

    You should check Exclude events included in FUTURE.DAT under Sort, to exclude events that are already included in Future.dat.

    The Autogenerate function works as follows

    • The output files are written to the directory Occult\AutoGenerated Asteroids.
    • The site file used to generate the Multisite prediction file is that specified at the top of the form;
    • The site file used to plot cities on the Occult maps is that specified in the User Settings;
    • An index of the events generated is written to the file FileIndex.txt. each entry is a text identifier of the event, followed by a unique identifier;
    • A series of control files is written to the subdirectory MJD. The file has the name of the unique identifier, and contains the Modified Julian day number of the date the relevant occultation element file was created.
      Before creating a set of output files for an event, the program checks to see if there is a corresponding file in the MJD directory. If the file exists, the program compares the creation date of the current occultation elements with the date stored in the check file - and only generates output if the elements creation date is later than that stored in the check file. [This means that, for example, you can generate a summary using Steve Preston's Future.dat file, and the Autogenerate function will only create files for new or revised occultation elements.]
    • If you are using an updated Sites file to generate multisite predictions, you can check the Site file has been updated box. This will result in the output files for that event being regenerated.

Camera corrections

If the observation was made using an integrating camera, details of the integration used must be specified in order to derive the correct time.

Depending upon the camera, corrections may be required for internal delays within the camera.

These two aspects are dealt with in the following box:

·         Select the camera in the 'Video Camera' box using the drop-down list.

·         Specify the number of frames integrated in the bottom left box. If the camera is a non-integrating camera, specify 0. For a Watec 120N+ camera, use 0 for all the 'fast' settings, and 1or larger for all the slow settings. [Slow 1 is an integration over 1 frame].

·         In the 'video system' box, specify whether the camera is NTSC or PAL, or ADVS. For videos measured with Tangra that information is not available in the .csv file versions; however AOTA will determine the setting if the frame times are included in the CSV file of Tangra. For users of Tangra version 3.0.81 or later accessing AOTA directly from Tangra, the system will be transferred. If the video system has not been set, the Video System box will appear as follows.

·         The total time correction applied is given in bold text at bottom right.

Final times

In this box the final results of the analysis are given for the current event being analysed. The event number is given at the top of the box. [Remember that up to five possible events in the light curve can be analysed.]

The Final times box includes a button Verify which is enabled when AOTA is called directly from Tangra. This allows the observer to display (in Tangra) the video frame corresponding to the computed event time, to confirm the event time - with the ability to specify a correction if needed (which may be the case if the video suffers from missed or duplicate frames.) More details are here.

Saving and Copying useful plots

It can be very useful to save an image of the analysis of an event. There are several ways of doing this

  1. From any tab, the menu item 'File Save image of this form as currently displayed' will save an image of the form as currently displayed. There are two submenus:
    a. Save image of form - which actually saves an image of the form
    b. Set scale (Currently xxx%) to match Monitor scale. This is important if you have set you monitor to display text and images at a scale greater than 100%. This Windows setting is found under Settings, Display, Scale and Layout.  When selected the following form will appear. Simply select the button that matches the Windows setting, and close the form. The setting will be retained between sessions. For most users, the setting will be 100%.
    .
  2. Similarly the menu item Edit... copy form will copy the form as currently displayed to the clipboard.
  3. From any tab, the menu item 'File Save plot' will save an image of the light curve plot (including, if the plot is wider than the display, that part which is not displayed). Similarly the menu item Edit... copy plot will copy the light curve plot to the clipboard.

  1. The buttons in the Save AOTA images provide specific functions for the current event analysis:
    • the Save Tab 5 image for event #X saves an image of the form as displayed under Tab 5 - that is, it provides all the details of the final analysis. The text under the button displays the current setting within Occult, which should match the Windows monitor scale setting. See item (i)(b) above.
    • the Save Tab 4 image of plot analysis will save the entire plot as shown in Tab 4. This provides an overall view of the entire light curve and what has been identified as possible events. For Miss events, it also assists in confirming that a Miss was in fact recorded.
    • the file format used for the save is specified in the drop-down box.

The files saved will be found in the same directory as the source file read by AOTA (or, in the case of AOTA being called from Tangra, the source file read by Tangra.) The file name will be the same as the source file name, except with additional text to indicate the nature of the saved data.

Final reports

The top line advises of the number of events that AOTA believes need to be analysed. Under that, four buttons provide the following functions:

  1. Set Event #X as a non-event. If an analysed event is considered to be a 'non-event' after due consideration, click this button to set the event status to 'non-event'. A label will appear over the Final Results Box to indicate the event is set as a non-event. This setting can be cleared from Tab 4.

  2. View report. This displays the current analysis report, giving details for each of the analysed events. An example report is:

  2. Light curve report. This will take you into the functionality for reporting the light curve (for archiving at VizieR). You will need to specify the event date, asteroid number, star number, and site coordinates. Also you will need to select that part of the entire light curve that is appropriate for archiving. Unless you are sure you have finished with analysing the light curve for the event times, you should avoid making any changes to Binning, Integration, Normalisation, or Integrity check changes - as such changes in the Light Curve reporter will affect the data being used to analyse the event times.

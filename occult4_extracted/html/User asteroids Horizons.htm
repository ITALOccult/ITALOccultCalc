JPL-Horizons

Adding Orbital Elements from Horizons

The JPL Horizons system can generate accurate orbital elements at any epoch. This has particular advantages when the orbital integration routine in Occult is inadequate. This particularly occurs with orbits that have a large eccentricity (eg greater than 0.4), or a small perihelion distance(eg less than 1.5AU). Accuracy can also be lost if the event date is many years from the current date.

In these circumstances, Occult can 'automatically' download the elements for an object for a user-specified date. This can occur in three locations: on the Asteroid Search form, on the List and Display form, and on the form for editing the file of UserAsteroids.

Adding elements for a single asteroid when searching

The process for using Horizons for a single asteroid when conducting a search on the Asteroid Search Form is set out here.

The process is limited to searches with a duration of less than 35 days, and limited to a single asteroid. The intent is to provide an accurate prediction for an object of special interest - and in particular, when that object is in an orbit with eccentricity greater than 0.4 and semimajor axis less than 2AU.

Creating a User file of asteroid elements to match events of interest

The work-flow for this option is
  • generate a set of predictions using MPCORB, Astorb, or AstDys-2. Typically they would be for a time-limited period of a month or two, appropriate for generating an OccultWatcher feed or personal predictions
  • use the filter options on the List and display form to limit the listed events to those that are of potential interest
  • select this option, to create a User file of asteroid elements, by downloading orbital elements form JPL-Horizons
  • Re-do the search for the time period, but this time check the box on the Search form use USER file of elements

the resulting search will be based on Horizon elements. More information is here

Adding elements using the Editor for the UserAsteroids file

The basic function of the UserAsteroids editor is the manual addition of asteroid and comet orbits. 

The functionality to add orbits from Horizons is enabled by clicking the Menu item Add elements from JPL-Horizons. This will disable the general editor functions, and will display a panel that looks like

This panel can be moved around the form by dragging it with the cursor.

The top row of data elements is used to set the epoch for the elements to be downloaded. The panel opens with the current UTC date being displayed. The date can be set to any date (within reason....). All downloaded elements will be for this epoch.

A button is provided to delete all elements in the list.

Adding Asteroids

The left part of the form is for specifying the asteroids to be downloaded. There are two basic options.

  • Specify a set of asteroids either individually, or as a range; or
  • Specify all asteroids greater than a certain diameter
These options are selected by clicking the appropriate radio button. For the radio buttons relating to asteroid size, the number of asteroids that will be downloaded is indicated by the number below the radio button. If a Diameter radio button is selected, the text box below the buttons for specifying individual asteroids is disabled.

If the by Number option is selected, the text box below the buttons for specifying individual asteroids is enabled. You then specify the asteroids to be downloaded. The following are examples of valid inputs

    254   {for asteroid 254 alone}
    -100  {for asteroids 1 to 100 inclusive}
    456-543  {for asteroids 456 to 543 inclusive}
    4, 76, 3696, 25, 50000   {for asteroids 4, 25, 76, 3696 and 50000. The order is not important}

Spaces can be included between numbers. But any other characters will result in an error message.

Be careful about how many asteroids you specify. Assuming a reasonable internet speed it will take about 1 second to download for each asteroid. [The program imposes a limit of 5000 asteroids.]

To start downloading, simply click Retrieve elements. If there are any issues with the specified criteria you will get an error message, and you will need to correct the relevant entry.

If the criteria are valid, the process for retrieving the elements will begin. At the bottom left of the form the number of the just-downloaded asteroid will be displayed on an asteroid-by-asteroid basis. After every 10 asteroids, the displayed list of asteroids held in the editor will be updated (with the display being focused on the last asteroid in the list).

Also displayed is an Abort button to cancel the downloads. It may require clicking more than once to be effective (and it will only abort after the end of the current asteroid download).

When the download is completed, all the displayed asteroids are saved in an updated UserAsteroids file.

You can view the asteroids in the list while the Horizons panel is displayed, using the scroll bar. However you cannot edit any entry.

You leave the Horizons panel by clicking the X at the top right of the panel.

Adding Comets

The form provides a list of all comets held in the MPC Comets file, which is download #2. For newly discovered comets, you need to download the latest version of that file.

There are three categories of comets listed. The identifications are in the form:

Identifier Description
C/yyyy id (Name) C = near-parabolic orbit, single apparition
P/yyyy id (Name) P = periodic comet, single apparition
xxxP/Name Periodic comet observed at multiple appearances
xI/Name Interstellar comet
 

The id is structured as the half-month of discovery, and the sequence of the discovery in that month. That A2 is the 2nd comet discovered in Jan 1-15, B5 is the 5th comet discovered in the period Jan16-31.

IMPORTANT. The positions of comets are far less reliable than for asteroids. This is a combination of the nucleus not being readily identifiable within the coma, and the presence of non-gravitational forces. Within the first month of discovery the ephemeris is usually quite inaccurate - far too inaccurate for sensible occultation predictions. In general, periodic comets at multiple returns have the best orbits - and hence will provide more reliable predictions than for other comets.

IMPORTANT #2. Becaiuse of non-gravitational forces, as well as orbits being not as well defined as for asteroids, ensure the Horizons elements are downloaded for about the mid-point of the time period you are covering for your search, and keep the search duration relatively short (a month or two)

IMPORTANT #3. Comets of the for C/ have near parabolic orbits. The integration scheme in Occult may be suboptimal. As a genera rule, keep predictions for such comets to a relatively short period, such as a month or two.

To add comet elements to the User file, check the comets you want to add (double click). You can clear all the checks using the Clear all checks button. Or you can clear checks individually. The number of comets that have been selected is given in the yellow download button.

When you have made your selection, click the Retrieve xx elements. This will query Horizons, and add the elements to the main list. The file will be automatically saved at the end of the download.

When you have added comets to the UsersAsteroids file, do not check the Horizons button on the search form. If checked, the program will attempt to download elements again, and may fail.

Occultations by Asteroids, Planets and Planetary Moons

This page provides access to the forms for
  • searching for occultations,
  • generating predictions for any location, or to plot the occultation on the earth
  • downloading occultation elements, and
  • managing user files of stars and asteroids for generating predictions of specific objects.

Summary of the prediction process

This form is the location for generating predictions of occultations by asteroids (or planets). The prediction process involves the following steps.

  • Download a current version of MPCorb, AstDyS2 or Astorb - files of orbital elements of asteroids. This is done from the Maintenance tab, using the Downloads button. The download is 40 to 60 MB. The preferred sources of elements are MPCorb and AstDyS2; however you should ensure you also have downloaded Astorb, as error information is available from that file.

  • Extract asteroids larger than a certain size (e.g. 25km), and convert their data into a format that Occult can use (which includes diameters for all asteroids) - again using Convert Astorb and MPCorb. It is desirable to have Astorb downloaded so that error information can be included with elements from MPCorb or AstDyS2. The conversion process also adds the taxonomic class to the record.

  • Conduct a search for possible occultations. This is done from the Asteroid Predictions form, using the Search for Occultations button - where you can specify the date range, the asteroids to be considered, and the star catalogue to use. Possible events are listed as the search progresses.

  • The events that have been found are listed on the List and Display form. On this form you can filter the events by a wide range of criteria, such as:
    • your location
    • a distance to the occultation path - 300km is a good value to allow for prediction uncertainties
    • a magnitude drop of at least 1.0 magnitude
    • a duration of at least 3 seconds

    You can view events by clicking on them. From the plot of an event you can display a range of related prediction details, as well as various special output files.

  • You can save the occultation elements of either the filtered list of events, or all computed events - for retrieving at a later date.

Button functions on the form

Search for, & list, occultations

Brings up the form for conducting the search and creating a file of occultation elements. Details

List & Display occultations

Brings up the form for listing and displaying occultations. Details

Convert Astorb, MPCORB, & AstDyS-2

Opens the form for converting Astorb, MPCORB and AstDyS-2 into the format used by Occult. While this form is usually accessed from the Maintenance tab, its provision here is to facilitate the recreation of the Occult file of asteroid orbital elements using a different orbit source, or a different set of parameters (such as the asteroid diameter).

Edit the coordinates of a 'User' star

In some circumstances there can be a need to conduct a search for occultations of a particular star with details of the star being provided by the user. The star details are held in a 'user' catalogue that contains just one star. This option allows for editing of the star details.

Once the star coordinates have been specified, the search is conducted against that star by selecting User Star in section 3 of the Search form.

Edit the 'User' file of minor planets

In some circumstances there can be a need to conduct a search for occultations of an object not in the usual file of orbital elements, or using an updated set of orbital elements. Such elements are held in a 'user' catalogue. This option allows for editing of the elements. A particular use for this User catalogue is for predictions of occultations be comets - the form for editing the User file includes the capability to import comet orbits. 

If the User file of elements is selected in the search form (with the check box in section 4 of the search form), the search is conducted against all objects in the file. For this reason, the number of objects held in this file should be kept to a minimum.

Select DE Ephemeris

Allows the user to select the JPL DE planetary ephemeris to use in the search. Normally you would use the latest available JPL ephemeris. [This functionality was provided for Pluto, when its ephemeris was troublesome.]

Asteroid shape models - Damit & ISAM

The Display routine can make use of Asteroidal shape models which are available on DAMIT and ISAM web sites. However if either of those web sites is 'down', the functionality can become very slow. A tool is provided to establish whether those sites are currently on-line. The routine will only attempt to access the shape models if the relevant check box is ticked. [Note - if either DAMIT or ISAM is 'down', the check can take in excess of 20 seconds - and double this if both sites are down.] The result of that check is applied to all three locations where the check is provided.

Download prediction files

Steve Preston generates updated predictions of occultations with orbital elements that he has determined using the latest available astrometry. These files (Future.xml and FutureALL.xml) can be downloaded with these buttons, and then used to generate predictions.

    Occasionally users may want to be able to access previous versions of the Future.xml file. In the User Settings form, in the box for FTP URL's, there is a check box under Future.xml - "On download, preserve previous version of Future.xml". If this box is checked, an existing Future.xml file is renamed by adding the date of the new download to the file name - formatted as 'Future_yyyymmdd.dat'. That is, the file name includes the date it was replaced. The current setting is indicated below these two download buttons.

The LuckyStar files are predictions involving Pluto, TNO asteroids, and other outer asteroids generated by Paris Observatory. To use this feed you must agree to certain conditions of use - which are explained when this option is selected.

  • The basic operation of this feed involves downloading a group of files from a web site (each file relating to a separate object), and converting the data in those files into the Occelmnt format.
  • The object files are held in directories covering 1-year periods
    • .The first step is to select the year of interest. Data is generally not available for many years in advance.
  • Having selected the year, the object files will be downloaded.
  • Once the download is complete, the conversion will occur. If the cursor is held over an Occult for, a rotating cursor symbol will indicate progress.
  • The converted data is automatically stored, with the file name being of the form:
    LuckyStarYYYY_yyyymmdd.XML, where YYYY is the LuckyStar event year, and yyyymmdd is the date of the conversion.
The RIO TNO files are predictions involving Pluto, TNO asteroids, and other outer asteroids. To use this feed you must agree to certain conditions of use - which are explained when this option is selected.
    The basic operation of this feed involves downloading three files from a web site, and converting the data in those files into the Occelemnt format. On the Asteroid predictions page of Occult, three options are provided:
    • Exclude events more than one month old. The data in the feed will typically include old events. This self-explanatory option limits the output to current and future events.
    • Use existing files. This option is applicable when you have (for some reason) edited the RIO source files. Or you need to regenerate the Occelmnt file on the basis of a previous download.
    • Use local 'LOG.dat' & 'table_occult.txt' (only) The RIO/IMCCE groups may be in a position of issuing an update to their prediction just a few days before an event. If there is enough time, the files that are accessed via the normal download are updated, and the update can be accessed in the normal manner. However occassionally the updates are only available by way of two files - 'table_occult.txt' and 'LOG.dat', which are emailed to certain users. This option creates an Occelmnt file from those two files. In this situation, the user needs to copy the two files from the email to a directory of their choice - then run this option. The user will be asked to select each of these files (by way of two dialogue boxes), following which Occult will create the Occelmnt file in the Generated Files subdirectory of Occult, ready to generate prediction maps etc.

PPMXL Catalogue - file format

Occult uses a special binary format version of the PPMXL catalogue for use in searching for occultations of faint stars. The file structure and star content is the same as the original PPMXL catalogue. However the data has been limited to positional data (with error information) and B, V and R magnitudes.

This catalogue has been created from the full PPMXL catalogue. The version of the catalogue used is that made available on Vizier on 6 June 2011. That is, the version that has a number of magnitude issues from PPMX and USNO-B1 corrected.

This catalogue has been created for the purpose of efficient searching for occultations of fainter stars. To save disk space, and to keep the catalogue at a reasonable size for distribution on DVD's, the data included has been limited to positional information (including uncertainties), and basic magnitude information. Index files have been created to allow for efficient searching for occultations. All data is stored as either 16-bit or 32-bit integers.

The catalogue consists of 720 zones, each 0.25 deg wide in declination. The file name is structured in 4 parts, as follows:
XYYZ
where:

  • X is either 'n' or 's', indicating the hemisphere;
  • YY is the declination band in integral degrees; and
  • Z is one of a, b, c or d - and indicates the relevant part of the declination band. a = 0 - 15'; b = 15' - 30'; c = 30'-45'; and d = 45' - 60'.
Each file is present with two extensions:
  • dat = the data file. [File sizes range between 68,068 and 99,286,358 bytes]
  • .inx = the index file for that zone. [Each file has a length of 7,204 bytes]

The total file size of the catalogue is 30,957,481,984 bytes, equating to 910,514,176 entries [i.e. almost 1 billion entries].

The format of the star catalogue is a 34-byte record length random-access file with the following record definition.
BytesMeaningBinary format:
0 - 3 RA at J2000, mas. 32-bit integer
4 - 7 Dec at J2000, mas32-bit integer
8 - 9 Blue mag, units of 0.01 mag 16-bit integer.
10 - 11 Red mag, units of 0.01 mag 16-bit integer
12 - 13 Visual mag, units of 0.01 mag 16-bit integer
14 - 17 Annual Proper motion in RA*cos(dec), 0.1mas/yr 32-bit integer
18 - 21 Annual Proper motion in Declination, 0.1mas/yr 32-bit integer
22 - 23 Error in RA*cos(dec), mas16-bit integer
24 - 25 Error in Declination, mas16-bit integer
26 - 27 Error in pmRA*cos(dec), mas16-bit integer
28 - 29 Error in pmDeclination, mas16-bit integer
30 - 31Mean epoch in RA, 0.01 years from 200016-bit integer
32 - 33 Mean epoch in Declination, 0.01 years from 200016-bit integer

Magnitudes

The magnitudes in this version have been derived from the values given in the full PPMXL catalogue as follows:
  • If the catalogue indicated the data had come from the PPMX catalogue:
    • B mag: Johnson B mag when available in 'b2mag'; else the 'catalogue' mag in 'b1mag'
    • V mag: Johnson V mag from 'imag'
    • R Mag: The calculated Ru magnitude from source, from colun r1mag.
  • All other cases:
    • B mag: The mean of columns b1mag and b2mag. If only one value given, that value.
    • V mag: Not specified
    • R Mag: The mean of columns r1mag and r2mag. If only one value given, that value.
  • If no magnitude is specified, the magnitude field is set to -1000 (ie, mag = -10)

Data content of the .inx file

The index file contains 1801 32-bit integers giving the record number of the first star in the zone file that has a right ascension greater than the corresponding increment in Right Ascension. The increment is 0.8 mins (time) [0.2 deg] in RA. The last record gives the record number of the last star in the file.

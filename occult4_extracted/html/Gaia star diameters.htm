Star diameters from the Gaia DR3 Supplementary files

This form enables the retrieval of star diameters from the Gaia supplementary files. There are two supplementary files:

paramp 1D astrophysical parameters produced by the Apsis processing chain developed in Gaia DPAC CU8
paramsup Additional parameters from the Apsis processing chain, compared to the main table astrophysical parameters, from modules that produce more than one result for a parameter

The diameters provided in these files are not 'measured' diameters, but diameters inferred using different techniques.

The data element provided in these files is the star's radius in units of Solar Radius. These radii are converted to the apparent diameter as seen from the Earth using the star's parallax, using:

Dia (mas) = 9.304 * Parallax * SolarRadius

where the star's parallax is specified in arcsec.

To retrieve the diameters, follow the instructions on the form. The VizieR query will return with 4 data elements for each basis for estimating the diameter

  • the estimated radius of the star in Solar Units
  • the apparent diameter of the star, in mas
  • the + and - uncertainty values of that diameter

 

The basis of the diameter measurements are:

paramp file

  • Radius from GSP-Phot Aeneas best library using BP/RP spectra
  • Radius of the star from FLAME using teff_gspphot and lum_flame

    • paramsup file

  • Radius from GSP-Phot Aeneas for MARCS library using BP/RP spectra
  • Radius from GSP-Phot Aeneas for PHOENIX library using BP/RP spectra
  • Radius from GSP-Phot Aeneas for OB library using BP/RP spectra
  • Radius from GSP-Phot Aeneas for A library using BP/RP spectra
  • Radius of the star from FLAME using teffGspspec and lumFlameSpec



    • For those who want to know more about the tools used:

    FLAME

    For an explanation of  FLAME (Final Luminosity And Mass Estimator), click here. The FLAME results are in the paramp file for approximately 300M sources ( G<18.25).

    A second FLAME result is produced only for the bright sources based on GSP-Spec parameters for approximately 5M sources, and these are found in the paramsup file.

     

    GSP-Phot

    For an explanation of GSP-Phot (General Stellar Parametrizer from Photometry), click here. The 'best-library' results are provided in paramp. The separate results for MARCS, PHOENIX, A, OB (modulo filtering) libraries are provided in the paramsup file.

    Some more explanation of the set of GSP-Phot categories, as set out in the documentation:

    GSP-Phot has processed all sources down to G≤19 with all four libraries (MARCS, PHOENIX, A, OB). Consequently, there are, in principle, four sets of GSP-Phot results for every source (some results are removed during filtering). After filtering, GSP-Phot still provides 1 665 152 313 results (73.97%) for 470 759 263 sources (83.65%)

    All results are reported in the paramsup file such that users favouring certain libraries can always access those results. However, in the paramp file we also report the results for the “best” library. 

    The best library is identified as the one
        having the highest mean log-posterior value averaged over the MCMC samples
    with formulae being set out in the documentation.

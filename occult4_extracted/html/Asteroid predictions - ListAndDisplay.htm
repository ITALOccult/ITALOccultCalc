List and Display

This form provides the following functionality

  • Displaying a list of the results of a search just completed
  • loading and displaying the results of a search conducted previously, that has been saved to file
  • Merging the results of plural searches
  • sorting the list of search results by any of 14 parameters
  • filtering the list of search results by the application of one or more of 19 criteria
  • saving the entire list of events held in memory
  • saving just those events that meet the filter criteria.
  • plotting any event in the list, and from that plot accessing the full range of additional prediction information
  • automatically generating a range of prediction files (as specified in the User Settings, group 11) for each listed events.
  • generating the files needed for an OccultWatcher feed
  • generating a UserAsteroid.csv file by downloading orbital elements form JPL-Horizons for each listed event
  • generating a more accurate prediction for a single event, by right-clicking on it

Sort Events

Sorts the list of events by any of Date, Diameter (km), Diameter (arcsec), duration, Star magnitude, combined magnitude, magnitude drop Visual, Magnitude drop Red, elongation, distance, probability, name, number or star number. The list as initially displayed is sorted by date.

  • List stars occulted more than once

The last option in the Sort menu list is to list events involving the same star. This is only likely to be relevant when a large number of events are predicted - but it enables the ready identification of events involving the same star. This can be useful for studying characteristics of that star using different occultation events, or for applying positional offsets to the star position based on the first observed event (if such is justified).

Generate predictions for all listed events

This menu group automatically generates prediction files for all events listed on the summary. It is particularly designed for those creating web pages, including those creating feeds for OccultWatcher. To avoid generating predictions of events that are already contained in Steve Preston's Future.dat and FutureAll540.dat files, check 'Exclude if in Future.dat' in the 4th filter box. {You should ensure you have up-to-date versions of the Future.dat and FutureAll540.dat files.}

  • Set scale for World maps ( = x.x)   =>

This setting is for the scale of the world map that is generated by the autogeneration routines. The drop-down lists provides the available scales - from 1.0 to 8.0. The scale setting is retained between sessions.

When the center line of the occultation path misses the Earth, the world map is scaled such that the map shows the occultation path relative to the World - irrespective of the selected plot scale.

  • Plot the maps Path/Site-centered [xxxx]

When the map scale is greater than 1, the plotted map is centered on one of two bases. The reference point for plotting the map is the site location set in the selection filter box for Visibility - irrespective of whether or not the visibility filter is checked.

The map can be centered on that site location, or on the point along the occultation path that is closest to that site location. The current setting is shown in brackets at the end of the menu entry (and the menu item is checked when the setting is 'Path'). The setting is retained between sessions.

Anyone using the Autogeneration function should assess which of these two options best meets their needs. There is no material difference when the path passes close to the nominated site. However the difference can be quite significant when the path is located some distance from the nominated site.

  • Create files  [StationSort using site file =>

This function generates a set of files that can be used to create a web page containing detailed information about each event. One of the files that can be created is a file containing local predictions and path distances for locations in a Site file. The drop-down lists provides an alphabetically-sorted list of available Site files that can be used to generate that output file - including an option of 'None'. The selected site file is retained between sessions - but has to be re-selected each time the function is used.

IMPORTANT To use this option, you must specify the files you want to have generated under User Settings, in the box Asteroid occultations - Search and display. To include city locations in the Occult map, you need to specify the site file in the User Settings, Group 11 (near the end).

IMPORTANT - change introduced with Occult v4.5. The Autogeneration function is automatically enabled on the List and Display form. In the User Settings you do not need to check the box Enable generation of output files unless you are intend using the old functionality (which is being retained for a period of time.) With Occult v4.5 the intention is that the user adjusts the filter settings (including the Regional filter) to create a list of suitable predictions; there is no option available to select/deselect events on the basis of check boxes.

The Autogenerate function works as follows

  • The output files are written to the directory Occult 4\AutoGenerated Asteroids.
  • One of the potential files that is generated [{xxxx}_Site.txt] gives the mid-time of the event for each site in a Site file, with the results being sorted by order of distance from the path. When you click this menu item, a submenu will list all the available Site files. Click the Site file you wish to use, or 'None' if you do not want to use a site file.
  • The site file used to plot cities on the Occult maps is that specified in the User Settings.
  • An index of the events generated is written to the file FileIndex.txt. Each entry is a text identifier of the event, followed by a unique identifier;
  • A series of control files is written to the subdirectory MJD. The file has the name of the unique identifier, and contains the Modified Julian day number of the date the relevant occultation element file was created.
  • Before creating a set of output files for an event, the program checks to see if there is a corresponding file in the MJD directory. If the file exists, the program compares the creation date of the current occultation elements with the date stored in the check file - and only generates output if the elements creation date is later than that stored in the check file. [This means that, for example, you can generate a summary using Steve Preston's Future.dat file, and the Autogenerate function will only create files for new or revised occultation elements.]

Accurate prediction for a selected event

Events found when searching over a lengthy period of time, or without using the JPL Horizons search option, may have reduced accuracy because of limitations with the basic numerical integration scheme implemented in Occult. If you identify an event in a list where you want a very accurate prediction, select the event, and then right-click on it. This will enable you to generate an updated prediction for that event.

Note that this functionality has been superseded by the check-box option on the Search form: use Horizons ephemeris (predictions will be slow) - which provides for accurate predictions for multiple objects.

The information displayed in the list is:

  • Date, U.T: The date and time of closest approach. The hierarchy of the U.T of the event time displayed is:
    1. if filtered on site location - the time is for the closest approach to the location;
    2. if filtered on the region (Path crosses...) BUT NOT the site location - the time is for the closest approach to the centre of the region.
    3. otherwise the time is for the closest approach to the centre of the Earth.
  • Diameter: The objects diameter, both in km and arcsec. For the planets it is the equatorial diameter. For Venus the diameter is for the top of the clouds.
  • Duration: The expected duration of the occultation on the basis of the diameter, assuming a central occultation. The time is typically given in seconds (s), but fort long events can be given in minutes (m).
  • Star Mag: The visual magnitude of the star. Note that this magnitude may not be a strict V magnitude - depending on the source of the star position.
  • Mag-Drop: The expected magnitude drop, on the basis of the star's V and R magnitudes and the magnitude of the occulting object.
  • *: a flag to indicate there are 'nearby' stars that are brighter than [target star magnitude + 1.0].
      ‡ = there are stars within 5", and the magnitude drop has been corrected to include all such stars.
      † = there are stars within 15".
  • Elon: The solar elongation of the star at the time of the event.
  • % Illum: For occultations by the Planets, the percent illumination of the planet. This is of particular importance for Mercury, Venus and Mars.
  • Star No. : The star identification number
  • d : A flag to indicate there is double star information available, variable star information, or that the stellar diameter is significant. The flags are:
    • W the star is listed in WDS
    • D the star is a double from some source (such as the Interferometric catalogue). It may or may not be listed in WDS
    • V the star is a variable star. (It may also be a double star)
    • If the flag is one of s, w, d or v, this indicates that stellar diameter is at least 1 mas - and will therefore have an effect on the occultation. The codes w, d or v indicate the star is also listed in one of the catalogues as per the capital letter equivalent
  • Rely The reliability of the star position (RUWE in GAIA), Values <1.4 indicate the star position is reliable. For larger values there may be questions about the reliability of the position. Similarly if no value is given. A value of 0.0 is usually associated with a lack of proper motion, and as a result the position of the star has significant uncertainty.
  • Planet No Name: The identification of the occulting object. For asteroids, it is the asteroid number and its designation. For planets it is the name of the planet or moon. The identification numer used is of the form PxMyy, where X is the planet number, and yy is the number of the moon in the system of that planet.

For Events not filtered on site location:

For asteroids and Satellites of the major planets

  • MinD: The minimum distance from the center of the Earth, in Earth radii
  • Error: The uncertainty in the minimum distance

For Major planets

The approximate longitude range from which the occultation is visible

For Events filtered on site location:

  • Alt: The altitude of the star at the time of closest approach as seen from the location
  • Az: The azimuth of the star at the time of closest approach
  • Dist: The apparent minimum separation of the star and object, in arc secs
  • Sun Alt: The altitude of the sun at the time of the event. It is only given if the Sun's altitude is higher than -12 deg.
  • Moon Alt: The altitude of the moon at the time of the event. It is only given if the moon's altitude is higher than -2 deg.
  • Probability: The nominal probability that an occultation might be observed from that location.
  • Moon Elon Illum: The elongation of the moon from the star, and its percent illumination.
  • § : A flag to indicate a shape model is available for the asteroid.
  • ☾ : A flag to indicate the presence of asteroidal moons
  • ◉ : A flag to indicate the presence of asteroidal rings
  • RA Dec : The star's coordinates on the J2000 equinox.
  • (final column) : A reference number to link the displayed event to the events in memory

When the events are filtered by a site location, and an occultation by a planet occurs at that site, the event line gives contact times at that location. The information for Date, UT and Diameter in km is the same as for other lines. Thereafter the information given is:

  • The name of the planet
  • The star identification
  • Visual magnitude of the star
  • Solar elongation of the star
  • %illumination of the planet
  • For the Disappearance event, the time, PA on the planetary limb, and altitude of the star
  • For the Reappearance event, the time, PA on the planetary limb, and altitude of the star

Filter options

The event list can be filtered on a range of options, using check boxes at the top of the form. The status of those check boxes is retained between sessions.

To filter the list, set the check boxes, then click List events.

The various filters that can be applied are:

=================================================


Filter on whether or not the event is visible from the specified location. Includes events if, at the specified location and time of closest approach to that location

  1. For stars brighter than mag 4, the only criterion is that the asteroid must be at least 1 deg above the horizon
  2. For stars fainter than mag 4 - no events if the sun's altitude is greater than -1 deg. And no events if the asteroid's altitude is less than 5 deg and the sun's altitude greater than -9 deg.
  3. A general star faintness test applied for varying altitudes of the sun. The limits are 4.0 @ -1°, 7.9 @ -4°, 10.5 @ -6°, 13.2 @ -8, 15.8 @ -10°

The latitude/longitude values are retained between sessions. You can use the site files to insert the coordinates of a selected site, by clicking the Select site button.(This also sets the telescope aperture in the next filter group.) This will also allow you to insert your home site location (set in the User Settings) by clicking the Home button on the form that appears to select from the site files.

The longitude and latitude values here are also used as the default site location for plotting the occultation maps.

If this filter is selected, you can also set

  • the probability of the event occurring at the specified site location
  • the distance of the path from the location (km) OR the minimum distance of the asteroid from the star, and the altitude at that location The path distance is intended to limit events to those where the observer is prepared to travel to the occultation region. However the distance calculation is approximate, and becomes unreliable when the distance is of the order of 1000km or more or the star is at low altitude.
  • the sky-separation of the star and asteroid. This may be more relevant for filtering events that might occur at the home location.
  • exclude events that occur below a specified altitude at that location.

You can also filter on the basis of your local horizon. This allows you to include the effects of trees, buildings, mountain ranges, and the like. For this filter to be functional, you need to set your local site horizon. You do this from the Menu bar - File... Set the elevation of local horizon. This will bring up the form:

Simply set the horizon altitude (in degrees) that you want to apply for the azimuth range above each entry. To retain the settings, simply click Save & Exit. Undo all changes will return all fields to the value they had when the form was opened. Set all elevations to zero does just that!. And Exit with no changes quits the form without saving any changes.

 

 =================================================


Filter on any of the following parameters:

  • the expected magnitude drop
  • the star magnitude
  • the expected maximum duration of a central occultation
  • the diameter of the occulting asteroid (including the moons of major planets)
  • the solar elongation of the star. Usually only events involving a quite bright star cab be observed at solar elongations less than 20°
  • The capabilities of your optical recording system. You can specify the telescope's aperture (up to 95 cm) and whether or not you are using a camera where you can set the exposure duration (including integrating cameras). These will be set on the basis of settings in the Site file (if you use that option in the previous group).
    This test works on the basis (assumption) that a 20cm telescope, using an exposure of 0.037 secs (half-way between PAL and NTSC video frame rates), can reliably record a star at magnitude 12.0 under reasonable conditions. Changes in the aperture appropriately changes the magnitude limit. If the exposure can be varied, the magnitude limit is set such that at at least 3 exposures can be taken during the maximum duration. If the predicted magnitude drop is less than 0.5 mags, a larger number of exposures must be possible during the maximum duration.

=================================================


This filter identifies those events where the occultation path (as defined by its limits) crosses into a 'quadrilateral'. This filter is new to Occult 4.5.

The purpose of this filter is to identify all occultation events that might occur in a region. That region might be defined by the various locations of a group of observers, or by the region an observer is prepared to travel to observe events.

This filter will exclude all events where at least some of the path does not pass across the defined quadrilateral. However it does not test for the Sun's altitude or the objects altitude - you need to apply a different filter for those.

To set the filter, the user needs to specify the four corners of the quadrilateral area defining the region. Those coordinates are specified as latitude/longitude coordinates, in degrees. There is no requirement for the quadrilateral to be in any way regular in shape. To assist in specifying the coordinates, the box indicates the traditional directions of NW, NE, SW and SE - but in fact the ordering of the four corners of the quadrilateral is irrelevant.

The region can be specified by manually entering the coordinates of the four corners of the quadrilateral. However, more conveniently you can specify multiple quadrilaterals by using the Menu option File... Edit list of regions, and select any of the regions you have created from the drop-down list of regions at the top of this filter box.

Using the Region filter

The region filter can be used in several different ways:

  1. For personal predictions, you can specify a relatively small quadrilateral surrounding your home site, to create a filter that identifies events that come close to you.
  2. For personal predictions, you can specify a quadrilateral surrounding your home site, taking into account roads & topography where you might be prepared to travel. This will create a filter that identifies event that cross the region you are prepared to travel to - rather than just being close to you.
  3.  For creators of feeds for OccultWatcher. You can set up a region that matches the distribution of observers in your part of the world.
  4. For creators of feeds for OccultWatcher - where you have discrete groups of observers. By way of Example, in Australia/New Zealand region, there are groups of observers in Western Australia near Perth, in South-Eastern Australia (Melbourne/Canberra/Sydney), in North-Eastern Australia (Brisbane/Townsville), and New Zealand. To create an optimal feed, what one can do is:
  • specify regions for each of the several locations
  • run the search for the desired time interval
  • select a first region using the File... Select Region... submenu, and filter the events for that region (applying any other filters as appropriate). Then Save the list using the Save listed events button  (or the equivalent menu item).
  • select a second region using the File... Select Region... submenu. Then Save the list using the Save listed events button
  • Repeat for each of the regions
  • Now, using File... Open, open one of those saved files. [This will result in the loss of the overall search results, unless you have saved them.]
  • Then, using File... Open and Merge, open each of the remaining files you have saved.

This will give you a list of all events that cross one or more of your selected regions (with any duplicate entries being removed), from which you can create an optimized OccultWatcher feed.

Features to note in this list are:
  • The system has three components: Camilla, S2001-107-1, and S2016-107-1
  • The Miriade system has three different orbit solutions (represented by #1, #2 and #3). However orbit solution #3 is only available for two components.
  • One entry is unqualified by a # symbol. [It need not be the first entry.] This entry is based on the normal ephemeris for the asteroid, having no regard to any information from the Miriade system. For systems where the satellites are small compared to the primary, the prediction for this unqualified event will almost certainly be the same as for the events qualified by #1, #2 etc. However in systems where the masses of the two components are similar, each component will be offset from the system's barycenter such that the unqualified prediction will unreliable.

Each event in such lists are treated as entirely different events, and all the usual outputs are available in the usual manner by clicking on an event.

However there may well be special interest in binary asteroid events, and the following provides some guidance about how you might select & display relevant entries. In all instances, if you have a search covering multiple events, the starting point will be to use the Date filter to limit the output to the relevant date, and the asteroid number filter (but not name) to limit the list to the one event and asteroid system.

  • To select events for a particular solution: In the filter for Asteroid name contains, enter the solution tag (and nothing else). In this case, the options are #1, #2 or #3
  • Having limited the list to a single solution, you can sensibly plot the paths of all components for that solution in Google Earth, using the menu item  File... Plot all paths in GoogleEarth.
  • Similarly, you can create a .KMZ file including the paths of all components for that solution (which can then be distributed or used on a web site), using File.... Save all paths as GoogleEarth KMZ file

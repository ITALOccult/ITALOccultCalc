Tycho2 star catalogue

Tycho2.bin is the main catalogue used for predicting occultations by asteroids and planets. It is been created from the Tycho2 catalogue and the 2UCAC catalogue. It is structured for highly efficient searching, but does not contain error information for the star positions. For information about how the file was created, see: Tycho2 creation

The format of the star catalogue is a 27-byte record length random-access file with the following record definition.
BytesMeaningDecode with:
0 - 3 RA in deg. Stored as a long integer. Decode by dividing by 107, and adding 180RA in deg
4 - 7 Annual Proper motion in RA stored as single. Units arcsec
9 - 11 Dec. in degStored as long integer. Decode by dividing by 107
12 - 15 Annual Proper motion in Declination stored as single. Units arcsec
16 - 17 Parallax Stored as Integer. Divide by 104 to give units or arcsec
18 - 19 Mv Stored as Integer. Divide by 100. Mv=90.00 when not given in source cat.
20 - 21 Mp Stored as Integer. Divide by 100. Mp=90.00 when not given in source cat.
22 Catalogue ID Stored as byte.
23 - 26 Catalogue no. Stored as Long integer.

For Hipparcos stars - number given directly

For Tycho stars: last 5 digits are number in Zone. Integer divide by 105 to get zone number, e.g. 777401412 = TYC 7774 01412

For USNO-B1 stars, read this number as an Unsigned integer. Then
  • Zone = Catalogue number >>21
  • Number in zone = Catalogue number & 0x1FFFFF

    • The catalogue ID byte identifies the following information:
    If ID = 2 - star is a Hipparcos star.
    If ID = 130 - star is a UCAC2 star that is not in Tycho2
    If ID = 135 - star is a USNO-B1 star

    If ID and 8 is true, the star is a Tycho-2 star, and the following extra parameters apply:

    • Component number = (ID and 7) +1
    • If (ID and 16) is true - proper motion not available
    • If (ID and 32) is true - the Tycho catalogue had separate positions for the components of a double star, but the catalogues used for proper motion only gave a combined position. The position given is based on the separate Tycho positions, using a proper motion derived on the combined position.
    • If (ID and 64) is true - the Tycho catalogue indicated that the star was a double star with the position being for the photometrically combined object.
    • If (ID and 128) is true - the star is a Tycho star with co-ordinates taken from UCAC2

    Occult displays these extra parameters in the following manner:
    • TYC 7774 - 01412 - 1 (full number - with component identifier)
    • TYC 7774 - 01412 - 1p (no proper motion)
    • TYC 7774 - 01412 - 1s (separated stars)
    • TYC 7774 - 01412 - 1d (double star, unseparated)
    • TYC 7774 - 01412 - 1u (position derived from 2UCAC)

    If the catalogue is "Goffin.bin", the following additional catalogue definitions apply:

      1 = PPM
      2 = HIP
      3 = TYC
      4 = TAC
      5 = CMC
      6 = ACRS
    and the star identifiers are given as:
      For PPM, HIP, CMC and ACRS - number given directly

      For TYC and TAC:

      • last 5 digits are number in Zone.
      • Integer divide by 105 to get zone number
      • For TAC numbers, a -'ve number indicates a negative zone
      E.g. -1205345 = TAC -12 05345
      E.g. 777401412 = TYC 7774 01412

    The file Tycho2.inx is an index catalogue to identify field locations within Tycho2.dat. Tycho2.inx is accessed as a random access file of length 4. Each entry in Tycho2.inx is a record number, stored as a long integer.

    The data in the index file is held as follows:

    • For each 1-deg band of declination there are 361 entries - giving a total of 64980 entries to cover the 180 bands from +89 to -89.
    • Each entry in a band corresponds to a 4min (i.e. 1 degree) increment in RA, from 0h 0m to 24h 0m (i.e. 0deg to 360deg)
    • The value of the entry is the record number of the first star whose RA is greater (or equal to) the RA increment corresponding to the entry.
    Thus, record 3 will contain the record number of the first star whose declination is in the +89d band, with an RA greater than 0h 08m (i.e. 2deg). Record 364 will contain the record number of the first star whose declination is in the +88d band, with an RA greater than 0h 08m (i.e. 2deg). Record 384 will contain the record number of the first star whose declination is in the +88d band, with an RA greater than 1h 28m (i.e. 22deg)

    Particular stars can be found in the file using:

    • For Tycho star numbers, use GSC Fields to find the RA and Dec. limits for a scan of the file
    • For Hipparcos stars, use Hipparcos Index to find the record number (unary-based) in the Tycho2.bin file.
    • For 2UCAC stars, use UCAC2 Index to find the start and end records (unary-based) to conduct the scan in the Tycho2.bin file.

    Creating the Tycho2 catalogue

    The following is detail about how the Tycho-2 and Hipparcos catalogues were processed to generate the Tycho-2 catalogue for use in OCCULT.

    Identity and Position processing

    In Hipparcos catalogue:

    263 entries had no precise positions. These stars were not cross-referenced from the Tycho-2 catalogue - so their null positions have not been transferred. Most of the 263 stars have entries in the Tycho-2 catalogue.

    In Tycho-2

    For stars marked "X" in column 14 - ie no proper motion available. Position at epoch (~1991) used. 'Catalogue Name' field OR'ed with 16, to indicate this status.

    For stars marked "P" in column 14 - ie photocentric position of double, each component has an entry. The position used is the position at epoch of the component, with proper motion for the combined object being applied to the component to refer the position to J2000. 'Catalogue Name' field OR'ed with 32 to indicate this status.

    For stars marked "P" in column 201 - ie photocentric position used for a double star, with the components unresolved (& thus only a single entry.) No manipulation of the coordinates, but the 'Catalogue Name' field OR'ed with 64 to indicate the double status.

    Correlation between Hipparcos & Tycho-2

    Hipparcos positions were used in place of the Tycho position, in the following circumstances

    • The Hipparcos reference number in Tycho has no component identifier;
    • The Hipparcos component identifier in Tycho exactly matches the component identifier in Hipparcos;
    • The component identifier in Tycho is either '*' or '-', and Hipparcos refers to a specific component;
    • Tycho specifies an 'A' component (with or without others), but Hipparcos uses '*' or '-' ;
    • Tycho identifies more than one component (eg AB), and Hipparcos refers specifically to one of those components; and
    • Seven Hipparcos stars (2444, 29737, 39606, 42543, 54976, 71377, and 110766). These stars are flagged as '*' in Hipparcos, but have a specific identifier in Tycho that does not involve an 'A' component. [See item 4 above.]

    Two stars required special treatment.

    • H27000 - flagged as double with '*' in Hipparcos, but separate entries in Tycho Supplement 1 (as components D and E). Entry consists of parallax from Hipparcos; all other details from Tycho.
    • H67626 - flagged as double with '*' in Hipparcos, but separate entries in Tycho (with no component identifiers) having no proper motions or parallax. Entry consists of Hipparcos proper motion and parallax, and Tycho positions (moved to J2000) and magnitudes.

    Magnitude processing

    In Hipparcos:

    Johnson V magnitude given for all but one star (which was a star with no precise position - H120412)

    Johnson B-V given for all but 1281 stars. This was combined with V to give the B magnitude. For another 344 stars, a Johnson B magnitude was derived from the magnitudes VT and BT using the approximate expression:

    B=.76*BT+.24*VT

    For the remaining 931stars the B, magnitude was set as 90. [No BT magnitude was available for any of these stars.]

    In Tycho-2

    The Johnson B and V magnitudes were derived from the BT and VT magnitudes as

    V=1.09*VT -.09*BT

    B=.76*BT +.24*VT

    Where one of those magnitudes was not available, no correction was applied to the other magnitude.

    Adding UCAC2 data

    The catalogue has been modified with the availability (August 2003) of the UCAC2 catalogue, by:

    • replacing the positions/proper motions of all non-Hipparcos stars of magnitude fainter than 9.0 with those from UCAC2; and
    • adding UCAC2 stars of magnitude brighter than 12.00 that were not in Tycho2

    The criteria used to replace positions with UCAC2 data were:

    • no replacement if a star in Tycho2 is within 2” of another star, or Tycho2 magnitude brighter than 9.00
    • replacement if a UCAC2 star matched the Tycho2 star to better than 1” in RA and Dec; BUT
    • no replacement if more than one UCAC star matched the Tycho2 star

    The criteria used to add a UCAC2 star were:

    • magnitude <12.00 on the UCAC magnitude system; and
    • if UCAC2 star matched a Tycho2 star to better than 1.8” in each coordinate - not added
    • Mv and Mp were both set to the UCAC2 magnitude - which is slightly redder than V.

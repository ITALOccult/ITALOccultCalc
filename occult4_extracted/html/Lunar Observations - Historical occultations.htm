Historical occultations - Overview

This form gives access to the data for observed occultations (other than Grazing occultations.) The functionality that is provided includes:

  • selecting all observations for a specified star, and reducing those observations
  • selecting all observations where the observer's name, and/or site name, includes a specified string;
  • reconstructing observation reports for any specified site
  • viewing any observation in the file, including tools for detecting the cause of any error in the reported observation

The observations fall into two groupings:

  • those collected by Royal Greenwich Observatory (RGO) - which cover the period 1623 to 1980; and
  • those collected by the International Lunar Occultation Centre (ILOC) in Japan - which covers 1981 to the present.

The data is held in a series of files, as follows

  • The actual observations, in a consistent format, as a series of files. Each file covers a span of years. The files have either RGO or ILOC as part of their name.
  • Files containing the data concerning the site location and observer. For the RGO data, there is a single file containing limited information on the site and observer. For the ILOC data, there are separate files holding information for the site, and for the observer.

The observations in all the files have been processed to identify and remove a number of errors - details of this processing are here.

The RGO dataset is missing observer names in the period 1943-1971. The reasons are explained here.

NOTE The ILOC dataset includes observations for 2002 to 2005, which were received shortly before release. The observations were received in a different data structure to previous years - due to certain problems ILOC had with their previous data structures. In the medium term the entire dataset of ILOC observations used in Occult will need to be reformatted, with consequential major code changes. In the short term the data is provided in the 'old' structure so that it can be accessed in Occult. Unfortunately until the ILOC data is fully reformatted into a new format, Occult cannot associate an observation with the observer's coordinates for many observations in the period 2002 to 2005 .

People wanting to research the oldest lunar occultation observations should refer to Simon Newcomb's Researches on the Motion of the Moon, which analyses lunar observations made prior to 1750. This provides a useful understanding of the issues associated with times of old observations.


To view all observations of a particular star

  • Set the option button to search for a star. The following controls will appear.

  • Specify the star of interest - you can select the star from the drop-down box, or enter its number in any of the ZC, SAO or XZ boxes. For the drop-down box there are 6 sets of names available under the menu item 'Name of Star' list. Those sets are:
    • All names
    • Proper names
    • Bayer letters
    • Magnitude brighter than 2
    • Magnitude brighter than 3
    • Magnitude brighter than 4
  • To clear the star number fields, set the XZ number to 0.

    Double star options

  • To highlight observations for the particular star where the report indicates actual or possible double star circumstances - check the Highlight box.
  • To list only those observations of the particular star where the report indicates actual or possible double star circumstances - check the only list doubles box..
  • To list all reports of unidentified double stars, check the list ALL unidentified doubles box.

    Additionally, you can generate a set of double star report files for use with the Double Star analysis. The process requires manual selection and ordering of observations, and is decribed here.

    click Read to display the list of events.

    The file will be displayed. Selecting a record will result in the record being decoded and reduced, in the box at the bottom of the form. Also given are the observer's co-ordinates, and the name of the observer (which are retrieved from a separate file of observer names.)

    If the Highlight check box was checked, potential double star reports are highlighted. Note that in this mode, clicking an entry will turn a highlight on or off.

    A star will be identified as possibly double if the Double star field is set, or if Remarkable Circumstances is set to Not instantaneous, gradual [that is, a value of '1']. [Note - the Remarkable Circumstances field is 3 columns to the right of the double star field]

    Under the list are some edit facilities for adjusting parameters where an observation would appear to be erroneous. Adjustments can be made to the year, month, day, hour or minute of the event, as well as the star number. [Note - this adjustment facility does not make any changes to the data held in the file - it merely facilitates identifying what the correct data should be.]

    Reduction of all observations

    You can generate a file containing the reductions of all observed occultations of the particular star, using the menu items with List... Reduce all Valid & Save.. or with List... Reduce all & Save.. The 'all Valid' option limits the output to events that have a phase of D, R, B or F, and which have not been flagged as invalid. The reduction process is apparent from the form as each event is displayed at the bottom of the form. The reductions are saved to a file in the directory ...Occult\Observations\Results. The file name will be (in order of preference) the ZC number, the SAO number or the XZ number of the star - and is a .txt file.


    To view all observations made by a particular observer name, or at a site name

  • Set the option button to search for name. The following controls will appear.

  • Specify the characters you want included in the Observer's name field, or the Site name field. To ensure all instances of an observer or a site are retrieved, you need to limit the characters to those for which you are confident will appear in all instances of the observer's name or site. In doing this you need to consider pooptential format variations - such as whether the oberver's personal name is present as a full name, an initial followed by a space, or an initial followed by a decimal point (with or without a following space). Note that wild cards (?, *) should not be used. THe check made is a simple check of whether the observation record contains the specified text. Note also that the comparison made is not case-sensitive.

    To view all observations made from an RGO or ILOC site

  • set the option button to read a file
  • from the drop-down box, select either RGO sites, or ILOC sites, as appropriate.

    The site file will be displayed, and the buttons to Create or Merge a report from a site are enabled.

    The information provided for each site is: The data given for each site is:

  • Station number (3 digits for RGO, 5 for ILOC)
  • Telescope number (2 digits)
  • Longitude
  • Latitude
  • Altitude (m)
  • Station name (generally Country, City)
  • Telescope aperture (cm) [ILOC only]
  • Telescope focal length (cm) [ILOC only]
  • Telescope code (type, mounting, driven) [ILOC only]
  • Number of observations for the site
  • Year range for the observations
  • Name of observer [RGO only]

    Note: The ILOC files have the observer names in a separate file that is not linked directly to the site details, the observer name does not appear with the site details.

    The list of sites can be sorted using the Sort sites menu item, by any of

  • Number of observations
  • Site code
  • Longitude
  • Latitude
  • Observer name [RGO sites only]
  • Site Name

    Viewing observations from a site

    You can view and analyse all the observations from a particular site by transferring the observations for that site into the Lunar Occultation Observations Editor. To do this:

  • select the site of interest; then
  • click the Create report for.... button [which will have the site identification listed in the button.]

    You can add the observations of a site to those from another site by clicking the Merge report for.... button.

    Once the observations are in the editor, you can use all the functionality of the editor to reduce and plot the observations, and to save them in a separate report.

    Suggestion. If you have submitted occultation reports from several locations, you can create a consolidated list of all your reported occultations - by merging them into a single report.

Observations editor

The observations editor is the tool for recording new observations in the IOTA2009 format, and analysing old observations.

The editor can read a number of 'old' formats of occultation reports [Including the '76' format], but writes all reports into either the IOTA2009 format, or a specific ILOC compatible format that includes the full year in the report.

To deal with reports created outside of the Occult editor, star numbers are checked (& automatically converted when required) to be in the heirarchy of ZC, SAO, XZ. Also where only one of the Method items has been completed, the progam ensures the first method item is used in preference to the second.


Structure of the observation editor

The observation editor has 4 pages:
  • a page for viewing a formatted report details
  • a page for editing the Header details
  • a page for editing site details and observer names details
  • a page for editing the observations details The ability to submit the report, to view observer site locations, and to reduce observations, is available from all four pages.

    The ability to submit a report with respect to a double star occultation (to be combined with other observers to derive an accurate position angle and separation.)

    Functionality to support the reduction and archiving of observations by regional collectors is provided on the form that is shown when observations are reduced.


    Editor functions

    To create a new occultation report

    Click the File.. New menu item. A new report is created.


    To open/display an existing report

    For observations in a file:
    Click the File... Open menu item, and select the relevant file

    For observations on the ClipBoard (e.g. copied from an email message)
    Copy the entire observation report
    Click the File... Paste menu item.


    To merge an existing report with a report in the editor

    Occult can combine reports into a single report. This is particularly useful for graze reports where you can combine reports from different observing teams into a single report for joint analysis.

    When reports are merged, the sites and observer names are compacted into contiguous series, and the events are sorted by time. After merging, it may be appropriate to sort the observations by site and then renumber the events.

    For observations in a file:
    Click the File... Import and merge menu item, and select the relevant file

    For observations on the ClipBoard (e.g. copied from an email message)
    Copy the entire observation report
    Click the File... Paste and merge menu item.


    To Save a report

    Options to Save, and Save as... are provided under the File... menu. The format of the save is the IOTA2009 format.
    You can save in the 'old' format, using the Save [old format], and Save as... [old format] under the File... menu. Note that if you load a file written in the IOTA2009 format, and then use Save [old format] to save that file, you will overwrite the new-format file with an 'old-format' file - and that may result in loss of some data.


    To verify data entry

    Click the File... Verify data entry menu item. This will perform a data validation check of the current data. This data check is also done whenever a file is loaded, merged, or pasted. Details


    To identify unidentified stars

    When working with a file that contains unidentified stars, or stars identified by star numbers that are not ZC, SAO ar XZ numbers (or planet or asteroid numbers), click the File... Identify unidentified stars menu item. For each such entry possible stars are listed for selection. Details


    To report the observations

    Click the Submit report... Submit report menu item. Make sure you are connected to the Internet. You will be asked to confirm that you want to report the obervations, and then taken to a form where you select the address to report the observations. Details Note - the ability to email observations from Occult assumes you have an email account with your ISP, using an 'SMTP' server. If you use a browser-based email service (such as Hotmail or GoogleMail), you will have to manually create the email and attach the observation file.

    The Email addresses used to report observations may change over time. Click the Submit report... Download current Email reporting addresses to retireve the current reporting address..


    GoogleEarth functions

    View current site locations

    You can easily check for any data entry errors in site coordinates by plotting the sites in GoogleEarth. To do this, simply click the menu item Google Earth Plot current sites. the sites are converted from the Horizontal datum specified in the report to the WGS84 datum before plotting.

    The relevant .kmz file is 'ViewGoogle.kmz', and is located in the Autogenerated Asteroids subdirectory.

    Plot moon limb

    You can plot the limb of the moon on the Earth at the time of some or all of the events in the report. The limb is corrected for the Watts limb (including the Morrison & Appleby corrections). The menu provides choices to plot the first 10, 20 or 50 events, or all events. under Google Earth plot moon Limb - first xx events. There is also a data-rentry field in the drop-down menu to specify the approximate altitude of the site of interest. Note that each limb plot can involve up to 1,800 longitude/latitude coordinate pairs. Consequently if a large number of events are plotted, it will take a little time to generate the correspondingly-large kmz file, and GoogleEarth will become sluggish in its operation. The observing site for the events will be located at the common intersection of all the limb plots. The limbs do not intersect at a common location - but over a region of 2 or 3 km - mainly for the following two reasons:

    • The limb is drawn assuming the site has an altitude of zero meters. If the event was observed at a low altitude, the limb may be displaced by (site altitude)/tan(moon altitude)
    • The limb is drawn on the basis of the Watts charts, and will typically differ from the actual profile by several hundred meters. In some regions it may differ by several km.
    Generally the site will be near the centroid of the intersection points.

    You can selectively turn plotted limbs on or off in the Places, Temporary Places area of the GoogleEarth form. Note that if the lunar limb is not entirely on the Earth, it will be created as two arcs, separated at a PA of 0 deg, and starting/ending where the limb leaves the Earth. The icons are colour coded to match the colour on the plot. Because of this issue, clearing the first of the two icons might not remove the limb in the region of interest - you may need to click the other icon.

    This functionality is especially useful for investigating a report where you suspect the site location might be in error, If the limbs all intersect at a common location, the true site location is in that region (and typically this will identify an obvious error in the specified site coordinates - such as digit transposition. However if the limbs are scattered, it may be assumed that there is some other error infecting the observations - such as an error in the time.

    The relevant .kmz file is 'Moon Limb Plots.kmz', and is located in the Observations subdirectory.


    LiMovie

    The LiMovie tool generates plots of occultation light curves., and includes functionality to copy the light curve to the clipboard - but no functionality to save it. This menu item will save the image copied to the clipboard from LiMovie (or any image copied to the clipboard from any other program).


    To reduce and display the observations

    Click the Reduce and Plot button.


    ConvertOldGrazeFiles

    If Administrator access is enabled, the menu will show a dis-abled item ConvertOldGrazeFiles. This is a place-holder for a routine that converts the graze files of earlier versions of Occult to the IOTA2009 format, and should be ignored.


    Y2K issues

    The files of ordinary and grazing occultations maintained in Occult include observations as far back as 1623. The report format used by ILOC (and variants such as the '76 report format) only include the last two digits of the century year. This data structure is a classic Y2K problem, and is incompatible with maintaining a database extending over several centuries. For this reason, Occult uses a variant of the ILOC format that includes the full year number.

    When an old ILOC or '76 report is read into Occult, it is necessary to complete the year number. The digits to be added are set in the User settings, under the box 'Lunar occultation reports'. The value is normally set at '20'. But if you are reading an old report, it may be necessary to change the value to '19'

Asteroid Diameters

Two files of asteroid diameters are provided. Both files are based on Infra-red satellite measurements, supplemented with a small number of TNO asteroid determinations. One file gives a weighted mean diameter, while the other file gives all the available measures (providing there is an associated uncertainty value).

The satellite missions/results used in these files are:

  • IRAS Minor Planet Survey (mimps). Datsets: diamalb and single, V6 at NASA PDS
  • MSX Infrared Minor Planet Survey (mimps). Dataset: mimpsalb - V1 at NASA PDS
    The Midcourse Space Experiment (MSX) satellite. (mimps are) of areas of the sky either missed by IRAS or in which the IRAS sensitivity was degraded by confusion.
  • AKARI AcuA  (Asteroid catalogue using Akari) Revised - at VizieR
  • NEOWISE Near-Earth Object Wide-field Infrared Survey Explorer, at NASA PDS. Datasets: centaurs, hildas, jupiter_trojans, mainbelt, neos
  • TNO and Centaur Diameters, Albedos, and Densities, at NASA PDS. Dataset: V4

Together, these sources provide 186036 measures of 125647 asteroids, with numbers up to 522276 (as at June 2018)

Binary asteroids - Splitting the diameter into two bodies

If an asteroid is discovered to be a binary asteroid (that is, two occultation chords of similar length with each being more than ~50% of the expected diameter of the asteroid), the diameter from these sources is proportioned between the two components on a R2 basis, not R3 basis. The reason is (personal communication with NEOWISE team, and applies to all satellite missions):

The diameters we determine are effectively the time averaged projected circle-equivalent diameters, although the model under the hood is actually fitting spherical equivalent surface area.  Either way, an R^2 scaling would be appropriate, caveat any self-heating that we aren't modeling (but would probably be an important component for a close, equal-mass binary).

As an example. Assume chord lengths of 14 and 12 units, for a total of 26 units. These equate to 0.54 and 0.46 of the total chord length. If the diameter is X km, the corresponding diameters are:
  D1 = X *√0.54  and  D2 = X√0.46
To compute the associated uncertainties, repeat these calculations using the upper and lower values of the diameter + uncertainty.

AsteroidDiameters.bin

This binary file holds a weighted mean diameter for asteroids in the number range 1 to 550000. The diameters are multiplied by 10, and held as a 2-byte (int16) unsigned integer, with the file position corresponding to the asteroid's number -1. Accordingly an asteroid diameter to 0.1 km can be directly read from the corresponding location in the file with a resolution of 0.1 km (after dividing the integer value from the file by 10).

Additionally, the file holds the weighted uncertainty in the diameter. The uncertainties are  multiplied by 5, and held as a byte, with the file position corresponding to 1100000 plus the asteroid's number -1. Accordingly the uncertainty in  the asteroid's diameter can be directly read from the corresponding location in the file with a resolution of 0.2 km (after dividing the byte value from the file by 5).

The following approach was used to derive the weighted mean.

  1. For AcuA, the uncertainty was taken to be the listed uncertainty (the minimum uncertainty was either 10% or 20% of the listed diameter).
  2. For Neowise, the uncertainty was taken to be the listed uncertainty + 10% of the measured diameter (in accordance with the Neowise documentation)
  3. For IRA and MSX, the uncertainty was taken to be the listed uncertainty + 10% of the measured diameter for asteroids with more than 4 observations, and the listed uncertainty + 14% of the measured diameter for asteroids with less than 5 observations.
The zero-based file format is:

For diameters
Bytes Format Content
0-1 int16 Diameter x 10 (km)
with 660000 entries starting at byte 0

Values less than zero are the negative of the diameter estimated from the asteroid's H0 absolute magnitude.

For Uncertainties
Bytes Format Content
0 byte Uncertainty in dia x 5 (km)
with 660000 entries starting at byte 660000

for a total file size of 1,980,000 bytes

 

AsteroidDiametersAll.bin

This file contains weighted measures from each of the satellite missions, giving the asteroid diameter (km), the uncertainty (km), and the source. The zero-based file format is:

Bytes Format Content
0-3 int32 Asteroid number
4-5 int16 Diameter x 10 (km)
6 byte Uncertainty in dia x 2.5 (km)
7 byte Source
8 byte Albedo x 500, range 0 to 0.50

The values for source are:

0= from a visiting spacecraft
1= Neowise
2= AKAR AcuA
3= IRAS
4= MSX mimps
5= TNO Centaurs etc

Note that:

The diameter is obtained by reading the relevant integer, and dividing by 10 (as in AsteroidDiameters.bin). However the uncertainties associated with the various missions are larger, and are decoded by dividing the relevant byte by 2.5 - giving a resolution of 0.4km.

To retrieve the data for a particular asteroid, you need to search the file for the asteroid number (if doing a binary search, once you get a match, go back at least 6 records, and then scan forwards to get the first matching asteroid number.)

 

AsteroidDiameters_Indiv.bin

This file contains the individual diameters that are provided in Akari AcuA, and NEOWISE. Decoding the values is the same as for AsteroidDiametersAll

Bytes Format Content
0-3 int32 Asteroid number
4-5 int16 Diameter x 10 (km)
6 byte Uncertainty in dia x 2.5 (km)
7 byte Albedo x 500, range 0 to 0.50
8 byte Source

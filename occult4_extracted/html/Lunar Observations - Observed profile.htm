Observed profile

The reduction profile is plotted slightly differently to the graze prediction profile. In the reduction profile, the smooth limb of the moon is represented by a horizontal straight line.

Basic details of the profile

  • the observed event, or graze events, is plotted as a thick circle - Red for D, green for R, grey for other types of events.
  • the range of Axis Angle, and height, is set from the data plotted. The ranges can be manually changed using the boxes at the top of the form. [Note - the numeric value of the Axis angle at the left must always be less than the Axis Angle at the right.]
  • the mean limb of the moon - is represented by the horizontal axis. The vertical axis gives the height of the limb or event above the mean limb.
  • the lunar profile. There are 4 options available, selectable using check-boxes at the top-left of the form.
    • LOLA data points. This plots the altimeter data points from the LRO LOLA, leaving it up to the user's eye to identify the location of the limb.
    • LOLA Hi-resolution profile line. This plots a 'best-fit profile line to the LOLA 
    For more information about the LOLA data, see LOLA data.
  • Observations from past grazes and occultations [optional], plotted using the following coding:
    • squares - graze events, either visual or instrumental;
    • circles - instrumental observations (eg video, photoelectric);
    • '+' - visual observations
  • information identifying the star, date, and librations used for the profile prediction.
  • The Axis Angle and height above the mean lunar limb at the location of the cursor, is shown as a tool-tip.

Observed graze and occultation data

Occult provides the ability to plot observed grazes and occultations against the profile, thereby allowing you to compare your current observations with past observations. The several files of results from past occultations are available as a separate download, with the file for the most recent events being regularly updated.

The options available to plot past observations are provided under the menu item 'Observed graze and occultation data', and are as follows:

  • however ordinary occultations are often the only source of information for the relevant libration.

    Additionally, the path of the star as seen from that observer's location can be plotted, by checking the box Include star path in graze plot at the top of the list of current observers. If the observer reports observations of separate components of a multiple star system, paths will be drawn for up to 5 observed components.

    List Historic observersThis list the observers for the historic observations plotted against the profile
    Draw in colourControls whether the overall plot is B&W or colour

    When a profile is first drawn the settings used for plotting the observed data are taken from the User settings - 'Lunar Occultations - grazes' - 'Reduction profile - Librations for plotting observed data'.

    When observed data is plotted, you can:

    • identify the observer and event by placing the cursor on the plotted point. The details are given above the plot. The coding used is: Event codings:
      • DD DB DU - disappearance at the Dark limb / Bright limb / in Umbra
      • RD RB RU - reappearance at the Dark limb / Bright limb / in Umbra
      • B F M - Blink / Flash / Miss
      • Reliability codings - "ok" = certain, "?" = possibly spurious; "??" = almost certainly spurious
    • right-click on a plotted point and you can highlight (i) all events by that observer, (ii) all events involving that particular star, or (iii) all events on that particular day.
    • Additionally, if you double-click on a graze in the List of listed in the list of the grazes used, all events in that graze will be highlighted.
  • You can return to the original plot highlighting the 'current' observations by clicking the Re-draw button.


    Interpreting the plotted observations

    The libration reference frames

    A general discussion of the libration reference frames is given with the graze profile help file topic.

    Past observations should be plotted using the L, B, AA reference frame, using the smaller ranges of possible values. Avoid the P, D reference frame, which shifts data left and right according to perspective.

    Using grazes to determine double star separation and PA

    Where a graze observation records events form the components of a double star, and the WDS code has been set to A and B for the components. Under these conditions the 'double star control group at the top right of the form is enabled. Also the colors used for the two stars components are different - enabling the events involving a particular component to be identified. The extra functions available are:
    • check boxes to control whether or not the events involving a particular component are plotted;
    • up-down controls to adjust the separation and position angle of the B star relative to the A star, in arcsecs.
    • The reduction may already have incorporated a separation and position angle when coputing the residuals. The value so used is pre-set into the up/down controls, and is displayed next to the up/down controls.
    • The rate of change in the up/down controls can be varied by double-clicking. When the controls have the normal Window color, they change by 0.1" and 0.2 deg respectively. Double clicking will turn the control background to green - and the change will be 0.01" and 0.01 deg. Double-clicking again will return to controls back to their normal state.
    To use this functionality, adjust the Sep and PA values to bring the profiles determined by the two components into alignment. [Note: the plot is highly stretched in the vertical direction - with 1 deg in Axis Angle corresponding to about 16".].

    The derived Separation and PA can be copied to the clipboard from with Plot... Copy Double star solution. The copy includs extra information, as illustrated in the folloiwng example: 

    R2366
Double star solution from grazing occultation
Sep = 2.59"  [predicted 2.79"]
PA = 275.47°  [predicted 275.77°]
Date of graze: 1990.13
Number of events in the graze: 183

    Determining limb slope for double star solution

    NOTE. This functionality is largely superceded if the user has the LOLAR lunar limb installed. In this situation the form is merely used to view and confirm the lunar slopes; any slopes measured using this form will be replaced as part of the double star solution process.

    The observed profile plot is also used as part of the analysis of double star observations - to measure the slope of the lunar limb at the point of the occultation. The form can be opened from the form for solving double stars.

    The form has the following extra features:

    • the Axis Angle of the occultation is indicated by three vertical dotted lines. The central line is the observed axis angle. On either side is a line located approximately 1 arcsecond on either side.
    • Place the cursor near the limb at the occultation point, hold down the left button, and move the cursor. A line will be drawn commencine from the point where the button was first clicked. Use this line to measure the gradient of the lunar limb at the point of the occultation. The lunar slope is displayed in the tooltip text.
    • When the left button is released, the line will disappear, but the last-measured value of the lunar slope will appear just above the top of the profile plot region. From there, the value can be transcribed to the double star solver.

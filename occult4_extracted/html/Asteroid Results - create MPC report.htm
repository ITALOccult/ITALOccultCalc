Make / Submit Astrometry Report to the Minor Planet Center

This option creates the dataset to be sent to the Minor Planet Center to report astrometric positions. The option creates a .psv file in the directory Occult 4\Asteroids\Results\MPC Files\. with a name structured as:
MPC report_20241123_xyz.psv
where the numbers give the year/month/day the report was generated. and xyz indicates the report gives (x,y,z) coordinates.

Important. Before creating the report, you must ensure that the observations submitted in the previous report have had the MPC number added to the observation details If this is not done, the previous events will not be marked as having been reported, and will be reported a second time (which the MPC will not appreciate.)

Format of the Report

The report is created in accordance with the IAU standard for reporting astrometry - ADES. That standard can be accessed from this form, at the bottom.

The report has 5 sections. The first three are:

# version=2022
# observatory
! mpcCode 275
# submitter
! name A. Bnnnnn, full address
# measurers
! name C. Dnnnnn
! name C. Dnnnnn
! name C. Dnnnnn
# observers
! name E. Fnnnnn
! name E. Fnnnnn
! name E. Fnnnnn

The text in color are names ands an address to be included with each report. The names of the Submitter, and Measurers, are maintained in the Maintenance tab, User Settings, group 15 Administrative Functions. These settings are not distributed by Occult. Whoever has the responsibility to submit reports needs to maintain the list of these names in their copy of Occult.

Observer names

The list of observers is created from the observations being reported. The names are strictly Initial + Family name, and is sorted on the family name. Names of observatories or organizations are not allowed, and will usually be identified and excluded. Similarly for appellations such as Mr., Prof., PhD. Compound family names are allowed. While the program endeavors to extract names in the required format, the names should be checked for correctness and format [A. Name]. When the report includes names inconsistent with this requirement, names, a list of such will be displayed on screen - together with the relevant event details. When this occurs, you will need to go to the event, identify the relevant observer, and rectify the observer's name.

This file also contains the lines for measurers. This line is taken from the setting at the end of the User Settings page (which is blank in the distributed version). The names set are the principal individual collecting observations in each of the regional collecting groups.

Also required are the contact person, and acknowledgement email address. This is of the person submitting the report to the MPC. Again, the fields are set in the User Settings file.

4th section: telescope details

# telescope
! design reflector
! aperture 0.2
! detector CCD
These details are fixed. The fields must exist for the MPC processing. The values specified are are completely arbitrary.

Generating the report

For the routine submission of astrometry to the MPC, the left panel of the form should be set as illustrated. The only setting that should be considered for change is the Constraint 'Limit to before the end of', which is used to exclude events that occurred between the month specified, and today's date. The name of the file that is created, and its location, are as given at the beginning of this page.

Important. The choice  between x,y,z, and RA/Dec, must always be set to x,y,z for any astrometry sent to the MPC. The RA/Dec option must not be used.

The option 'Generate test report for yyyy only', is a test option that generates the report using just the observations of the current year. Not to be used for actually creating a report for submission

The option "Astrometry only, with extra data' generates a report with no observers listed, but with extra information about teach event. This is not for general use.

All reports generated are written to Occult 4\Asteroids\Results\MPC Files\. The file name will include extra elements to indicate the basis on which it was created. For example, if it relates to a specific asteroid, or observer name, that asteroid or name will be part of the file name. This is particularly useful when creating an astrometric report for a specific asteroid, for submitting to an orbit computer (such as the Horizons system). An example is

MPC report_for_(65803) Didymos_20240720_xyz_skrut.psv

where the file name indicates that it relates only to the asteroid (65803) Didymos, the fact it is an xyz report, and it is limited to events containing the name of an observer that includes the letters skrut

 

Submitting the report to the MPC

When you have created a report and want to send it, the steps (and relevant buttons on this form) are:

i. Validate report

The button Validate report will take you to  https://minorplanetcenter.net/submit_psv_test , which provides a test to confirm the report meets the ADES requirements.

ii. Submit the report

If the the report passes the test, the button  Submit report will take you to  https://minorplanetcenter.net/submit_psv to submit the reoprt. Follow the instructions on the page.  Upon submitting, there will be a delay of several minutes for some initial processing. You will then receive at the email address you specified a report from the address vmsops@minorplanetcenter.net. All being well, that email will report ....has been accepted for processing by the Minor Planet Center. This report is very important. It contains a Submission ID (which looks like 2025-06-00T16:03:33.676_0000GtUU), and a list of unique observation IDs (which look like LodF2TAu0000GtUU010000006) which are added to the Asteroid Observations file - see next section. You should move the email/copy the content of the email to a location where it can be easily retrieved

NOTE. There are certain characteristics of the report as it relates to occultation results, which will result in the submission being rejected unless the individual submitting the report has been authorized by the MPC to submit such reports. This rejection is not indicated by the test page - only when you submit.

iii Check processing status

Click Submission status to bring up https://data.minorplanetcenter.net/wamo/ . WAMO means Where are my Observations. Paste the Submission ID of a batch into the box, and click Submit. Information on what the report means is given vi links on the page.

iv Check observations are in the system

You can view the astrometry for any object by clicking MPC Explorer to bring up https://data.minorplanetcenter.net/explorer/ From here, specify an object. Then you can view all its astrometry, with residuals for many. Occultation astrometry is under Observatory 275.  To find these, sort the Observatory column to ascending.

 

Adding publication details to the observations file

This is an extremely important step. It is used to keep track of which observations have been submitted to the MPC, so that the astrometry from an event is not included in subsequent astrometric reports. Click the button in the Update observations file box. You will first be asked to select the relevant file you have submitted to the MPC. Then the following form will appear:

This gives two options.

  1. Add MPC Observations ID. This records the date the observations were submitted to, and accepted by, the MPC - together with the unique Observation ID
  2. Add Minor Planet Circular number for the observations. This adds the MPC number containing the summary of the observations.

 For both these options, locate the relevant email containing the list of Observation IDs (referred to above). Copy the entire text of that email (making sure all the ID's are included. (The presence of other text does not matter.) then click Paste observation ID's This will populate the box will all the observation ID's.

The first option should be done soon after you have received the acceptance report from the MPC. It adds the date of submission, and the unique observation ID, to each observation included in the pasted list. To do this, make sure that option is checked; specify the Date of submission, and click Add Observation ID's. A scroll bar at the top of the screen will indicate progress. When the list is fully processed, it will be re-displayed with either a tick or cross after each line, to indicate whether the date and observation ID have been added to the event.

The 2nd option require keeping a watch on the Minor Planet Circulars to find when & where they are published. The relevant address to retrieve the circulars is: https://minorplanetcenter.net/iau/ECS/MPCArchive/MPCArchive_TBL.html , with the relevant links being to the MPC######-### in the 2nd column - like MPC 177253-179246 The date in the first column provides an indication of the likelihood of the observations being in those MPC's

You can change the MPC details for an event from the observations editor, using the menu option File... MPC: Allow edit of date and MPC. This will display a group of edit boxes at the right of the form, just above the list of observers

Viewing astrometry at the Minor Planet Center

The Open MPC Explorer button will open your browser showing the MPC Explorer tool. Here you enter an object's identifier, and then list all reported astrometry for that object.

Enter the object identification, and press Enter. To view observations, click observations. Wait a little, and a large table will be displayed showing all astrometry or that object. If you want to look at what Occultation Astrometry has been reported, the easiest way is to click the column header Observatory, and select Sort Ascending. Then scroll down to find observatory code 275

ADES

The bottom block provides links to the ADES format specification, and the field values required by that standard.

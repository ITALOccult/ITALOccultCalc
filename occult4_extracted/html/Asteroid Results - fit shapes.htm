The Shape model control on the plot form facilitates the fitting of shape models and the retention of fits. The control looks like:

The Shape model control can be moved anywhere around (but inside) the plot form - by simply dragging it with the mouse.

The tools available on this control in normal mode are:

  • A series of buttons labeled 1 to 6 for both DAMIT and ISAM that allow the selection of particular shape models. Buttons are green when a shape model has been fitted, and plum whena shape model has not been fitted.
  • When a button is been selected, the button will be bright green if a model has been previously fitted, or pink when a model hasn't previously been fitted. The button number is also made bold.
  • Many models in ISAM are the same as models in DAMIT. When this occurs, a small white box with a red number will be next to an ISAM button, giving the number of the DAMIT button that has the same model
  • A button Light Curves, for downloading information about the dates light curves were obtained to generate the shape model, and the fit of the shape model to those light curves. This is only available for DAMIT models. The button is greyed out and disabled when an ISAM model is displayed
  • A box Source, which contains the source of the model being displayed (DAMIT or ISAM). It is set by the program (and forms part of the model fit data that is retained).
  • A box ID, which contains the model identification in the source. DAMIT models have an identification that is independent of the asteroid number. For ISAM, the asteroid number is implicitly part of the identification, and otherwise falls in the range 1 to 6.
  • A box labeled Surf/Vol. This gives the ratio of the Surface-equivalent diameter to the Volume-equivalent diameter of the shape model, and is fixed by code from the download. [The Surface-equivalent diameter is always greater than the volume-equivalent diameter.]
      By way of further explanation... A diameter is a quantity inherently associated with a smooth sphere. However an asteroid is an irregular object, which raises the question of - what is its diameter? There are two approaches to this issue. A volume-equivalent diameter is the diameter of a sphere having the same volume as the asteroid. A Surface-equivalent diameter is the diameter of a sphere having the same surface area as the asteroid. Prima facie, volume-equivalent diameters are relevant to assessments of density, whereas surface-equivalent diameters are relevant to assessments of brightness.
  • A drop-down box labeled Fit Quality, to be completed by the user. See below. If the current displayed value needs to be selected, simply click on the drop-down box. (There is no need to display the list and select from it.)
  • A box labeled Ph Corrn. The value here is automatically completed if the user adjusts the rotational phase of the asteroid - see below.
  • A button for changing the basis the the uncertainty regions on the plot. When Green with 'Err', the uncertainties represent the reported event time uncertainties. When Yellow with 'Dia', the uncertainties are reduced to match the uncertainty in the chord length. This button will get automatically cleared whenever the main Editor form is selected (although the plot won't change until the plot form is selected).
  • Boxes labeled Minimum dia and Maximum dia, which are set by the user by clicking the adjacent Set button - see below
  • A text box labeled Version Info - which gives information about the model that can be relevant to interpreting the results. The content is automatically set from the download. See below for its significance.
  • A slider bar for adjusting the size of the asteroid image on the plot. If this slider is to be used, it can be very helpful to check the menu item Keep form on top - as otherwise the display will oscillate between the Plot Event observations form display and the DAMIT and ISAM Shape Models form display. [Having clicked that menu item, you can return the form to its normal state by clicking the menu item Return form to normal]
  • A Save/update button to save the results of the fit of the shape model to the occultation chords. The label of the button changes to match the shape model being displayed. When a DAMIT model is selected, and there is a corresponding ISAM model, the Save/update button with indicate both the DAMIT and ISAM models, and will save/update both models.  
      IMPORTANT. This button stores the results of the shape model fit in memory. It does not save the results in an observations file for the event. The results are only saved when the observation as a whole is saved or updated. See near the end of this Help topic page for information about Saving/deleting the Shape model fits

A model can also be displayed for a range of rotational orientations. This is achieved using the menu item with Phase models in the Shape Model display form. When this is selected, the control is changed in the following manner:
  • The buttons for DAMIT and ISAM models are replaced by a series of buttons labeled 0 to 9 - corresponding to increments in the rotational orientation. Button 4 corresponds to the 'zero' point of the phase adjusted models.
  • Selecting a button allows the orientation of the shape model to be adjusted, to give the best fit. The rotational phase adjustment is displayed in the Ph Corrn box.
  • The functioning of all other items on the control are unchanged.
  • As a general rule, phase adjustments should not be required. See below for more information.
  • When the control is in the Phase model mode, the usual 'exit' button at the top right of the control operates to close the Phase mode, and return the control to its normal mode of functioning.

3. The fitting process

The process involves considering each shape model in turn.

For each shape model

  • two fits of the model to the observed chords are made. One fit is for the minimum size of the shape model that can reasonably fit the observed chords, while the other fit is for the largest size of the shape model that can reasonably fit the observed chords.
  • In making these fits, note that:
    • the uncertainty in the chord lengths needs to be displayed (menu item Plot options... Show error bars)
    • the location of the center of the shape model will frequently be different in some degree between the two fits. That is, do not look for minimum and maximum diameters on the basis of a fixed location of the shape model relative to the chords - treat the fitting independently.
  • Make the fit by adjusting one or other (or both) the plot scale of the shape model (using the slider on the Shape model control, or the slider on the DAMIT and ISAM shape models form) or the plot scale of the chords on the Plot form.
  • When satisfied you have identified the minimum size shape model that fits the chords, click the Set button next to the Minimum dia box. that will populate the box with the corresponding volume-equivalent diameter of the asteroid.
  • Repeat for the maximum size shape model, and click the Set button next to the Maximum dia box.
  • From the drop-down box for Fit quality, select the most appropriate description of the quality of the fit of the shape model to the observed chords. The settings are described below. If the value you want is already displayed, but the text underneath the box is different, simply click the box. (There is no need to display the list and select from it.)
  • Click the Save/update model... button to save the values on the shape model control in memory. [See near the end of this Help topic page for information about Saving/deleting the Shape model fits.]
  • Move to the next shape model, and repeat the process.
  • You can go back to models you have already processed, and adjust the values. The value of each quantity currently held in memory for the selected shape model is displayed below the various boxes.

Making the best fit

COMMENT - This section will need to be expanded with experience. Suggestions for advice to give here welcome!

When fitting shape models, it is important to have the observations uncertainty displayed - by checking the menu item on the Plot form Plot options... Show ERROR bars

The Control provide the following list of Fit Quality settings

  • Not fitted
  • Bad occultation data
  • Model wrong
  • Minimum diameter
  • Diameter but no fit
  • Poor fit
  • Good fit
  • Not constrained

Guidance for these options are:

For events where the chords are a good fit to the shape model:

  • This situation would require several chords on either side of the asteroid that are well aligned to the shape model.
  • For the minimum diameter, make a best-fit alignment of the shape model with the 'innermost' ends of the error bars; and
  • For the maximum diameter, make a best-fit alignment of the shape model with the 'outermost' ends of the error bars;

For events where the chords are a poor fit to the profile

  • This situation would require several chords on either side of the asteroid, but those chords do not align well with the shape model.
  • The first step is to ascertain where a rotational phase shift adjustment might result in a better fit (see the next section). Otherwise, the discrepancies between the occultation chords and the shape model may be a combination of 'errors' in both the occultation observations and the shape models. For these reasons it is appropriate to fit the chords to the shape model:
  • For the minimum diameter, make a best-fit alignment of the smallest shape model that fits a relatively large number of the observed chords; and
  • For the maximum diameter, make a best-fit alignment of the largest shape model that fits a relatively large number of the observed chords

Diameter but no fit

  • There may be times when the occultation chords can be fitted to a shape model, but cannot be used to draw any meaningful conclusions about the shape model. Typical situations include having two chords on the same side but reasonably well spaced. In this situation it is unlikely that any meaningful conclusions about the shape model can be drawn. However it is quite likely that the diameter of the asteroid can be determined - particularly if the two (or however many ) chords align with the shape model.
  • The situation of having two chords on opposite sides of the asteroid would normally be covered by Good or Poor fit.
  • For the minimum diameter, make a best-fit alignment of the smallest shape model that fits a relatively large number of the observed chords; and
  • For the maximum diameter, make a best-fit alignment of the largest shape model that fits a relatively large number of the observed chords

Minimum diameter

  • This most commonly arises when there is a single chord, or several closely spaced chords
  • Fit the observed chord along the longest transverse of the shape model, with the edge of the shape model aligned with a point about 2/3 of the distance from the event location, towards the 'innermost' end of the error bars.

For events where the shape model is wrong

  • normally this would only be set if there is another shape model that clearly fits the event chords. When this quality is set, the maximum and minimum diameters cannot be set.
  • This setting is not about whether one model is preferable to another. Rather it is an indication that the shape of the shape model is totally inconsistent with its actual shape
  • If all the available shape models clearly do not match the observed chords in any meaningful way, it is appropriate to select Model wrong for all shape models.

Bad occultation data

  • This is used when there are problems with the occultation data which cannot be reliably resolved (such as by excluding obviously discordant chords, or applying time shifts to correct obvious time base errors)
  • This issue is more common before ~2000, when the majority of observations were visual. Time base errors can also be associated with NTP (Network Time Protocol)
  • When it is apparent that the occultation data has unresolved problems, maximum and minimum diameters cannot be set. Essentially the occultation observation is excluded form any further analysis that might be undertaken.

Not constrained

  • this is used when ever the chords are insufficient to place any reasonable constraints on the size of the asteroid. Typical situations are:
    • a single chord which is shorter than about 60% of the asteroid's nominal diameter. However care needs to be exercised when the shape model is highly elongate, as a short chord fitted to a short cross-section may give a realistic diameter.
    • several chords located on one side of the asteroid, with it being possible to fit the chords using greatly different diameters for the asteroid. normally this would only be set if there is another shape model that clearly fits the event chords. When this quality is set, the maximum and minimum diameters cannot be set.

Not fitted

  • For events from about 2021, this setting is only present when no fit has been made to a shape model. Prior to 2021, this setting was also applied when the fit was Not constrained

Rotating the Shape model

In certain circumstances it may be appropriate to consider whether the shape model would fit better if the rotational phase was adjusted. Changing the rotational phase is only justifiable if:

  • the time between the event and the observations used to create the shape model is large - typically >10 years;
  • the light curve variations are generally small; and/or
  • the shape model is heavily dependent on 'scattered' brightness measures rather than a continuous light curve measurement. That is, observations from Lowell, USNO, Catalina or Gaia

It is generally useful to have the axis of rotation of the asteroid displayed. This can be done by checking the menu item on the DAMIT and ISAM shape models form  ...with Model image... Include rotational axis

You can assess whether it is appropriate to vary the rotational phase for any shape model in DAMIT by clicking Light curves on the Shape model control on the Plot form, or the menu items with Model image...  List DAMIT parameters and light curves, and with Model image...  Display model fit to light curves (DAMIT). These will provide details of the light curve data, and show the fit of the shape model to the observed light curve.

The display of phase-adjusted shape models must be done from the DAMIT and ISAM shape models form. Select the shape model for which you want a series of phase-adjusted shape models (either DAMIT or ISAM). Then from the menu item  with Phase models... click the relevant step intervals. This will display 10 phase models at the specified step interval.


Hint 2. You can change the zero-point of the phase-offset series using the Zero phase up-down control on the DAMIT and ISAM shape models form. This can be particularly useful if you start with a coarse set of phase models, and want to have a closer look at a series of models near a particular offset.

Where there are multiple occultation events for the asteroid, one would expect any phase adjustment to be associated with a change in the rotational period. That is, the adjustment should vary linearly with time. The Adjust rotation control on the DAMIT and ISAM shape models form allows the user to apply changes to the rotational period to assess whether there is a basis to adjust the rotational phase.

To return to the selection of shape models - on the DAMIT and ISAM shape models form, either
* click the usual Exit button at the top right of the Shape model control; or
* click the menu item with Phase models... Close phase model display control

The DAMIT Quality values

  The DAMIT system incorporates a Quality Factor QF that provides some information about the reliability of the derived shape model The value ranges  between 0 and 5. However for many of the older models a value is not yet assigned. The scale, as set out on the DAMIT site, is as follows:
5 – "ground-truth" models reconstructed from images obtained by spacecraft (usually not included in DAMIT but available through PDS);
4 – models based on a large photometric data set and disk-resolved data and/or stellar occultations, the pole ambiguity is resolved, they should well correspond to the real shape of the asteroid;
3 – reliable models based on large photometric data sets;
2 – models reconstructed from only few dense light curves combined with sparse data, the shape is not very certain;
1 – coarse shape models based solely on sparse data, large errors both in the shape and pole direction;
0 – just one possible model out of many.
n/a - no uncertainty value is available for this model.

Regard should be had to the values (when available) when trying to identify the shape model that best fits the observed chords. Note that 'sparse data' is a reference to single point photometry from several observatories made over many years, in contrast to the continuous monitoring of an asteroid for many hours to obtain a light curve on a single night - this being repeated with the asteroid being at differing locations in its orbit

Dealing with poor occultation observations

It is inevitable that from time-to-time users will have to deal with an event where the observed chords do not match any of the available shape models. Issues to be considered are:

  • The validity of the shape model(s). 
    • For a DAMIT model with a quality factor of 4 or 5, the model should be highly reliable. As a result, any mismatches between the occultation chords and the shape model is reason to have a very close look at the occultation observations - on the basis that the cause of mismatch will most likely (but not always) be with the occultation observations.
    • For DAMIT models with Quality factors of 3, any significant mismatch could be caused by either inadequacies in the shape model, or erroneous occultation data
    • For DAMIT models with a quality factor 0, 1 or 2 - the starting point is that the occultation data is likely to be more reliable than the shape model.
  • Erroneous occultation observations
    Erroneous occultation results are an infrequent but regular occurrence. Sources of error include
    •  poor time base. The most reliable time base (as at 2018) is a video-inserted GPS-sourced time stamp on each video frame. Other time sources can have a range of issues with uncertainties of differing magnitude. When the position (but not length) of an observed chord is inconsistent with other observed chords, it will usually be appropriate to make enquiries about the time base that was used.
    • visual observations. Before 2000, the majority of observations were made visually, whereas by 2010 the majority are made using video or the like. Visual observations have issues of reliability arising from observer tiredness and the need to concentrate for long periods of time, as well as the need to apply Personal Equation (PE) - with PE usually severely underestimated for the D, and somewhat underestimated for the R. Asserted PE's of 0.5 sec or less should be treated skeptically.
    • event only marginally visible. The reliability of an observation can be adversely affected if the star is barely visible, or there are significant atmospheric disturbances or cloud, or if the magnitude drop is very small.
    • Observer monitors the wrong star This is relevant for Miss chords which are inconsistent with adjacent chords that record an occultation event.

4. Saving/deleting the Shape model fits

Shape model fits are saved/retained in the following ways:

  • If you are working with an individual file for an event, by saving that event after having made the fits(s)
  • If you are working with the Historical Observations file, by updating the event in that file

You can view the details of the shape model solutions from the Observations Editor, using the menu item Shape models... List current-event shape model solutions.  If you have inadvertently created extra shape model solutions over that which is appropriate, you can delete these using the Observations editor menu item Shape models... Delete current-event shape model solution(s). Simply check the entries to be deleted, and click Delete.

Occasionally a shape model will be deleted from DAMIT or ISAM, and solutions for those models should be removed - using the delete function. The presence of such models will be indicated on Observations Editor form, where shape model fitting information is displayed.

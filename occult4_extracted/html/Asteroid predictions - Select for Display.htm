This topic is for a legacy function that will be removed.

The legacy functionality relied on the occultation elements being held on disk with a fixed-length record structure - with the data for any one event being read from disk. The fixed-length record structure prevented easy modifications to incorporate extra data elements - and as a result became a barrier to the development of improvements to Occult. The replacement approach maintains all the prediction data in memory, allowing for easier access and filtering; it uses variable length .csv files and .xml files for holding relevant data.

Select and Display Asteroid predictions

Selection criteria

The following selection criteria may be applied, by checking the relevant check boxes.
ItemComments
Visible from...Includes events if, at the specified location and time of closest approach to that location
  • For stars brighter than mag 4, the only criterion is that the asteroid must be at least 1 deg above the horizon
  • For stars fainter than mag 4 - no events if the sun's altitude is greater than -1 deg. And no events if the asteroid's altitude is less than 5 deg and the sun's altitude greater than -9 deg.

The latitude/longitude values are retained between sessions. You can use the site files to insert the coordinates of a selected site, by clicking the Select site button. This will also allow you to insert your home site location (set in the User Settings) by clicking the Home button on the form that appears to select from the site files..

The longitude and latitude values here are also used as the default site location for plotting the occultation maps.

Distance of site from path...This is intended to limit events where the observer is prepared to travel to the occultation region. Note that the distance calculation is approximate, and become unreliable when the distance is of the order of 1000km. This should not be used at the same time as the next filter.
Distance of asteroid from star....This filter focuses on the sky-separation of the star and asteroid - and for a stationary observer is more relevant in filtering events that might occur at their home location. This should not be used at the same time as the previous filter.
Magnitude drop...Limit the output to events where the magnitude drop will be at least the amount set. Visual observers should set a value of at least 0.7 mag here.
Maximum duration at least...Use this to exclude short-duration events. Visual observers should set a value of at least 1 second here.
Diameter at least...Use this to filter out small asteroids from a large file of occultation elements.
Asteroid number contains...Use this to extract predictions of a particular asteroid from a large file of occultation elements.
Asteroid name contains...Use this to extract predictions of a particular asteroid from a large file of occultation elements.
Date between...Use this to extract predictions during a particular period from a large file of occultation elements.

The end date is automatically set to be a period later than the first date, whenever the first date is changed. The difference is set in User Settings, in the box Asteroid occultations - search and display, the item Default date range....

Planet Use this to extract predictions of a particular planet and its moons from a large file of occultation elements.
Trans-Neptunian asteroidsUse this to extract predictions of a TNO's from a large file of occultation elements. The criterion applied is whether the semi-major axis of the orbit is greater than 6AU - which is a little larger than the orbit of Jupiter. As a result the filter will include non-TNO objects.
Binary asteroidsUse this to select asteroids that are binary or probably binary asteroids. [Note - this filter is based on the asteroid being marked as binary when the Occelmnt file was created.]



Plot scale for prediction plots

This is used to set the plot scale used for plotting the occultation path.

IMPORTANT. The setting here is the default setting for all the asteroid plots. In particular, if you adjust the plot scale for one event on the plot of that event, when you plot the next event that will be plotted at the scale set here. Similarly automated plot output is drawn at the scale set here.



Plot using Summary listings

When any of these buttons are clicked (except the Draw paths from the last summary) you will need to select a file of occultation elements. For this reason, when you specify the file name for occultation elements when you conduct the search, you should give careful consideration to the file name so that you can easily identify the file when you want to display events.

The Summary lists are provided in two formats.

  • The 'normal' format provides a simple list - with the occultation map being drawn automatically whenever an event line is selected
  • The second format is designed to assist the automatic generation of output files - for example, for making available on a web page. This mode is selected by checking the Enable generation of output files from the Summary form check box. When selected:
    • the summary list has check-boxes on each line,
    • the menu bar contains a drop-down box to select a relevant site file to use for predictions
    • the menu bar contains a button Auto file generation - which will create output files for each event that is checked in the summary list. The files that are generated are specified in the User Settings form.
    • Output files are only overwritten if the occultation elements have been updated. There may be a need to overwite output files if the site file being used has been updated - in which event the check box Site file has been updated should be checked to enable overwriting of the old files.
ButtonComments
Local events: SummaryProvides a text summary of events that meet the criteria - including visibility from the specified site. From here you can plot individual events, automatically generate a range of output files [details], and draw a regional map showing all the occultation paths [map details].

If output is being generated to create a feed for OccultWatcher, check the box Local events Summary - Save elements for OccultWatcher to ensure an Occelemnt file is written for each event.

Global events: SummaryProvides a text summary of events visible anywhere is the world that meet the criteria (other than the site location.) From here you can plot individual events, automatically generate a range of output files [Details], and draw a regional map showing all the occultation paths [map details].

NOTE: If this function is selected, the first two selection criteria (site location and distance from path) are not applied.

Draw paths from the last summaryThe data used to plot the paths of events shown in a Summary is held in a temporary file, and is available to plot paths even after the summary has been closed. Click here to replot the paths.
Planets: local times summaryFor occultations by the major planets, this gives a list of the D and R times for the occultation by the planet's disc as seen from the site location. Details

Observable occultations by the outer planets are not common - mainly because of the slow motion of the outer planets, and the brightness of the planet. However occultations of reasonably bright stars by Mercury and Venus are somewhat common. But the region of visibility is quite small because these planets are always close to the Sun, and as a result observable occultations are often not publicised. Users should consider conducting searches for occultations by these planets, using this option to identify those events that will be visible from their location.

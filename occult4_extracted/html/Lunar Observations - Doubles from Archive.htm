Creating Double Star reports from the Lunar Archive

The observations listed in the Lunar Archive can be used to generate double star reports for use with the Double Star analyser. To achieve this, the list of observations must be reduced to pairs of observations of the two components, with the brighter component being listed first (and with all other observations removed from the list). Once this is achieved, the menu item with List... Create double star files can generate a double star observation file for each pair of observations - and those files can be read into the double star analyser for analysis. The menu option with List... Validate data will check to confirm that the list of observations is ready to create the double star files.

The process involves a manual selection and editing of observations. Intermediate lists of observations can be saved for later re-loading and continuing, using the File... Save menu option, and the File... Open previously saved list menu option. You can also break the task up into multiple segments for easier editing of smaller portions of the file, followed by a merge using the menu option File... Merge previously saved list (which merges a previously saved list with the currently displayed list.)

NOTE: The functionality discussed in this topic does not, and cannot, result in any changes to the data held in the Lunar Archive. That is, you do not need to be concerned about inadvertantly making changes to the Lunar Archive in this process.

The process is as follows:

Step 1 - create a list of events

  • Open the form to View/analyse historcal occultation observations.
  • Check the option button search for star
  • In the line of check boxes after Double stars:
    • Check the check box Select for Analysis - which is the 4th check box . This will display a box containing edit controls specific to the process. [It also enables Multi-selection of displayed events.]
    • If you are confident that in all instances where an observer has reported times for both components, the archive has both components flagged as a double star, check the box only list doubles. [This is unlikely to be true, especially for older observations.] Otherwise, leave the checkboxes for Highlight, only list doubles, and list ALL unidentified doubles unchecked.
  • Specify the star of interest - either by name or star number.
  • Now click Read, to list the observations of the star.

Step 2 - course filtering

Having displayed a list of stars, the next step is to edit it to create pairs of observations involving the two components. The Edit controls - double stars tools are intended to facilitate thios process. The basic process is to highlight and delete non-double-star events, and re-order double-star pairings so that the brighter (or A component) occurs first. The tools provided have the following functionality:

  • Move highlighted UP Moves an individual highlighted line up one line. Used to place events in the correct order.
  • Move highlighted DOWN Moves an individual highlighted line down one line. Used to place events in the correct order.
  • Select grazes Double star observations made during grazes are not well suited for analysis in the Double Star analyser. Rather they should be analysed in the functionality provided when the graze results are plotted against the lunar limb. This button highights all observations marked as being made during a graze - with the intention of the user deleting all such observations from the list.
  • Select D events Highlights all Disappearance events. For some stars the companion is only seen at either the Disappearance, or the Reappearance phase. This button (together with the Reverse HIGHLIGHTED button, allows all events where the companion cannot be seen to be highlighted and deleted.
  • Bright Limb events Highlights all events that are against the Bright limb. For the majority of double stars the companion can only seen at a Dark limb event. This button all bright limb events to be highlighted and deleted.
  • dT > 15secs Highlights all events where the time difference from both the previous and following events is greater than 15 seconds. When clicked, the algorithm will initially highlight all observations that are within 10 secs of each other; when this is complete, the highlighting is reversed.]
  • Reverse HIGHLIGHTED This swaps the highlight. That is, unhighlighted events become highlighted, and highlighted events become unhighlighted. Particularly relevant for the Select D events button.
  • Delete HIGHLIGHTED This deletes all highlighted events from the displayed list.

Example

Antares provides a useful example. The companion of Antares is only visible at Reappearance events. Furthermore, at mag 5.6 it is not visible against the bright limb. So the initial processing steps would be:
  • Set the form to Analyse, and list all events involving Antares (ZC2366)
  • Click Select grazes. All graze events will be highlighted (it will take a few seconds...). Click Delete HIGHLIGHTED to remove them from the list.
  • Click Bright Limb events All events that were against the Bright limb will be highlighted (it will take a few seconds...). Click Delete HIGHLIGHTED to remove them from the list.
  • Click Select D events All events that were Disappearances be highlighted (it will take a few seconds...). For Antares, the companion can only be seen at Reappearance events. Click Delete HIGHLIGHTED to remove all the Disapperance events from the list. [For stars where the companion can only be seen at the Disappearance, after clicking Select D events , click Reverse HIGHLIGHTED ; this will result in all the Reapperance events being highlighted - which can then be deleted.]

At this point you now have a much smaller list of events to deal with. It may be appropriate to save the list at this time, so that it can be retrieved if you muck up the subsequent editing.

Step 3: manual editing

You need to remove all events where the observer has not reported times for both components. Note that while the usual situation for a single event is for it to be of the primary star, the data for Antares shows instances of old observations where the only reported time is for the companion (eg 1897 Jan 28).
To do this, visually scan through the list to identify and highlight observations for which there are no observations of the other component. As MultiSelect is turned on, you can highlight multiple lines. However be alert to the presence of two observations by the same observer but with slightly different times - as it is quite possible that the double star flags have not been set in the Archive. [That is, do not be too reliant on the 'A' or 'B' WDS identifier in this process.]

During this process it is likely that you will be looking at observations where both components have been reported. It can be convenient to place such observations in the correct order - brighter star (or A component) before the fainter (B) component. [If this is done incorrectly, it can be corrected in the Double Star analyser by clicking the button to swap the T1 and T2 times.] The ordering is adjusted by highlighting a relevant observation and clicking either the >Move highlighted UP or >Move highlighted DOWN button. When doing this, make sure you match both the date, site coordinates, and observer.

If the list of observations for manual editing is long, it can help to break the list up into smaller sublists. To do this:

  • Save the full list of course-filtered observations.
  • Highlight a convenient span of years; click Reverse Highlighted to change the highlighting to be for all the other events.
  • Click Delete HIGHLIGHTED. Then Save (with a file name appropriate to the date selection, for subnsequent identification).
  • Continue editing the displayed data; OR
  • reload the saved full list, and select a different span of years.

Step 4: confirm that the data is ready to generate the double star report files

To generate the final set of double star report files, you must have removed all extraneous observations, and have all other observations ordered as pairs (with the bright star occurring first).

When you think you have achieved this, select the menu item with List... validate data. This will check whether the requirements for generating the double star report files have been met. A message box will either advise that the data is ready to genertate the files. Or it will advise of the number of 'errors' in the list. The errors can be found in the file:
Occult4\Observations\Doubles\{starNumber}\Test.txt
This file lists pairs of lines which are 'incompatible' with the requirement of pairs of observations relating to the two components of the star observed by the same observerconfirm that . In this regard, a single extraneous line can result in all subsequent pairings appearing to be in error.

Step 5: generate the double star report files

Select the menu item with List... Create double star files. This will create a set of .txt files in the directory
Occult4\Observations\Doubles\{starNumber}\ .
The file name is structured as: YYYYMMDD_{starID}_{Name}.txt
where 'name' is either the observer's name, or the site location.

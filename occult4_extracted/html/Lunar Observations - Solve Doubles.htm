Determine Separation and PA of doubles from Lunar Occultations

This form provides for a least-squares solution to determine the position angle and separation of the components of a double star, using two or more observations.

While it would be desirable to use observations made on the same night, you can use observations from different lunations, or even differnt years - provided the double star is not changing rapidly .

The Event box at the top of the form is for the star identification, and date of observation. This is used to form the file name, and to identify the event. This data is only sued to identify the file; it is not used in the analysis in any way.

  • Year- a year - used for file name.
  • Mth - a month.....
  • Day- a day....
  • Star ID- the star identifier.

    This box also provides links to the WDS catalogue, and the 4th Interferometric Catalogue. Additionally, to assist with preparing papers for publication, the List Equivalents button is provided - to list the star identification in a range of other catalogues.

    The Details of an observation box is an area to enter or edit the details for an observation The data falls into 5 groups, as follows:

      Data relating to the double star observation
    • T1 - the seconds of the event involving the brighter star. Avoid having a negative value;
    • T2 - the seconds of the event involving the fainter star. Make sure the seconds are referenced to the same minute as used for T1. Negative values are allowed, but the reference minut must be the same as for the primary star;
    • the button Swap T1, T2 allows the user to swap the T1 & T2 times. The need for this might arise from incorrect data entry. However when the stars involved are of similar brightness it may be necessary to swap the times for some observers in order to have consistence in the star which is being treated as the primary star.
    • PA - the observed PA of the event (in degrees). Preferably obtained from a double star report. However it can be taken from the reduction of an observation or the lunar archive.
    • rv - the radial motion of the moon (arcsecs/second). Preferably obtained from a double star report. However it can be taken from the predictions.
    • cct - the angle between the normal to the lunar surface, and the moon's motion (in degrees). Preferably obtained from a double star report. However it can be extract from the predictions.
    • Scale - the ratio of the apparent lunar diameter to the mean lunar diameter. Preferably obtained from a double star report. However it can be extracted from reductions, or the lunar archive.
    • limb slope - the slope of the lunar limb relative to the horizontal. If the user has the LOLA lunar limb installed, this field will be automatically completed during the solution. Otherwise the user will need to manually fill this field - as detailed below.

      Date of observation, and Observer
      This data is used to identify the specific observation, but is otherwsie not used in the solution.
    • Year- the year of the particular observation.
    • Mth - the month of the particular observation.
    • Day- the day of the particular observation.
    • Observer - the observer's name.

      Librations, to determine the limb slope
    • l - Longitude libration
    • b - Latitude libration
    • AA - Axis angle at the event.

      Circumstances relevant to assessing reliability
    • CA - Cusp Angle
    • %Illum - % illumination
    • Alt - Altitude of moon

    Entering observations

    There are two preferred modes of entering observations. The first is to read one or more double star report files generated by Occult. The 2nd is to paste the data from a report generated by Occult. Alternatively the data can be manually added.

    Reading double star report files

    Select this option (either from the menu under Files, or using the button Read observation file(s), and navigate to the folder where one or more double star report files are held. Select all the files to read. The files will all be read, with the data being placed in the Observatiuons box at the bottom of the form. The first event is used to populate the star ID and date the Event box. At the completion, it may benecessary to add the local limb slope for each event if the LOLA lunar limb has not been installed - see below.

    Pasting the data

    Merely copy (CTRL-C) the entire content of the email report, then click Paste an observation. If it is the first event, the star ID and date will be populated in the Event box. If it is a subsequent event, an alert will be raised if the star number does not correspond with the star number displayed on the form. Additionally, the lunar limb for the location of the event will be displayed - see below.

    Manual entry

    If needed, the Details of an observation can be entered manually. The limb slope is added - see immediately following.

    Setting the local limb slope

    If the LOLA lunar limb has been downloaded, this field will be automatically set duriung the solution process - and re-evaluated whenever the double star solution changes.
    If a double star report does not include any of the librations l, b, AA, or the scale, the program will attempt to retrieve these values for the event from the Lunar Archive. If the event cannot be found in eth Lunar Archive, the event will be unchecked to exlude it from the analysis. In this siutation it is appropriate to confirm that ou have correctly specified the date, star number, and the observer's family name. The event can be included by checking the box; however the limb slope will likley be incorrect - resulting in a poor solution.

    Otherwise the slope of the lunar limb needs to be manually included to allow a proper analysis of the observations. This is done by plotting the event location against the lunar limb, and drawing a line that is tangential to the limb. If the event is pasted, the limb is automatically plotted. Otherwise the limb for the currently displayed event is generated by clicking Review limb slope.

    The lunar limb plot will include two vertical lines near the center. The lines are located approximately 2" of arc on either side of the event [The 2" displayed is based on the mean radius of the moon, not the actual radius at the time of the event.]

    To obtain the limb slop from the limb plot, place the mouse on the profile at a point of interest, holding down the left mouse button, and drawing a line parallel to the limb. Next to the cursor the slope of the limb will be displayed. When the left-button is released, the last-measured slope is written just above the plot area, to the right. Enter the displayed value (including the sign) into the 'limb slope' box. Then click either Add or Replace, depending on whether the limb slope is being incorporated into a new observation entry, or is being added to an existing observation entry.

    Editing observations

    To edit an observation - select the observations from the list of observations using the up/down cursor keys. Make the required changes in the Details of an observation area. Then click Replace.

    To Delete an observation - select the observations from the list of observations using the up/down cursor keys, so that it is listed in the Details of an observation area. Then click Delete. You will be asked to confirm.

    Plot and Solution

    Generally the observations are plotted, and a solution is generated, whenever data in the list of observations is changed. A solution can be 'forced' by clicking the Plot button.

    Providing the Lola lunar limb has been downloaded, the solution is an iterative process. The first iteration simply assumes a smooth lunar limb to generate an initial solution for the separation and PA. Thereafter the speration and PA are used to determine the limb height at the location of the secondary star (with allowance for the curvature of the limb) to generate a new solution. This last step is repeated a 2nd time to generate the final solution.

    The Limb Plot

    All checked observations are plotted in the Limb Plot. The limbs are plotted relative to the central 'bright' star, at a location corresponding to when the limb occulted the secondary star. The limbs are drawn as straight lines, and do not show the lunar topography. However provided the Lola lunar limb has been downloaded, their location closest to the secondary star as shown in the plot is accurately positioned having regard to the lunar profile. This results in the secondary star being located at the point of intersection of the limb lines. [This is on the assumption that that the brighter and fainter star have been correctly identified in the report. Also, if the observations are on different dates, the possibility of motion of the companion must be considered.]

    The Limbs are colour coded, and have a number plotted at the location closest to the central star. A legend is provided which gives the colour coding, the reference number, the T1 value, and the observer's name. The legend includes all events - not just those plotted.

    The plot can include an indication of the uncertainty associated with the positions of the limb lines - by checking the checkbox Plot nominal uncertainties. The uncertainty region is indicated on the basis of an uncertainty of ± half a video frame for both the primary and secondary star (a combined total of 0.7 frames), and a frame rate intermediate between PAL and NTSC. That is the uncertainty is indicated as the displacement of ± 0.025 secs in the location of each limb.     The plot can be drawn either in color or B&W with the menu item with Observations... Plot in B&W

    The solution

    The solution is based on those observations that are checked in the list. At least two observations must be checked to generate a solution. Three or more observations are required for a least-squares best fit. The best solution will come from events that have a significant difference in PA of the event. Occultations results made on different dates can be used on an assumption that there has been little change in the separation and PA over the period. However in these circumstances the user MUST be careful to verify the validity of this assumption, and/or qualify the derived PA and separation as being subject to this assumption.

    The ± values are based on the consistency of the fit. The degree of convergance of the limbs in the plot is a good indicator of consistency. It will also identify anomalous observations. To ascertain the effect of uncertainties in the observed time difference (which is required if there are only a small number of observations), change the times by the estimated uncertainty, and note the change in the solution values.

    Under some circumstances the solution might not converge. The code allows for 100 iterations. If the solution has not converged, the RMS value will be followed by the words ' Invalid solution'.

    Other options

    Viewing the WDS and Interferometric catalogues

    If a valid star identifier has been specified, click 'Show WDS/IF' to view the entries in these catalogues.


    New/Open/Copy/Print/Save

    The data can be saved using the File... Save solution (text)menu item. The default directory is the Occult 4\Observations\Doubles directory. The saved file includes the identification, and the displayed solution, and has the extention .sln. Details of the file format are provided here.

    A saved file can be loaded with the File... Open solutionmenu item, or by clicking the Read a solution file button..

    The displayed data and solution can be printed or copied to the clipboard with Observations... Copy, and Print menu items

    The form can be initialised with File... new.

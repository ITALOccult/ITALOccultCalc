Overview of the prediction process

An asteroidal occultation consists of a path across the Earth having a width equal to the diameter of the asteroid. That is, the width of the path is only a fraction of the Earth's diameter. Furthermore, some of the path may be in locations which are sunlit at the time of the occultation, and/or may be in locations which are relatively inaccessible.

 Occult generates predictions in a two-stage process.

Stage 1 is

  • specify the asteroid or asteroids to be considered in the search;
  • select the star catalogue to be used in the search;
  • set the start and end of the period to be covered by the search; and
  • generate a set of Besselian elements for all events where an edge of the occultation path comes within a specified distance of the Earth's center (default distance is 1.01 Earth radius - with allowance for the effects of stellar diameter being incorporated where appropriate).

The Help topic for Stage 1 is here

Stage 2 is

  • filter the list of events to those that are of potential interest. Those filters include parameters like solar elongation, local altitude, minimum duration, minimum magnitude drop, faintest star magnitude, and distance relevant to the observer's location.
  • save the list of events for future reference
  • create a 'personal feed' for Occult Watcher
  • sort the events according to a range of parameters

At any stage, selected events can be displayed, with a wide range of related information being accessible.

The Help topic for Stage 2 is here

Demonstration of Asteroid predictions

Generating predictions of asteroid or planet occultations is a three-step process

  • ensure you have a current set of orbital elements for asteroids
  • compare the position of the asteroid or planet with a star catalogue to generate a list of global;
  • filter the events using a range of selection criteria to create a list of predictions of interest to you.

NOTE: The initial installation of Occult does not include any catalogues suitable for Asteroid predictions. Three relevant star catalogues can be downloaded from the Downloads form of Occult:

  • Tycho2 (pre Gaia) (#32) [ a 64MB download],
  • Gaia11_DR2 (#41) [ a 70MB download] and
  • Gaia14_DR2 (#40) [ a 847MB download].

Occult can also use UCAC4, UCAC5 & PPMXL - which need to be sourced elsewhere. This example assumes you have downloaded the Tycho2 (preGaia) catalogue (download option #32).

Step 1 - Create a current set of orbital elements

  • Select the Maintenance tab, and click the General downloads button to open the Downloads form.
  • Make sure you are internet connected.
  • The preferred source of elements is MPCORB. However we want to have Astorb available as it contains error estimates that are not present in MPCORB or AstDyS-2.
  • The conversion process also makes use of information contained in the files relating to Binary Asteroids (download #6), Asteroid Rings (Download #7), Asteroid Diameters (download #8) and Asteroid Classes (download #9). You should ensure these files are up-to-date
  • Under Files for Asteroid Predictions, click the Download button next to Astorb (option #24).
  • After a few seconds, a progress bar will appear showing download status. If you cannot connect to Astorb, try clicking the adjacent button for the VizieR Mirror (option 24a).
  • When the download completes, click OK in the confirmation box. The file will be unzipped, and then a conversion form containing three boxes will automatically open. We do not want to convert Astorb at this time - so simply close this form.
  • The downloads form should still be open. Click the Download button next to MPCORB (option #22) - which is the file we want to use
  • When the download completes, the conversion form containing three boxes will automatically open.
  • In box 1 Select File, check MPCORB. It should be pre-checked)
  • In box 2 Set selection criteria, set the diameter to 15 km, and check the Numbered minor planets only check box.
  • Box 3 Specify 'mandatory' minor planets has several asteroids listed. Do not make any changes here at this time. But if later you want to generate predictions of a particular small asteroid, you can enter the number here to make sure the asteroid is included in the file.
  • Click Convert. A file dialogue box will appear. It should show the file MPCORB.dat in the directory Downloaded Files. In the Name section of the dialog, double-click MPCORB.dat. The conversion will now take place.
  • Click OK in the confirmation box.
  • Close the Downloads form.

You now have a file of asteroid elements that can be used to search for occultation events. That file is located in the Occult 4\Resource Files\ directory with the name AsteroidElements.csv. The format is described here.

Step 2 - Search for occultation events

  • On the Occult Main Menu, select the Asteroid predictions tab, and click the Search for, & list, occultations button.
  • In box 1 set the date range for the Search, click the button for 2018. [If you are doing this after 2018, set the date range to cover all of 2018].
  • In box 2, set the search for Minor Planets.
  • In box 3 Select star catalogue for search,
    * select the Tycho2 (pre Gaia) catalogue:
    * Leave the three check boxes at the bottom of box 3 unchecked.
  • In box 4 Select the minor planets for the search, click the Set range of minor planets button. In the form that appears, read the text. But click OK without entering anything into the text field.
  • In the list of asteroids that now appears, highlight asteroids 1 through to 30. Then click the OK button at the bottom of the screen. Note how the range of selected asteroids is now indicated on the search form next to the Set range of minor planets button.
  • Repeat step 5, but this time enter -40 into the text field in the form that appears. Note how the range is now set to cover the asteroids 1 to 40.
  • Ensure the diameter in the line for Exclude minor planets smaller than is set to 15 km. The check box for User file of elements should be clear. The drop-down box labeled 'Limit search to' should be set to All (this box limits a search based on taxonomy). The check box to Exclude if duration less than should be checked, with a value of 2 secs being a reasonable value.
  • Box 5 is only enabled when the asteroid is known to have one or more moons with known orbits. If this box is enabled, read the Help topic for the Search form to learn how to use this box.
  • In the group of boxes at the bottom of the form, the button for 1995 Oct 10 == 2023 Feb 23 should be greyed out (disabled). If not, click it. If you are going to generate a prediction for a date outside this range, you will need to click other buttons so that the entire date range for the prediction is covered.
  • In box 6, Set the file to save the occultation elements from the search,
    * the check box to Automatically save the search results should be left unchecked.
    * leave the filename as generated.
  • In box 7 Do the search, set the Increase miss distance to zero.
    * Note at the bottom-left of this box the source of asteroid elements being used, and their date;
    * At the bottom-right of this box, ensure the check box Clear the previous results from Memory  is checked.
    *     click Search.
  • You will now see events listed as the search progresses. Progress can be monitored by the progress bar; the name of the asteroid currently being considered is listed at the top of the display box.
  • When the search is completed, you will be taken to the form List & Display, where the events are listed in chronological order.

Step 3 - List & Display predictions

  • The number of events listed is shown at the top left of the form, under the button List Events. In the test run conducted for this topic, 535 events were found (but this may change as a result of different orbital elements being used).
  • Generally, you will not be trying to observe events when the solar elongation is small. In the orange filter box, check Solar elongation >, and set a value of 30. Click List Events, and note how the number of events is reduced to 426.
  • Events with a small magnitude drop can be hard to detect. In the orange box, check the Magnitude drop >, and set the value to 0.2. Click List Events, and notice how the number of events is reduced to 337.
  • Click the menu item Sort events... by star magnitude. The list will now be sorted with the brightest stars at the top of the list. Using Sort events, sort the list by date.
  • Now let us identify events that might be observable in either North America or Europe. In the purple filter box check the Path crosses check box. Then enter the following longitude/latitude values :  (-100, 50), (30, 50), (-100, 30), (30, 50). (You can pre-define regions using File... Edit list of regions, and select predefined regions using File...  select region...) This will select all events where the occultation path crosses some part of the quadrilateral set by those coordinates. Click List events. 33 events are listed.
  • Click the menu item Sort events... by star magnitude. The first entry is an occultation by 25 Phocaea on 2018 Jan 23. Place the cursor on the line, and click to select it. This will bring up a map showing the occultation path, with details given in the header area.
  • On the map, place the cursor somewhere on the path on one side of the Earth globe and double-click. Note how the coordinates of that location appear at the top of the form, just above the plot. Do the same on the other side of the Earth globe, and note how the Earth appears to rotate a small amount. This rotation represents the orientation of the Earth at the selected location at the time of the occultation at that location.
  • Click the button at top left with '4' on it. Note how the map is drawn at a larger scale. Try the other numbered buttons to see the effect.
  • Click the button '16'. Select the menu item Options... Export buttons - enabled, At the top right of the form there will be two drop-down boxes for Sites and Names. Select the USA Short.site from the left drop-down box. You may notice some small circles drawn on the map. On the right-hand drop-down box, select all. The names of towns will appear on the map. [To control how sites are drawn, edit the Site file.]

The centre line of the occultation path is green; the path limits are blue, and the 1-sigma variation of the path limits are shown in red. While in GoogleEarth, under the View menu item, click Switch to Google sky. Now place the cursor back on the Occult map of the occultation path, right-click, and select View star in GoogleSky a second time. GoogleSky will now zoom in on the star. [Close GoogleEarth/Sky]
  • On the Occult map, right-click, and select Path coordinates. Note the effect of checking the Include path where sun is up. Close the Path coordinates box.
  • On the Occult map, right-click, and select Path distances for multiple locations. In the drop-down box, select North America.site file. Change the site file to World Main.site, and note a different set of sites. Close the Asteroid - Multi-location predictions form.
  • On the Occult map, right-click, and select Prepoint stars. If you have downloaded the SAO catalogue from the Downloads form under option #42, the SAO number will be given for all stars (if one exists). This can assist setting GoTo type telescopes, which typically have the SAO star number available as a star identifier. [Note - you can retrieve SAO numbers using VizieR by checking the box at the top-right of the form - however this is a very slow process.]
  • Increase the Lead-time in the box by an hour. Note how additional pre-point stars are now displayed.
  • Assume the occultation at 'your' location will be at 23h 3m 30s. Enter that time in the field above the list, and check the Event Time box. Note how the list changes to give the actual time the telescope needs to be pointing at the pre-point star.
  • Adjust the fields for Magnitude and separation, and see how the list changes when you click List stars. Note that the value in the fields for Magnitude and Separation are retained between sessions.

  • On the Occult map, right-click, and select Show large star chart. Adjust the map settings at the top of the form to see how the map changes.

  • Uncheck the Visible from box, and click File... plot paths on a map. Note a map showing paths centered on the location set in the yellow box. Change the map limits and click Redraw, and note the change in the number of paths shown.
  • Note the menu item File... Saved listed events. This enables you to save the occultation data for each of the events listed.

  • It is possible that you will have many forms open at this stage. Locate the main Occult form (using the task bar at the bottom of the screen, if needed). In the menu of the main form click Cascade forms, and note how the forms are cascaded.

To generate a 'personal' OccultWatcher feed.

Individuals can generate their own 'personal' feed for OccultWatcher, allowing  them to use the OccultWatcher functionality to manage events of interest to them. Importantly, if the user indicates in OccultWatcher they are planning on observing a 'personal' event, that event will become visible to other OccultWatcher users who have the Personal Predictions feed selected. That means that others may join you in observing events in your 'personal' events.

To create a personal OccultWatcher feed, simply click Save all listed events as a 'Personal' OW feed This saves the occultation elements of the events that are displayed in a file called OccelmntSubsetForOW.dat, in the Occult subdirectory \Generated Files\. If you are running OccultWatcher 'OW' on your computer, simply copy this file to the OW subdirectory Personal (making no change to the file name), and the events will be included in OccultWatcher (after OccultWatcher has done an feed update). This way, you can generate your own list of events specific to your location, and have them appear in OccultWatcher. You can learn more about this option in OccultWatcher from the OccultWatcher Help system, under Configuration (towards the end).

Advanced users - generating files for a web site

Occult supports the generation of files to mount on a web site such as here, as well as files needed to support a 'feed' in OccultWatcher. This example assumes you have a list of events displayed in the List & Display form

To generate a feed for OccultWatcher.

Under Generate predictions for all listed event
  • set the scale of the map that will be generated to 4
  • check the item Plot the maps Path/Site centered [  ] (the current setting will be displayed in the brackets)
  • click Create files for OccultWatcher feed
  • In the dialogue box that appears, specify the name of the feed. That is used to create a directory under the Occult directory Occult 4\OWFeeds\ that will contain all the generated files
  • Next you will have a confirmation dialogue about the map scale. Click Yes
  • The files will now be generated, with a progress bar appearing at the top of the List & Display form

These files can then be uploaded to the OW site (assuming your feed has been approved.) The files that are generated for each event are

  • A world plot of the event (and for this you need to ensure you have selected an appropriate scale for the maps)
  • a file containing the path coordinates
  • an html 'summary' file that contains a world map, and a link to the path coordinates file
  • a file containing the occultation elements, used by OccultWatcher for local event times

In addition there is an html file containing a summary of all the events.

To generate files for a web page.

Go to the Occult Main form, Maintenance tab, User settings, bottom section - and check the files you want to have generated. A typical set of files would be World map, path coordinates, station sort, pre-point stars, star chart, GoogleMaps.HTM and GoogleEarth.KMZ.Then return to the List & Display form.

Under Generate predictions for all listed event

  • set the scale of the map that will be generated to 4
  • check the item Plot the maps Path/Site centered [  ] (the current setting will be displayed in the brackets)
  • click Create files [using station sort... A drop-down list appears of all site files available. The site file is used to generate local predictions for all sites in that file, sorted across the path. Select a site file - or select the first option 'none'
  • Next you will have a confirmation dialogue about the map scale. Click Yes to proceed.
  • Next you will have a confirmation dialogue about the site file, which will also advise of the number of files that will be generated. Click Yes to proceed.
  • The files will now be generated, with a progress bar appearing at the top of the List & Display form.
  • The generated files are held in the directory Occult4/AutoGenerated Asteroids. Included in the directory is an index file [FileIndex.txt]. A subdirectory MJD contains a file for each event, containing the Modified Julian Date of the date of generation of the Occelmnt file. These files are used to ensure output is only regenerated when a more-recent prediction is being used.

Event times from altitudes

For observations in the 1600's and early 1700's, an accurate time source was not readily available. In the early-to-mid 1600's occultation observations were timed by measuring the altitude of one or more stars. Later clocks were 'calibrated' by measuring the altitude of several stars. This form is designed to derive the UT of an event from the recorded measurments of the altitudes of stars, or for the times of transit of stars or the Sun.

The form looks like:

Menu item - Reset Date & Location

This clears the form of any currently entered data.

Observer & location

In this box, select the observer and site location. The site location is displayed. The drop-down list includes and option 'Enter coord manually', which allows the site coordinates to be manually set.

Altitude measurement

In this box, you first select the star that has been used in the altitude measuremnt. The coordinates of that star are displayed for both the J2000 equinox, and the apparent position on the relevant date.

Secondly you specify the UT date of the Greenwich transit of that star, and the measured altitude of that star. The 3rd box displays the UT, Mean local time, and Apparent Local time, for each of the times of local transit, local rise, and local set (at the specified altitude).
IMPORTANTThe UT date of the Greenwhich transit may differ from the date of the event. You must select a date that matches the date and time of the event as reported.
IMPORTANT For historical obsrvations, the day ran from midday to midday, not midnight to midnight. The form incorporates the required corrections for this change in definition of the day. Issues associated with the Gregorian calendar are also included.

Third box

This box displays the computed dates and times for transit, rise and set. Adjacent each of the Mean Time and Apparent time displays ius a small button. Click that button to transfer the time to the Reported local event time box.

Top right box

This is usewd to specify the reported event time.
  • When there is no clock to 'correct', the suer should simply transfer the relevant time from the 3rd box (as described above) to this field, as the time of the occultation event
  • When the altitudes are being used to determine the clock correction, enter the clock time in this field. Do this before adding the event to the Bottom box using the relevant button to specify the event is a Transit, Rise or Set event. This will result in the added line including the clock correction specific for that altitude measurement.

    Middle-right box

    This box is used to determnine the final event time, and compute the residual for the occulted star.

    You need to specify wherther the reported altitudes need to have a correction for refraction applied. This should be set to Yes unless there is a clear reason to believe the reported altitudes have already been corrected for refraction (which is most unlikely).

    When the observer is using a clock, and the observations are eing used to derive clock corrections, the form allowws the clock correction to be derived as an overall Mean, or a linear best fit. The values of the fits are displayed in text at the bottom of the form. In cases where there is an observation that seems quite inconsistent with the others, clear the check box from the Bottom box to rwemove it from the clock correction solution.

    To derive the final event time, and determine thr esiduals - enter the event time (either from the relevant altitude when no clock is used, or the reported clock time) in the top right box. Specify the star identifier using the usyal notation of Raaaa, Saaaaaa, Xaaaaaa, or (for planets) Pa. The residual is displayed below.

    when satisfied with the 'solution' for the time and star, click Add event to add the details to the bottom box.

    Bottom box

    This is used to gather all the results. In particular, when the observer has a clock, and the altitude measurements are being used to derive a clock correction, that analysis is performed on the data as lsited here. The check boxes are used to determine whether or not a particular observation is included in the derivation of the clock corrections.

    Note that when an event is added, two lines are in fact added. The first line is evaluated on the basis that the reported altitude needs to be corrected to allow for refraction. The 2nd added line is on the basis that the reported altitude was corrected for refraction (which is almost certainly never the case).

    The following buttons are provided for use with this box:

    • add transit Used to add details on the basis the altitude measurement was a time of transit.
    • add Rise Used to add details on the basis the altitude measurement was a time of Rise through the specified altitude.
    • add Set Used to add details on the basis the altitude measurement was a time of Set through the specified altitude.
    • add Clock Correction Used to add the solutions for the derived clock correcton.
    • add Comment Used to add text comments about the event. The firtst character of a comment line is a "'". If such a line is highlighted, selecting add Comment allows that line to be edited. Otherwise a new comment line is added. Added comment lines are sorted on an alpha-numeric basis (as is the whole list box).
    • Duplicate Adds an entry identical to the highlighted entry. This allows the user to weight particular observations in the Clock Correction solution, by duplicating them
    • Delete Deletes the highlighted entry
    • Paste observations Allows the user to insert events that have been previously saved - such that they can be included in the analysis (or reanalysed)
    • Copy output list Copies the contents of the lsit box to the clipboard as text.
    • Append list to Log file This is the primary option for retaining the analysis results. The log file is contained in the Occult4/Observations/ directory, with a file name made up as HistLog_ followed by the observer and location - with an extension of .txt. For example, as HistLog_Bullialdus at Loudon.txt. Clicking the Append... button adds the current data to the end of the relevant file.

    Other issues

    The data for Observer names and locations is held in the Resource Files subdirectory in a file called OldObservers.txt; that for the stars used in the altitude measurments is in a file called OldStars.txt. When the form is opened for the first time, these files are downloaded. Otherwise they can be updated to the latest version using the menu item Download Updates {this is a rare occurrence].

Observations.dat
.Obs files

The Observations.dat file contains all the observations of asteroidal occultations. Files for individual events have the extension .OBS, and have the same file structure. The file has a fixed record-length file (151 bytes) with the following data structure:

Header - Line 1

Columns Format Definition
0 A1 Header flag [= '*']
1 -20 A20 Asteroid number and name
21-29 yyyymmdd_ Year/month/day of the event. If event extends past 24hrs, specify the earlier date, and specify hours greater than 24h, as needed.
30-42 A13 Star number:
HIP xxxxx Hipparcos number
4UCzzz-nnnnn  UCAC4 number
pppp-nnnnn-xu Tycho2 number. The 'u' is a flag for the source of the coordinates. Blank = Tycho2; “u” = UCAC2
Bzzzz-nnnnnnn USNO-B1.0 number

While the following catalogue identifiers are catered for, only one entry remains using one of these identifiers (as at 1 June 2018).
Nzzzz-nnnnnnn NOMAD catalogue number [one entry remains - 2008 Aug 24]
2Unnnnnnnn UCAC2 number
3UCzzz-nnnnn  UCAC3 number
1UTzzz-nnnnn  URAT1 number
43-53 hhmmss.ssss RA of star J2000, at epoch
54-64 -ddmmss.sss Dec of star J2000, at epoch
65-70 _hh.h_ Zero time for motions
71-78 F8 RA motion of asteroid (sec/hr)
79-86 F8 2nd order RA motion (sec/hr/hr)
87-93 F7 Dec motion of asteroid ("/hr)
94-100 F7 2nd order Dec motion ("/hr/hr)
101-107 F7.4 Parallax of asteroid (")
108-113 F6.1 Major axis of satellite (km)
114-119 F6.1 Minor axis of satellite (km)
120-125 F6.1 PA of axis of satellite
126 A1 Gaia release version for star positions.
" " = Not  known
"0" = Hipparcos2
"1" = DR1 
"2" = DR2
127-136 I4, I3, I3 Year, month and day of last update to any data item in the event record.
137I1 The 5th decimal character in secs of RA. Divide by 100000 and add to the secs of RA
138I1The 4th decimal character in secs of declination. Divide by 10000 and add to the secs of Dec - before applying the sign of declination
139-148 I4, I3, I3 Year, month and day of last edit of any observation record in the event record
149-150 A2 CR/LF sequence

Observation lines

An observation line is provided for each D/R observation for each observation.
Where an observer reports events from a double star, a separate line is required for each star component.
Columns Format Definition
0-2 A3 Sequence number for observer
3 -39 A37 Observer identifier
40 A1 blank
41-51 -DDDMMSS.SS Site longitude
52-60 -DDMMSS.S Site latitude
61-65 A5 Site altitude (m)
66 A1 Code for Datum:
    < >= WGS84
    N = NAD1927
    E = EP1950
    T = Tokyo
    G = GSBN80
    * = unspecified, or other.
67-69 I3 Telescope Aperture (cm)
70 A1 Telescope type
    < >= unstated
    1 = Refractor
    2 = Newtonian
    3 = Schmidt-Cassegrain
    4 = Dobsonian
    5 = Binoculars
    6 = Other
    7 = None
71 A1 Stability
    < >= unstated
    1 = Steady
    2 = Slight flickering
    3 = Strong flickering
72 A1 Transparency
    < >= unstated
    1 = Clear
    2 = Fog
    3 = Thin cloud <2 [mag loss <2 mag.]
    4 = Thick cloud >2 [mag loss >2 mag.
    5 = Broken opaque cloud [that is, observed thru gaps in the cloud]
    6 = Star faint
    7 = By averted vision
73 A1 Time Source
    < >= unstated
    1 = GPS - video inserted
    2 = GPS - other linking
    3 = Video + audio time signal
    4 = Tape Recorder + time signal
    5 = Eye-ear + time signal
    6 = Stopwatch
    7 = Radio broadcast - calibrated
    8 = Other
74-82 HHMMSS.SS UT of disappearance
83 A1 Event code.
    D for main star
    d for 2nd star
    G for moon with main star
    g for moon with 2nd star
    M for miss
    C for Not Seen - e.g. because of cloud
    P for data point based on predictions
84-87 F4.2 Accuracy for D (secs) - 3-sigma value
88-91 F4.2 Personal equation (secs)
92 I1 Weight for D (< >, 0 to 5)
93-101 HHMMSS.SS UT of reappearance
102 A1 Event code.
    R for main star
    r for 2nd star
    B for moon with main star
    b for moon with 2nd star
    M for miss
    C for Not Seen - e.g. because of cloud
    P for data point based on predictions
103-106 F4.2 Accuracy for R (secs) - 3-sigma value
107-110 F4.2 Personal equation (secs)
111 I1 Weight for R (< >, 0 to 5)
112 A1 Method code
    _ Visual, with PE applied
    1 Visual, with predetermined PE applied (not observer's estimate)
    2 Visual, no PE applied
    3 Video with frame analysis
    4 Video, photo, or photoelectric
    5 Only duration timed
    6 Drift scan methods (CCD, photo)
    7 Preliminary times only
113 A1 Plot code
    < >= plot both
    x = plot none
    y = plot D only
    z = plot R only
114-118 F5.2 Adjustment to time base of observer, to fit to other observations
119-148 A30 Free text comments
149-150 A2 CR/LF sequence

At the end of each event there is a blank record.

Index file

The file Observations.inx is an index file for the Observations.dat file. The field definition for this file is as follows:

Columns Format Definition
0-3 yyyy year of event. If observations extend past 24hrs, use the earlier date
5-6 mm month
8-9 dd day
10-16 I6 asteroid number
18-30 A13 asteroid name
32-34 I4 number of observation records
36-40 I5 record number in the file of the first header line for the event

Asteroid shape models from DAMIT and ISAM

This form provides ready access to all the shape models held within the DAMIT and ISAM systems.

DAMIT

  • DAMIT = Database of Asteroid Models from Inversion Techniques. The database is operated by The Astronomical Institute of the Charles University in Prague, Czech Republic. A paper describing DAMIT has been published by Durech et al., DAMIT: a database of asteroid models, 2010 A&A 513, A46 (download PDF, 1.2 MB).

The DAMIT system was upgraded at the end of October 2019 to V3, with the V2 version of DAMIT ceasing at the end of March 2020. The changes implemented in V3 necessitated a major change in the way Occult handles DAMIT shape models. Specifically, rather than downloading images of the shape model for a specified date + time, Occult now generates the image locally. This results in a huge improvement in performance, as well as independence from internet connectivity. Associated functionality is described below.

ISAM

  • ISAM = Interactive Service for Asteroid Models. It is operated by the Astronomical Observatory of Adam Mickiewicz University, Poznań, Poland. A .pdf Poster Paper describing ISAM is available here. A detailed paper describing ISAM has been published by Marciniak et al., Photometry and models of selected main belt asteroids IX. Introducing interactive service for asteroid models (ISAM), 2012 A&A 545, A131. (download PDF, 3.1MB).

3D Asteroid Catalogue

  • An additional source of shape models is the 3D Asteroid Catalogue. Most of the shape models are in fact sourced from DAMIT. However it includes a number of asteroids not included in DAMIT or ISAM. However this source has not been used as it does not provide all the information needed to orient a shape model for a specified date+time.

New Shape Model implementation

The revision of the DAMIT website referred to above necessitated an overhaul of Occult to process the raw data that defines a shape model. That data is the orientation of the axis of rotation (λ, β), an epoch for the rotation, a series of vector coordinates that define the coordinates of the corners of triangles used to model the surface, and a series of numbers that links those vertices together to define faces. A typical model will have around 1000 vertices and 2000 linkages. For some asteroids (particularly in the ISAM database) the number of vertices is of the order of 4000, with over 7500 linkages {and for the asteroid (25143) Itokawa there are 6146 vertices and 12288 linkages}.

The previous shape model implementation involved downloading images generated by the DAMIT and ISAM websites - with new downloads being required whenever a parameter was changed. This was slow, and heavily dependent upon Internet availability.

The new implementation involves downloading the model data. Once the data for a model has been downloaded, the model can be drawn for any date/time without internet access. Furthermore, as the image generation is done within Occult, a number of options are available that are not available from the image downloads.

Downloading model data

Two approaches for downloading the shape model data are provided on the DAMIT and ISAM shape models form.

In the first approach, if the  models for a selected asteroid are not present on the computer, Occult will download the relevant files. Of course, the download requires internet access. Once the files for that asteroid have been downloaded, shape models for that asteroid can be generated without internet access.

In the second approach, Occult downloads all the available shape models. Currently (Feb 2020) there are 4207 shape model files for DAMIT, and with a reasonable internet connection they are all downloaded in about 15 to 20 minutes. There are 3122 shape model files for ISAM. However the ISAM server appears to be slow, and the download of all those files requires several hours.

It should be noted that the great majority of asteroids with shape models have never been observed in Occultation. This leads to the following recommended approach:

  • for DAMIT shape models, download the entire dataset. This is done using the menu option on the DAMIT and ISAM shape models form Shape Model Maintenance... DAMIT.... Download/update shape-model files
  • for ISAM shape models - if you are an occasional user of shape models, let Occult automatically download the required shape model files if they are not present on the computer - eventually you will have all the files you need in practice. If you are a regular user, set aside several hours for Occult to download all available models using the menu option Shape Model Maintenance... ISAM.... Download shape-model files.

Maintenance issues

The DAMIT shape model data is subject to regular revision. This may include the addition of new models, the deletion of models no longer considered valid, and changes to the shape model data. For this reason users should periodically (eg every three months or so) run the menu option Shape Model Maintenance... DAMIT.... Download/update shape-model files. This will automatically update the shape model data and files.

As at Feb 2020 the revision of ISAM data is limited to updating the list of available ISAM shape models (using Shape Model Maintenance... ISAM.... update list of ISAM models), after which any new models can be downloaded using  Shape Model Maintenance... ISAM.... Download shape-model files.

Files for shape models

The files for individual shape models are held in the Occult subdirectory ShapeModels. DAMIT shape models are stored under their DAMIT model number (eg 1234.txt). Note that the model number is not directly related to the asteroid number. ISAM shape models are stored using the asteroid number and the model number for that asteroid. The format used is Annnnn-m, where A is an alpha character that ensures the ISAM and DAMIT models are separated when the directory is viewed, nnnnn is the asteroid number, and m is the model number for that asteroid.  For example A1219-4. The total file size of all shape model files as at Feb 2020 is 320 MB.

Thee are two other files in that subdirectory that are critical to the shape model generation. These files contain information about the orientation of the shape model (including its rate of rotation) as well as associated comments and revision dates. These files are generated differently for DAMIT and ISAM:

For DAMIT, the data is obtained by downloading two files from the DAMIT site and linking/merging the data in those two files. This process necessarily creates a file that covers all models in the DAMIT database. This file can be updated using Shape Model Maintenance... DAMIT.... Update list of available models. However this step is automatically done if you use  Shape Model Maintenance... DAMIT.... Download/update shape-model files to update the DAMIT models.

For ISAM, the required data is held in each individual shape model file. As a result, details for an asteroid are only present for shape models that have been downloaded

Resulting improvements

The following is a summary of the changes that have been enabled as a result of the local generation of the shape model images

  • Once model data has been downloaded, shape model generation is very fast
  • The shape model can be drawn in color or Black & White. The color image is a bronze color (as shown below).
  • The axis of rotation can be drawn for both DAMIT and ISAM images - with the north pole being green and the south red. There is also a 'N' or 'S' drawn at the appropriate pole (if it is visible).
  • The orientation of the model is defined by the ecliptic longitude λ and latitude β of its axis of rotation. These values can be varied between ±16° in steps of 4° - to ascertain whether a small change in the orientation of the axis would result in a better fit to observed occultation chords.
  • ISAM models that are duplicates of DAMIT models are identified
  • The need to provide for a different scale as between DAMIT and ISAM images has gone
  • The edges of the faces on the model can be highlighted. This is quite illuminating for many 'old' models which are not based on 'regular' triangles. The following shows an 'old' and 'new' type of shape model, with face edges highlighted.

Measuring asteroid diameters

Asteroid shape models can be combined with occultation results to derive the mean diameter of an asteroid. Guidance on what is involved is provided here

Binary asteroids

As at 2020 there is only one binary asteroid with a shape model file - (90) Antiope. An algorithm has been implemented to provide for the components being colored differently. The colors are bronze for the primary component (that is, the first component in the file) and light green for the secondary. If plotted in grey-scale, the secondary is plotted darker.

The algorithm is based on an assumption of how shape models for binary asteroids may be constructed in the future, and may turn out to be wrong..

Menu bar choices

with Model image

This menu item provides the following general items
  • Draw mean Diameter on image to selectively draw the mean diameter used in the displayed shape image. The setting is applied to all models, and is retained between sessions. Two 'mean diameters' are shown. One is the diameter of a sphere having the same Volume as the shape model, while the other is the diameter of a sphere having the same Surface Area as the shape model. The larger diameter is associated with the sphere having the same Surface Area. [Their calculation is based on the expressions of  Dobrovolskis (1996) Icarus 124, 698-704 (1996), which are used in DAMIT]. A similar set of lines can be drawn on the plot of observations from the Plot event observations form  by setting the menu item Plot options... Show MEAN DIAMETER indicator. Indeed, drawing the lines from the Plot event observations form will usually provide clearer lines.
  • Plot model in Grey scale. This may be important for publication purposes, where a B&W image may be desired/required. The setting is applied to all models, and is retained between sessions.
  • Plot on White background. Draws the shape model on a white background. [Previously, DAMIT and ISAM downloaded models were always on a black background, and the downloaded image needed manipulation to achieve a white background when needed.] The setting is applied to all models, and is retained between sessions.
  • Show face edges on model. This is discussed above. The setting is applied to all models, and is retained between sessions.
  • Include rotational axis to selectively include the axis of rotation. The setting is applied to all models, and is retained between sessions.
  • Plot on Earth plane. The setting is applied to all models, and is retained between sessions.. The default orientation of the plot is the appearance of the asteroid in the sky as seen from the Earth. However for prediction purposes it is more useful to plot the shape as it would appear in projection on the Earth from an occulted star. To get the best benefit of this you need to match the image scale to that of map of the occultation prediction. The easiest way to do this is to plot the predicted path in GoogleEarth, select a point near the center of the path across the Earth, and adjust the scale of the GoogleEarth plot and the shape model plot so that the length of he scale line on the shape model plot - as projected on the GoogleEarth image - corresponds to the known diameter of the asteroid. To do this correctly, make sure you choose a location near where the path is closest to the center of the Earth, and measure the distance along the path. Measuring across the path, or at your observing location, may give erroneous results.
  • Change polar axis orientation. This has a submenu that enable the ecliptic longitude Lambda, and ecliptic latitude Beta, of the axis of rotation to be varied ion the range ±16°. Changes are applied to all models displayed for the current asteroid. However when a different asteroid is selected the values are reset to zero, and are not retained between sessions
  • List Diameters from IR-satellites. Lists all available diameters from the three IR satellites IRAS, AKARI and Wise.
  • List model parameters & light curve dates This lists the following data for each shape models on your computer for  the currently selected asteroid:
    • The identifier for the shape model
    • the ecliptic longitude λ and latitude β of the pole, and its period of rotation
    • The zero date for the rotational ephemeris
    • The ephemeris period of rotation - in hours
    • If the Internet is available, a list of references to light curve measurements used to create the selected shape model will be downloaded from DAMIT. This list is very useful as it indicates the length of time covered by the observations, and the date of the last observation. The longer the time period covered by the observations, the more accurate the rotational ephemeris. The longer the occultation observation is after the last observation, the greater the rotational uncertainty in the shape model.
  • Display model fit to light curves (DAMIT). This downloads a series of plots that show the fit of the light curve generated by the derived shape model to the source photometric observations. This can be useful in assessing whether the shape model is reliable.
  • Copy event JD, DAMIT epoch and period This copies the following data to the clipboard, for ease of access:
    • Model Epoch = nnnnn [... is the zero date for the light curve]
    • Model Phase at Epoch = n.n° [usually 0.0, but apparently not always...]
    • Daily rotation rate = nnnn.nnnnnnn°/day
    • DAMIT Period (hrs) = n.nnnnn [period of rotation]
    • Best fit: [This is associated with fitting shape models to occultation chords. The values correspond to the best fitted phase model]
      • Event JD = nnnnnn
      • Phase correction (revs) = n°
      • Rotation rate correction = n.nnnnnn°
  • Copy image
  • Save image

with Phase models

This set of submenus provides options in relation to multiple images of a selected shape models, showing the shape model rotationally offset. This menu option is disabled whenever a different asteroid is selected - being enabled only when shape models for that asteroid have been downloaded. The options are:
  • Download phase models at .... steps. There are 5 options available. Each option downloads 10 shape models at equal intervals of time, the 5th model being for the specified date + time. The first option covers a full rotation of the asteroid in 10° steps. The other 4 options download the shape images at smaller intervals of time - allowing for a finer fit to occultation chords.

  • Save Phase models as an animated GIF This creates an animated GIF file of the shape model as displayed on the present form.
  • Save Phase models as an animated GIF, against plot of occultations This option is only enabled when the shape models are called from the Asteroidal Occultations observations editor. It creates an animated GIF file of the shape model as plotted on the plot of the occultation observations. It provides a simple way to demonstrate the fit of the shape model to the occultation observations.
    NOTE animated GIF files cannot be created while the images are being displayed as an animation.

Notes on DAMIT & ISAM models

In the majority of cases, where ISAM models are available, they include the models available in DAMIT. However ISAM commonly includes additional models, and sometimes the best fit is obtained with one of those additional models. Accordingly it is important to check all available models against the observations.

Some issues with the list of DAMIT models are:

  • Up to 4 models might be available.
  • If an existing model is refined, the model will be updated under its DAMIT model number and the 'Version' (which is a date) will change. If the new model is very different from the old one (significantly different period and/or pole), which means that the old model was probably wrong, the old model will be erased and a new one will be created with a new model number. This way the existing model can be updated many times. [The case of replacing a wrong model with a better should not occur frequently.] Sometimes one of two models for an asteroid can disappear because it was rejected by new data (occultation or AO image, for example).

Interaction with the Asteroid Observations Editor

When the DAMIT/ISAM functionality is accessed from the observations editor, the program operates as follows:
  • Access the functionality from the Observations editor by selecting the menu item Light curve data & Shapes...Display DAMIT & ISAM shape models. This will open the DAMIT and ISAM form. If no model is available in either DAMIT or ISAM for the particular asteroid, a message box will appear advising of this. Otherwise the DAMIT and ISAM form will have the date and time set to the average time of all the D and R events recorded in the editor for the displayed event, and then proceed to download all available models (assuming your computer is connected to the internet!)
  • To have the model displayed on the plot of the observations, make sure the menu option Plot Options... Add DAMIT & ISAM shape images is checked.
  • You can control which model is displayed, and its size, from the Plot form by selecting the menu option Plot options... Show Shape model controls on plot.
  • The image on the plot of the observations is updated whenever
    • a different model is selected; and
    • the scale of the image is changed.
    This means that you can easily change the models to compare how they fit the observations, and you can easily change the model size to suit the plot size of the observations.

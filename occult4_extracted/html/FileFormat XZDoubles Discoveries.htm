XZDoubles Discoveries.dat

This file maintains data on double stars discovered in occultations. As part of the overhaul of the DSFile information, data in that file system relating to stars discovered as double during a lunar occultation was extracted and combined into a single file. For ease of reference, and compatibility with the WDS2001 format, each discovery observation was allocated a unique reference number, using the format OCCnnnn. The format is as follows:

Column Format Meaning
0 - 6 OCCnnnn Identifier for occultation discoveries. The number was allocated in order of discovery date when this file was created - but this need not be continued for new discoveries.
7 - 12 ... XZ number of the star.
13 ... Double star status
  • "A" definitely a double star
  • "P" a possible double star - reported as non-instantaneous
  • "E" definitely not a double star
    • 14 - 18 hhmmm Right Ascension (2000) - in WDS format
    19 - 23 +ddmm Declination (2000) - in WDS format
    25 - 32 yyyymmdd the date of the discovery observation
    34 - 57 ... Discoverer's name
    59 - 82 ... Reference for the discovery observation. [ON = Occultation Newsletter]
    84 - 91 yyyymmdd the date of the first confirmation observation
    93 -100 yyyymmdd the date of the second confirmation observation
    102-109 yyyymmdd the date of the third confirmation observation
    111-118 yyyymmdd the date of the fourth confirmation observation
    119 ... Light curve flag.
  • " " = no light curve available.
  • "?" light curve is arguable or inconclusive
  • "*" Light curve definitely shows a double star
    • 120-132 NNNnnnn ccccc Double star identifier, and component identifier, in WDS. An entry here indicates that the star has been entered in the WDS or Interferometric catalogues. An entry in WDS will only occur if the star is double. However an entry in the Interferometric catalogue does not mean the star is a confirmed double. The data in that catalogue must be interpreted..
    133-140 +ddnnnnn BD/CD number
    142-148 iiyyMdd Discoverer identifier in the former dsFile. [Format = iiyyMdd, where ii= observer's initials, yy = year; M = Month (1-9, O,N,D); dd=day]
    149-150 ,,, CR/LF sequence

    In 2009 the double star files were thoroughly revised (with columns 13 and 119 being allocated the above values). As part of that revision all stars in XZDoubles that were not listed in the WDS or IF catalogiues were given an entry in this file. Those stars are OCC 1134 to OCC1654. Those stars fall into two categories, with the following characteristics in the file:

    Internal check of the XZ80Q file

    These stars were identified as double stars in the process of creating the XZ80Q catalogue. The criteria used was a separation of less than 10". In the file:
    • the discovery date is set to 2000 1 1
    • the discoverer is listed as 'Unknown'
    • the reference field contains the text 'close in XZ catalogue'

    Unknown sources, from dsFile

    The remaining stars have derived from the old dsFile of double stars. In the file:
    • the discovery date is set to 1980 1 1 - although in a very small number of instances an actual date is given
    • the discoverer is listed as 'Unknown'
    • the reference field is usually empty, but occassionally contains a reference to a star catalogue number

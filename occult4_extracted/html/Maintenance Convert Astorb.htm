Download and Convert MPCORB, AstDyS-2 and Astorb

Download files

The program can retrieve orbital elements of asteroids from  the Minor Planet Center (MPCORB), the Asteroids Dynamic Site (AstDyS-2) which is sponsored by the European Space Agency, and Lowell Observatory (Astorb). Ftp links are provided to the primary site, and to a mirror site for MPCORB and Astorb.

The preferred source of elements is MPCORB, followed closely by AstDyS-2, and then Astorb. MPCORB is updated daily, AstDyS-2 is updated monthly, and Astorb is updated daily. Astorb has a useful data element that is not present in MPCORB or AstDyS-2 - an estimate of future positional accuracy which is called Peak Ephemeris Uncertainty (PEU). PEU can be used to estimate the uncertainty in an occultation prediction. Since all three essentially use the same set of observations to derive orbits, the PEU values used in Astorb are added to MPCORB or AstDyS-2 in the conversion process (providing Astorb is present on the user's computer.)

MPCORB download

The primary location for the MPCORB download is the Minor Planet Center. MPCORB is updated daily. The file is downloaded as a Zip file. After the download, the program will unzip the file.

If the MPC server is unavailable for some reason, the file can be downloaded from a mirror site run by the Czech Astronomical Society at Klet Observatory.

NOTE. In late 2017 Harvard University limited internet access to the Minor Planet Center to the https protocol. This is implemented in Occult v.4.5.0.0 and subsequent. If you have trouble getting a connection to download MPCOrb, read MPCOrb database.

AstDyS-2 download

A second source of elements is the Asteroids - Dynamic Site generally known as AstDyS-2. This site is sponsored by the European Space Agency, and is updated on a monthly basis. The file that Occult downloads is 'allnum.cat' - which contains Keplerian orbits for all numbered asteroids (but no un-numbered asteroids). However it does not include the names of the numbered asteroids; when AstDyS-2 is converted (see below) the asteroid names are taken from either Astorb or MPCORB.

Astorb download

The primary location for the Astorb download is Lowell Observatory. Astorb is updated daily. The file is downloaded as a GZip file. After the download, the program will unzip the file.

If the Lowell server is unavailable for some reason, the file can be downloaded from VizieR, which is ipdated on a monthly basis In both instances, the file is downloaded as a GZip file. After the download, the program will unzip the file.

 

Convert the file

The orbital elements need to be placed in a format used by Occult. Apart from the format, the conversion adds diameters - either from the file Asteroid Diameters.dat, or calculated from the H0 magnitude. Also added is the number of moons (from the file BinaryAsteroids.csv), the number of rings (from the file AsteroidRings.dat), and the taxanomic class (from the file AsteroidClasses.csv). For conversions of MPCORB and AstDyS-2, the PEU value from Astorb is added. For conversion of AstDyS-2, the asteroid's name is added from either Astorb or MPCORB.

The conversion step allows the user to set some selection parameters, so that the objects contained in the converted file are those having a greater likelihood of a successful occultation observation.

1.Select the file to convert

Check the relevant file to convert. Usually it will be the file that you have just downloaded!

2. Set the selection criteria

This is the most important part of the conversion process. It controls what asteroids will be included in the file. The choices are:

  • Asteroid diameter. This is the primary selection criterion. The diameter set here is retained between sessions. Things to remember are:
    • the probability of observing an occultation reduces as the diameter reduces
    • The following plot shows the number of asteroids against diameter. The vertical scale is log10, and ranges from 1,000 to 1,000,000. As a general guide, halving the diameter will increase the number of asteroids by a factor of at least 2.
  • Limit to numbered asteroids. This should be checked, unless there is a special reason to consider un-numbered asteroids. Remember that the fundamental reason why an asteroid is not numbered is because the uncertainty in the orbit is too large. Consequently the uncertainty in an occultation prediction of an unnumbered asteroid will usually be so large as to make the prediction meaningless.
  • If un-numbered asteroids are included, limit to those observed at 3 or more oppositions. In the event that un-numbered asteroids are to be included, check this box to limit the asteroids to those that have been observed at a minimum of three oppositions - with a consequential presumption that there is some accuracy in the orbit. Note: Only MPCOrb includes a field for the number of oppositions. For Astorb the criterion applied is an observed arc of at least 1500 days.
  • If un-numbered asteroids are included, include a particular asteroid. This allows the user to specify a particular un-numbered asteroid to be included - irrespective of the diameter criterion or the number of oppositions. As a result you can generate predictions of a particular un-numbered asteroid without needing to include a large number of unnumbered asteroids generally, or reducing the diameter of asteroids to imclude.
    Note: you must make sure the asteroid identifier is strictly in accordance with the standard format, as in: '2006 RN102', '2006 TX17', '2000 DN8', '1999 YH'. In all instances, there is a single space between the year and the letters, and the letters are uppercase.

3. Specify 'mandatory' minor planets

This location is used to specify numbered planets that should be included in the conversion - irrespective of the selection criterion. There are two uses for this list:
  • to ensure small asteroids that have been observed in an occultation are included in the asteroid file, for use when analysing observed occultations. Such asteroids should be retained in this list.
  • to include a particular asteroid for a special prediction. In this situation you would sensibly remove the asteroid from the list after the conversion has been completed, to ensure the asteroid was not unnecessarily included in subsequent conversions.

4. Convert

Click Convert. Select the source file. A progress bar will indicate progress.

When the conversion is complete, the program will automatically update the User File of asteroid elements used for lunar occultation predictions. Some users have reported an error at this point; the solution is to:

  • in the first instance, ignore the error - it is not fatal, although there may be a problem with Lunar occultations;

Gaia format

This format is designed to handle the precise positions provided by Gaia. It is used for Gaia-based catalogues, as well as 'user' catalogues derived from other catalogues downloaded from VizieR using the TAPVizieR tool. Two formats are specified below. The EDR3 format supersedes the DR2 format. The DR2 format is being retained in Occult for a period, so that catalogues created using the DR2 format can continue to be used. However the capability of creating Gaia catalogues in the DR2 format has been replaced by that for EDR3/DR3.

42 byte --- Reliability indicator  The reliability indicator provides a broad indicator of whether there are issues of concern that may affect the accuracy of the star's position beyond the formal standard deviation measure. The range of the indicator as provided is 0.0 to 12.6.

For stars where the position has come from Gaia DR3, the reliability indicator is the Gaia 'RUWE' value. Values of RUWE in Gaia range from about 0.3 to over 100; values greater than 12.6 are all coded as 12.6 in this field. Values of RUWE greater than 1.4 indicate reduced reliability of the position. Values of zero can also exist; in such instances the position is of poor reliability.

For the approximately 1000 star positions obtained from TGAS (Gaia DR1), the 'astrometric_excess_noise' element of that catalogue has been used as the reliability indicator.  This is described in 'Gaia Data Release 2 - the astrometric solution' A&A, 616, A2 (2018). Also in the presentation "Astrometry in Gaia DR1' at the Gaia DR1 workshop, ESAC, 2016 Nov 3.

For the small number of stars where the position is taken from Hipparcos, the indicator is derived using Eqns B1-B4 on page 15 of Daniel Michalik et al: Joint astrometric solution of Hipparcos & Gaia, A&A, Vol 571 (Nov 2014).These give the chi statistic for the star position, 'which is expected to be around 1.0 for a "good" solution'.
43 byte --- Flags The bits in this field are used to flag a range of situations:
Bit 1 - set if  the Gaia Duplicate Source flag has been set
Bit 2 - set if there is no Gaia proper motion for the star.
Bit 3 - when bit 2 set, the proper motion has been derived using the UCAC4 position at 2000.0
Bit 4 - UCAC4 identifier based on a poor match
44 byte 0.1 mas Star diameter Apparent angular diameter of the star. Derived using the gaiadr3.Astrophysical_parameters_supp file, field radius_flame_spec,  (Rad-Flame in VizieR), and  gaiadr3.Astrophysical_parameters file, field radius_flame,  (Rad-Flame in VizieR).
The solar radius value is converted to apparent solar diameter with:
      diam(mas)= 9.304 * parallax(asec) * radius_flame
Field is empty in EDR3

UCAC4 matching issues

  • In general, matching of UCAC4 to Gaia is undertaken by comparing epoch 2000 positions derived from Gaia with the UCAC4 catalogue positions. The categories for a match are:
    • good match: Separation < 0.2", and magnitude difference < 2  [about 97.5% of entries]
    • poor match: Separation < 1.0", and magnitude difference < 2  {about 99.6% of entries]
    • No match: Separation >1.0" or magnitude difference > 2  [about 0.4% of entries]

  • Where a Gaia star has no proper motion (RUWE is zero in such cases), a match of UCAC4 to Gaia is undertaken using UCAC4 positions with proper motion applied to the Gaia epoch of 2016. If there is a match to better than 1.0" and magnitude difference less than 2,
    • the UCAC4 number is added,
    • the catalogue identifier is set to 8,
    • RUWE is set to 0.05,
    • proper motions are added, being derived using the UCAC position at epoch 2000 and the Gaia position at epoch 2016.
    • Possible multiple allocations are dealt with in the general routine for dealing with multiple matches.
  • Where there are two possible Gaia stars that match with the same UCAC4 star, the criteria for allocating the UCAC4 number to a star is as follows, in order of priority:
    • if one or both the stars originally had no proper motion, the catalogue identification for the relevant one or both stars is removed, and the proper motions and RUWE are restored to 0.
    • If the match of one star is better than the other (as reflected in the Catalogue ID number of 6 or 7), the UCAC4 identification is removed for the star with CatID of 7
    • if RUWE is >2.0 for one, and <1.5 (but neither is less than 0.15) for the other, the UCAC4 identification is removed for the star with the higher RUWE
    • if the separation for one star is greater than that for the other star by more then 0.1", the UCAC4 identification is removed for the star with the greater separation
    • Otherwise, the UCAC4 identification is removed for the fainter star (Green magnitude)
  • The routine will only handle three close matches in seriatim if the match for the second  star is not removed in the first instance.

Gaia  .inx file format

This file is an index file to the Gaia catalogue. The name is the file that it provides the index for, but with '.inx' rather than '.bin'.  It is used for all Gaia-based catalogues

The Gaia catalogue is structured as 360 declination zones (0.5° wide) starting at +90°.

The .inx file has 361 entries for each declination band. Each entry gives the record number (as a long) for the first record in a band at intervals of 4 minutes in RA, with the last index entry in that band being for the last record in that band. That is, the record number gives the first entry in a declination band of a 4-minute block that contains the specified RA. The following record entry gives the index of 1 more than the last entry in that block.

Using a zero-based record number, the algorithm (in C#) to get the record number, given the star's coordinates (RA, Dec) in radians, is:

Accessing a star using its Hipparcos number

For each Gaia... file, there is a corresponding Hipparcos_Gaia ....dat file. This file is a binary file containing 120404 entries corresponding to full range of Hipparcos star numbers. Each entry (a 32 bit integer) gives the records number of that Hipparcos star in that Gaia catalog. The Hipparcos catalogue has some gaps in the numeration, with there being only 117953 stars. Entries that do not have a Hipparcos star have the record number set to zero.

To retrieve a Hipparcos star, simply retrieve the Gaia file record number from the record number in the Hipparcos...data file - then go to that record in the Gaia file

Accessing a star using its Tycho2 number

The Tycho2 star numbers are that of the Guide Star Catalog - GSC. Stars in the GSC have 3 elements. The first is a region number, the 2nd is a number within that region, and the 3rd is for components of a star. For the huge majority of stars, the 3rd component is 1. For a small number it is 2. Very occasionally it is a  numbers.

The file format is here (GSC Fields)

To locate a specific star in a Gaia catalogue from its Tycho-2 number:

  • use the Region number (the first element in the Tycho2 number) to extract from the GSC Fields file, the  file the limits in RA and Dec for that Tycho2 region.
  • for the Gaia file being accessed, use its corresponding .inx to obtain the range of records to be scanned in the 0.5° bands of that Gaia catalogue file.
  • Scan the Gaia file to find an entry with the required Tycho2 number

Not all stars with a Tycho2 number will be found. There are two reason:

  1. Each Gaia entry contains on a single star identifier. A Hipparcos number has precedence over a Tycho2 number, and as a result any Tycho2 stars which are also in Hipparcos cannot be located using the Tycho2 number.
  2. The Tycho2 numbers in the Gaia catalogues have been obtained via the TychoGaia file of Gaia DR1. Beyond that, no effort has been made to match Gaia stars to the original Tycho2 catalog.

NOTE 1: There are instances where a Tycho2 star from the TychoGaia that could not be reliable matched to Gaia. In such circumstances the star position from TychoGaia was added to the Gaia catalogue, with the likelihood of there being a relevant Gaia star being already present - creating (in effect) a double entry for that star. That Gaia star star will usually (but not always)  have a UCAC4 identifier.

NOTE 2: The Tycho2 star numbers within a Region are not ordered by Right Ascension or Declination. As a result you should not try to limit the scans by having regard to whether the current Tycho2 number is less than the last-found Tycho2 star number..

Accessing a star using its UCAC4 number

The UCAC4 catalogue has 900 zones, from 001 to 900. Each zone is 0.2° wide, with the zones being numbered from the south pole to the north pole. The UCAC4 number is a combination of the zone number, and the sequential number within that zone. The binary file U4_Gaia14.inx (located in the Resource Files folder) has 361 integers for each zone, corresponding to the start of 4-minute intervals in RA. The integer value is the UCAC4 sequence number of the first star in that interval. If this file is not present, Occult will download it automatically.

Sample code (in C#) to retrieve the Gaia declination zone numbers, and the RA (in degrees) of the 4-minute interval containing the star number) is:

Identify poor star positions

This option generates a list of entries where there appears to be a problem with the star position.

Selecting the events to check

The selection of events is the same as the selection of events in the Observations editor. Except that all the events satisfying the criteria are presented in a list, rather than a drop-down box. That is, use the option buttons and drop down boxes to select a range of asteroids; those asteroids will be listed, and the output will be based on that list.

Generating the list

Click Check star positions to conduct the check. If the check box use UCAC4 is checked, the program will use the UCAC4 catalogue as the basis for Hipparcos and Tycho2 positions.

List content

The output is a series of blocks of the following form: 
112 Iphigenia 2005 Feb 25
0834-00330-1 
 9 51 53.6971 :  9 51 53.6954
 12 30 10.418 :  12 30 10.431
with the list finishing with the line: 
Check completed
  • an event is output is the difference exceeds 0.0014s in RA, or 0.021" in dec (equivalent to 1e-7 radians in both coordinates)
  • The first line identifies the event.
  • The second line identifies the star.
  • The third and fourth lines give the RA and declination of the star. The coordinate before the colon is the value currently held in the AsteroidObservations file. the value after the colon is the value that this check routine thinks shoould be present.

Interpreting the results

The primary purpose of the list is to identify events where the star position should be re-determined. Note that not all mismatches are in fact actual errors (a residual programming bug somewhere...)

For Tycho2 stars, if there is a small difference in coordinates - the most likely explanation is that the star position currently in file is a Tycho2 position rather than a UCAC2 position.

Where there is a large difference in coordinates, the star identifier will almost certainly be wrong. Check formatting of the number carefully.

Processing eVscope observations, from Unistellar

The eVscope is a fully-electronic telescope system manufactured by Unistellar Inc. The system has an aperture of 11 cm, fully automatic alignment, time synchronized using a mobile phone, and has a fully-electronic detector. Some background to the system is provided here. Observations made by individual users are analyzed by Unistellar.

 

Unistellar reports its observations to IOTA via xml files in the AsteroidObservations format. Many of the fields are not populated; they will be populated when an event is read into the Observations Editor in Occult.

The files supplied by Unistellar fall into three categories:

  1. events containing observations for an event already in Occult
  2. events which are not in Occult
  3. events containing only Miss observations, with no event in Occult

These categories can only be properly considered if there has been sufficient time from the date of the event. The code applies a time minimum of 100 days - sufficient for events to get from the observer to the regional coordinator and then to the overall coordinator for inclusion in the Occult file of observations. However (1) can be assessed at an earlier date,  when the event has been included in the Occult file.

The functionality has two steps: Sorting events according to the above criteria, and processing events of type (1). Events of type (2) are processed using the Observations Editor File... Open Multiple Files to review and add the individual event files. Events of type (3) are to be ignored.

The sort process needs to be run on a regular basis, while ever there are files in the directory where the Unistellar files have been initially placed.

 

Sorting the Observations

Firstly, place the Unistellar observations in a folder of choice. A convenient location might be in a directory structure in Occult like:

     Occult 4/Asteroid/Observations/yyyy/eVscope

where yyyy is the year of the observations.

To sort the observations, from the Asteroid Observations Editor, select  File....eVscope: match to existing observations. This will display the following form.


The first step is to select the directory containing the files to be processed. This selection applies to both the Sort... and Process... options on the form.

To sort the observation files into the three categories above, click the Sort... button. The results of the sort are displayed like:


The sort process moves the files into new subdirectories of the directory holding the eVscope files. Those directories are:

....eVscope/EventsToAdd   containing new events that need to be added to the Occult file

....eVscope/ObservationsToAdd    containing events having observations to be added to an event already in the Occult file

....eVscope/EventsToIgnore    containing events with only Miss observations, with no corresponding event in the Occult file

The directory ....eVscope will retain any event files where the sort date is less than 100 days from the event date - except for any events matched to an existing event.

 

Subsequent processing

The files in ....eVscope might still be matched to an event in the Occult file. They will remain there until they are either matched, or the routine is run more than 100 days after the event date.

The files in ....eVscope/EventsToAdd  should be processed in the same manner as any new observation files provided in the Occult .xml format. That is, from the File... Open Multiple Files menu item on the Observations Editor form

The files in ....eVscope/EventsToIgnore require no further consideration.

The files in ....eVscope/ObservationsToAdd contain observations in an event that has been matched to an event in the Occult file. It cannot be assumed that all the observations in such files need to be added to the existing event in the Occult file, as some observations might already be present. Also (and as usual) all observations need to be checked for reliability etc.. The button Process the files for Existing Events assists with adding these extra observations. When selected, a form looking like the following will appear:


The form displays an event in the ....ObservationsToAdd folder. The file name is given at the top. The event date and asteroid are listed under the tags <Date> and <Asteroid>. The form is displayed with its full set of observations highlighted for selection. To process:

  1. In the Observations Editor, open the relevant event that exists in the Occult file

  2. Compare the Observer names as listed in this form, and the Observations Editor, and identify any observations on the displayed form that are not in the Editor

  3. Select (by highlighting) the observations that are displayed on the form that need to be added to the event. Make sure that the tags <Observer> and </Observer> are included in the selection (if either is not included, that observer data will not be included.)

  4. Copy those observations.

  5. On the observations editor, select the menu item Paste... New observer lines [one or more groups] - XML format This will copy those observer entries into the Occult record. The Paste will also sort and renumber the observers in accordance with distance from the path.

  6. On the form that plots the observations, click the button on the menu bar Set Miss Times, and from the form that appears click Correct times in report. This will ensure appropriate times are given for Miss events.

  7. Review the added observations, and the observations as a whole, in the usual manner. When satisfied, Update the record in Occult

  8. On the form, click Mark file as processed and display next file. This will add the text _done to the file name just before the extension. This will prevent the file being selected in future runs. It will then move to the next file and display it.
    It is important to click the  Mark file as processed and display next file. The form does not allow for user selection of files to process. Rather it steps through the files in the directory, skipping over files marked with  _done This is to ensure all files are processed.
    You can cease processing at any stage by clicking Exit. When you next run this form, it will continue by working through all files that have not had  _done added to the file name.

  9. When all files have been processed, you will get a message, and the form will close.

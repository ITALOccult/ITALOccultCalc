Asteroid Search

Overview

An asteroidal occultation consists of a path across the Earth having a width equal to the diameter of the asteroid. That is, the width of the path is only a fraction of the Earth's diameter. Furthermore, some of the path may be in locations which are sunlit at the time of the occultation, and/or may be in locations which are relatively inaccessible.

 Occult generates predictions in a two-stage process.

Stage 1 is

  • specify the asteroid or asteroids to be considered in the search;
  • select the star catalogue to be used in the search;
  • set the start and end of the period to be covered by the search; and
  • generate a set of Besselian elements for all events where an edge of the occultation path comes within a specified distance of the Earth's center (default distance is 1.01 Earth radius - with allowance for the effects of stellar diameter being incorporated where appropriate).

Stage 2 is

  • filter the list of events to those that are of potential interest. Those filters include parameters like solar elongation, local altitude, minimum duration, minimum magnitude drop, faintest star magnitude, and distance relevant to the observer's location.
  • save the list of events for future reference
  • create a 'personal feed' for Occult Watcher
  • sort the events according to a range of parameters

At any stage, selected events can be displayed, with a wide range of related information being accessible.

The present topic deals with Stage 1. The Help topic for Stage 2 is here

The Search     (stage 1)

This is the form for generating the Besselian occultation elements required to generate predictions of asteroid occultations. Each part of the form is described below.

Set dates


In this section, set the search period to be covered. Specify dates in years, months and days - which are all treated as UT. Buttons facilitate the selection of various periods.

  • The Year button sets a 12 month period commencing at the start of the current month specified in the Start date;
  • The Month button sets a period for the entire current month specified in the Start date.
  • The Now button sets a period of 10 days starting from 'today' (set on a UTC basis, not local time).
  • The three year buttons set a search period of January 1 to December 31 for the current year, the following year, or two years in the future.
  • Click the text 'Add one day' (located under the Start date and has a yellow background), and the end date is set to one day after the start date. Importantly, when this option is clicked, the end date is set to one day later, and the form for selecting an asteroid is opened. If a single asteroid is selected, the search will be automatically conducted  [after having downloaded the orbital elements from Horizons if you are connected to the internet.]

Search Filters


The search for occultations identifies events that may be visible in any part of the world, and with no regard to duration or magnitude change at occultation. As a result, the majority of events found in a basic search will be of no interest to a particular location and observer.

To ensure the results of a search are generally relevant to an observer, six 'filters' are provided at the Search stage. (Extra, and detailed, filters are provided on the List & Display form )

  • Mag drop less than an amount. This excludes events with small magnitude drops.
    This test is not applied to Planets, as the considerations are more complex (eg an occultation against the dark limb of a crescent Mercury or Venus).
  • Diameter less than an amount. Excludes asteroids and planetary moons on the basis of diameter. If the search is for a range of asteroids, you need to set the minimum asteroid diameter of asteroids to include in the search. If you clear the check box, you will get a warning message about the large number of events. The number of asteroids against diameter is shown in the following plot. The vertical scale is log10, and ranges from 1,000 to 1,000,000.
     
  • Stars fainter than an amount. Excludes stars based on magnitude. Values between 6.0 and 21.0 can be set. When applying this filter, you should select an appropriate Gaia catalogue. For example, if the star magnitude was set to 12.5, a search using Gaia14 will give full coverage, but will be considerably faster as the number of stars to be skipped over is less.
  • Duration less than an amount. Excludes events based on duration. Values between 0.01 and 4.0 secs can be set. Generally a value of 0.3 to 0.5 would be most appropriate. This filter should usually be checked - to avoid a large number of events with very short durations. When you uncheck this filter, you will be given a warning message.
    For extensive searches involving small asteroids, the output may contain a large number of events having a duration too short for reasonable observing. While such events can be filtered at in the List and Display form, the performance can be seriously degraded as a result of the large number of events. If the search is for a range of asteroids, you should set the minimum asteroid diameter of asteroids to include in the search.
    Clear the check box if you don't want the size limit to be applied. This is appropriate for predictions of a single small asteroid. If it is cleared for a search for many asteroids, you will be overwhelmed by the number of events for any asteroid that is small.
  • Taxonomy. A drop-down box that provides a list of taxonomic classes of asteroids. To limit the search to a specific taxonomic class, select the class from the list. The search can be limited to objects falling within any of the standard taxonomic classes (which are explained here) by selecting the taxonomic class in the drop-down box. For the usual search, set this to display All. All classes are unique except for (i) Potentially Hazardous Asteroids (PHA) which taxonomy that can also apply to Amor, Aten and Apollo classes, and (ii) Binary asteroids, which can apply to any.
    To search for a set of asteroids within a class, first select the Taxonomy class. Then  use the Select asteroid... button to set the range of asteroids to be considered. Only asteroids within that range having the selected taxonomy will be considered.
  • Using xx cm Telescope with X variable exposure camera. Set the aperture of your telescope, and set the check for whether you are using a camera which you can set the exposure (for example, a CMOS camera, or integrating video).
    This test works on the basis (assumption) that a 20cm telescope, using an exposure of 0.037 secs (half-way between PAL and NTSC video frame rates), can reliably record a star at magnitude 12.0 under reasonable conditions. Changes in the aperture appropriately changes the magnitude limit. If the exposure can be varied, the magnitude limit is set such that at at least 3 exposures can be taken during the maximum duration. If the predicted magnitude drop is less than 0.5 mags, a larger number of exposures must be possible during the maximum duration.
  • Passes within This identifies events where the occultation path passes with a specified distance from the observer's location. The location can be keyed in. Or it can be selected from any of the Site files the user has in Occult. The distance calculation allows for the diameter of the asteroid. It excludes events where
    • the distance to the edge of the path is greater than the specified distance. [This includes planetary moons, but not Planets.]
    • the star is more than 5° below the horizon at the time of the event
    • the sun is higher than 8° below the horizon, and the magnitude is fainter than 8.
    • the sun is above the horizon, and the magnitude is fainter than 5.
    However if the motion of the asteroid is particularly slow (taking more than 2 hours to cross the Earth), these filters are not applied.


Select asteroids (or comets) to search, or planets to search

The options that are available depend upon whether the object is an asteroid, or a planet (including moons of the major planets).

Select asteroids to search (main file of asteroid elements)

There are several options to select the asteroids to consider in a search. All are accessed by clicking Select Asteroids, which brings up the following form. This form is used both for the main file of asteroids, and the User file of asteroids.

The options for selecting asteroids are set out on the form.

If a specified asteroid is not found in the file, you will be advised and asked to re-enter the asteroid identifier. You can leave the form by selecting Cancel.

The bottom button Select from a list of asteroids, will list all the asteroids in the main file (or the User file). From there you can select either a single asteroid, or a range of asteroids. However you cannot select a series of asteroids separated by other asteroids.

 

use Horizons elements

You can conduct a search based on orbital elements downloaded from the JPL Horizons system. When the Internet is available, this option should be used when possible.  This is particularly the case where the basic integration scheme in Occult is inadequate, which may be the case when:

  • orbital eccentricity greater than about 0.4;
  • semimajor axis less than about 2
  • geocentric distance is less than about 0.5 AU

The functionality is only available for searches:

  • for a single asteroid; and
  • limited to 12 months duration;

If you set the Set automatically check box, the use Horizons Elements will be automatically set when the search meets the single asteroid, less than 12 months, requirements.

Clear this check box when you do not want to automatically download Horizons elements - for example, when using a USER file of elements. This option will be automatically set when the conditions for using this option are met .

 

use Horizons ephemeris (predictions will be slow)

This option will result in the most accurate predictions. It is not limited to any date range, nor to the number of asteroids included in the search. The setting is also independent of the 'use Horizons elements, or use USER file of elements settings. However there is a major issue of execution time....

When set, Occult will download an ephemeris of the asteroid (4 positions at 1-hour intervals, but at longer intervals when the asteroid motion is very slow) for each event it finds, and use that ephemeris for the final prediction details. When conducting a search, this download will occur for each and every event that is added to the Search Results listing. This necessarily slows the prediction speed - perhaps by a factor of 10 or more. This may become a nuisance when a large number of predicted events are likely - such as a search covering a large number of days, or when the asteroid is crossing a region with many stars.

When this is set, you will see a download notice as each event found is added to the Search results list. On the subsequent plot of the event, the ephemeris source (bottom right of the Header block) will include the text Horizons Ephem.

This option is the most preferred way of generating the most accurate predictions. It supersedes the option available in the List and Display form.

This setting is not retained between sessions.

 

Select asteroids to search - User file of elements

There can be occasions where a user wants to generate predictions based on orbital elements they have generated, rather than those available in sources such as MPCORB, AstDyS2 and ASTORB. The option User file allows predictions to be generated on the basis of elements contained in the user file, which is edited from the main asteroid predictions tab. Check the 'with USER file of elements' check box to use the User file. The search will be conducted for all objects in the file.

 

Binary asteroids - Miriade

The Miriade system at IMCCE- Paris Observatory can provide an ephemeris for the positions of the components of a small number of binary asteroids relative to the system barycenter (which is otherwise the basis of predictions). When Occult conducts a search for an asteroid included in the Miriade system, occult will automatically access Miriade to obtain an ephemeris relevant to the event date, and generate separate predictions for the each component. For some asteroids the Miriade system includes multiple orbit solutions; by default Occult will generate predictions for each component against each orbit solution.

The controls provided are as follows:

  • A list of asteroids included in the Miriade system. The Miriade system will only be accessed for asteroids included in this list. Currently (Feb 2019) this list includes 11 asteroids plus Pluto (134340). Over time more asteroids will be included in the Miriade system. Occult will automatically update the list every 10 days - and the date of the last update of the list is displayed next to the box.. The Update list button will also do an update - so that if you become aware that a new asteroid has been added into Miriade and you want to generate predictions before the next automatic update of the list, you can manually update the list.
  • A check box to exclude the querying of Miriade. Check this if your internet connectivity is slow (such that the search speed is unacceptable for these asteroids), or you do not want the output to include the multiple entries for the varies components of a system.
  • A check box to limit the output to the first orbital solution. The Miriade system generally includes multiple orbit solutions for each system - with each resulting in a different occultation path location. There can also be differences in component diameters. By checking this box, the output will be limited to a prediction that takes no account of the Miriade data, plus predictions for each component using the first orbit solution (which  IMCCE has assessed to be 'the best' solution.)
  • A button to display the basis of the orbit solutions available at Miriade. Details
  • A check box to save the last Miriade response. This is provided solely for test purposes. The Miriade response is saved in the DownLoaded Files subdirectory with the file name LastMiriadeResponse.xml  Details about the content of the XML file are available here

Output for Miriade binary asteroids

Occult will provide separate outputs for each component of a system, for each orbit solution - structured as follows:

  • the asteroid number is present for all components, with no modifications
  • The object name is the usual asteroid name for the primary component, and the satellite name for other components. The satellite name may be a 'proper name' if it has been so named; or it may be a discovery designation (such as S2016-107-1 for the first satellite discovery in 2016 for the asteroid 107)
  • Where Miriade data has been used, the object name has added to it a solution identifier made up of '#' and a number 1, 2, 3 etc. representing the solution number. Note that  an output that does not use the Miriade ephemeris will always be present - it will have the asteroid's name but no # tag.

Set output file


Here you set parameters relating to the output of a file of the Occultation elements. Those parameters are

  • The file name. The currently set file name is displayed in a box. For convenience, a default file name is provided, based on the current UT date [yyyymmdd] at the time the form is opened. However the user can set another name and location. Note that the output file is an XML file - so you should maintain the extension as XML, as prompted. IMPORTANT. If the file exists you will be asked whether you want to overwrite that file; if you specify no, you will be taken back to the search form where you can specify a new file name. Otherwise the previous file will be overwritten.
  • whether the search results are automatically saved.
    IMPORTANT. Saving of search results is intended to occur under the List and Display functionality, where the output can be limited to events of interest. As a result, you should leave this box unchecked. However you can automatically save the search results by checking this check-box.
    The status of this check box is retained between sessions.

It will often be desirable to give the output file a more meaningful name, or add the output to an existing file. Click the Set output file button, and specify the file name - either by selecting an existing file, or keying a file name in association with the directory of your choice.

Important. If adding to an existing file, make sure it is a file of occultation elements. Otherwise the file may not be readable.


Check/calculate coordinates for integration


This block at the bottom of the form indicates the availability of files needed for the numerical integration of the orbit. If a file is available, the relevant button is disabled. You need to make sure that the period from the osculating date of the orbital elements to both the start and end of the search period is covered by these files. [For searches of current events, only the 1995 Oct 10 == 2023 Feb 25 button is required.]

If you need to create one of these files, merely press the relevant button. The file will be generated in a few seconds, and the button disabled. HOWEVER you will need to have downloaded the JPL_DE ephemeris file - using Download #34. If you have downloaded several JPL files, it is not critical that you use the latest DE version for the integration scheme. You merely need to ensure you are using a version later than DE420. 

This functionality is also provided under the menu item 'numerical integration data... '.

If you attempt to conduct a search which requires one or more files to cover the interval that are not available, you will be prompted to create the file.


Do the search


In this region, click Search to conduct the search. Items in this area are:
ItemComment
Adjust asteroid (Planet) positionThis allows the user to apply a correction when computing a prediction. This ability is only enabled when the prediction is for a single asteroid, or a single planet. In the past such corrections were most relevant for Pluto (with a default correction being set in the User Settings, in the box Asteroid occultations - search and display) - where there were problems with its ephemeris. In recent times the need for a correction is associated with consistent offsets of an asteroid from its ephemeris, and for asteroidal satellites where an ephemeris is not available from Miriade

The correction is specified in mas (up to ± 2000mas) - with the correction being the angular separation in the sky plane. This is important for Right Ascension - as a difference in the Right Ascension coordinate values will need to be multiplied by cos(δ) to give the offset in the sky plane in the RA direction.

Examples of possible offsets are:
* to specify the location of an satellite relative to an asteroid. Insert the sky plane offset of the satellite relative to the asteroid;
* to correct the position of the asteroid.  Insert the sky plane correction to the asteroid's position;
* to correct the position of the star.  Insert the negative values of the sky plane correction to the star's position;
Increase miss distance by X earth radiiIn a normal search, occultation elements are output if the minimum distance of the center of the occultation path from the center of the Earth is less than 1.01 earth radii plus the asteroid's radius Occasionally there is a need to identify 'close misses', or appulses. This option increases the allowable 'miss distance' for events to be fully calculated and output - with a range of up to 999 Earth Radii.
JWST Check to generate predictions for the James West Space Telescope.  Presently (2020) this option is provided for testing purposes only, and uses a crude approximation that the JWST is exactly at the L2 point. Events are output if the distance from the JWST location is less than 5% of the Earth's radius (i.e. 319km)

Search/Cancel button

Progress bar

When the search is commenced, the search button is replaced by a cancel button. The functionality of cancel is slightly different between Asteroids and Planets. For Asteroids, the Cancel has effect at the conclusion of the search for the current asteroid, whereas for planets it has effect at the conclusion of the current search day.

The displayed progress bar has a similar difference. For asteroids, it displays how many asteroids have been completed in the search, whereas for planets the progress bar is reset at the end of each object, and displays the current position in the range of dates being searched

At the end of this section there is a label that indicates the orbital elements being used, and their download date. If that date is more than a month or two old, consider whether you need to do a fresh download and conversion.

Results summary


This last section of the form gives a summary list of events found as the search progresses. The list is renewed for each new object being searched, with the current object being indicated at the top left of the box.. A '#' symbol after the star's identifier indicates the star may be a double star. At the end there is text that sets out the period covered by the search, and the range of objects considered.


Conclusion of the search

When the search has concluded, you will be automatically taken to the List and Display form, where you can filter the list of events, and view individual events.

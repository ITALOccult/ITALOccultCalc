Minor Planet Occultation Observations - Overview

Add / Edit / Plot observed Asteroidal Occultations

This is the main function on this form. It provides an editor to record and analyze observations. Included is a file of historical observations - with over 1000 separate events.

The observations editor can make use of Asteroidal shape models which are available on DAMIT and ISAM web sites. However if either of those web sites is 'down', the functionality of the editor can become very slow. A tool is provided to establish whether those sites are currently on-line. The editor will only attempt to access the shape models if the relevant check box is ticked. [Note - if either DAMIT or ISAM is 'down', the check can take in excess of 20 seconds - and double this if both sites are down.] The result of that check is applied to all three locations where the check is provided.

Analysis tools

AOTA

Asteroidal Occultation Timing Analyser. A tool for analysing a light curve as measured by either LiMovie or Tangra, detecting possible events, and deriving the event times and uncertainties.

Star diameter analyser

This is a tool for analysing multiple light curves from an occultation to derive the diameter of the occulted star.

Light curve simulator

A tool to derive a 'theoretical' light curve taking into account the star's apparent diameter. The simulator allows the user to draw an asteroid shape, and plot the star path anywhere across the asteroid.

Light curves

Report

A tool for reporting the light curve of an asteroidal occultation. [The tool for reporting the light curve from a lunar occultation is part of the Lunar Occultation Observations Editor]

View

A tool for viewing past light curves held in Occult. It also allows you to view light curves you have prepared but not yet submitted.


Summary Lists

Observed diameters

Displays a list of diameters of asteroids determined from asteroid occultations. The information provided is:

  • Asteroid number and name
  • Major axis and uncertainty
  • Minor axis and uncertainty
  • Observed PA of the major axis
  • Quality of the fit. 3 = Excellent; 2 = good; 1 = poor
  • Date of the event

The solution assumes that the shape of the asteroid can be approximated by a tri-axial ellipse. The derived diameters are not the axes of that ellipse, but of the projected outline of the ellipse.

The list can be sorted by a number of fields, to better display the information.

Double Star discoveries

Displays the details of all close double stars derived from asteroid occultations - most of which are new discoveries. The information provided is

  • Star identifier
  • Separation, and formal uncertainty, in milli-arcseconds
  • PA, and uncertainty
  • Date of the observation
  • Asteroid involved in the occultation

Derived asteroid positions

Displays a list of astrometric positions derived from asteroid occultations. The positions are given as offsets from the star. The information provided is:

  • Asteroid number and name
  • Time of geocentric closest approach, and uncertainty. Reporting precision is 0.01 seconds
  • Geocentric separation from the star at closest approach, and uncertainty. Reporting precision is 0.0001"
  • PA of the asteroid relative to the star at minimum separation. Reporting precision 0.001 deg
  • RA and Dec of the star, J2000, corrected for proper motion to the event date, and for parallax when known.
  • Source catalogue for the positions (Hip = Hipparcos, U2 = 2UCAC, Ty2 = Tycho2)
  • Number of occultation chords used to derive the position.

Events involving distant asteroids

Displays a list of occultations where the heliocentric distance of the asteroid at the time of the occultation was greater than 4 AU.

Generate Graphics files for Many events

Automatically generate a graphic of all occultations where there has been a fit of an ellipse to the observed chords.

Observations by an observer

Provides a utility to identify all events observed by an observer. Operates by specifying a search string (no wild cards), and identifying all events containing that string in the field that specifies the observer's name and location.


Maintain observations file

The set of functions given here are maintenance functions for the file of historical observations. Most users will have no need to use them

Sort observations file

New observations added to the historical file are added sequentially to the end of the file. This function sorts the historical file into date order.

List events on same day, by same asteroid

This function is used to identify duplicate entries in the historical file. To function, the file must have been sorted into date order. Note that there are several instances where an asteroid has occulted two different stars on the same day. Also there is one instance (as at 2007) of a double asteroid occulting a star. Potential duplicate entries are marked with an asterisk.

Re-index Observations file

Occult maintains an index of observations in the file. This function re-creates that index. In normal use, this function should not be required.

List poor star positions

This option generates a list of events where there might be a problem with the star position.

Run data validity check on Historical file

[For Administrator use] This option creates a file that contains all data lines from the Observations file where data elements fall outside the valid range for that item. Each line is preceded by a line that contains the event identification, and tags to identify the field(s) that is outside range. The file created is called AsteroidObservations.val, and is located in the Resource Files directory.


'Official' reports [For administrator use only]

This set of routines is used in association with reporting and archiving asteroid observations. They are NOT for general use.

Make short MPC report

Generates a report of astrometric positions for submission to the Minor Planet Center, together with a list of observer names. details

Add MPC details

Once the Minor Planet Center has published the observations (in the Minor Planet Circulars Supplement) and the observation summary (in the Minor Planet Circulars), this option is run to add the Minor Planet Circular number, and publication date, to the file of historical observations. Details

Important. Make sure you select the correct report for running this function - to avoid placing the MPC number and date against the wrong events. It might be appropriate to rename the file (with a leading '_', for example) after the update has been run - to prevent the file from being listed for selection.

eMail report

This provides functionality to email the 'short MPC report' to the Minor Planet Center in a plain-text format.

Make MPC report [new format]

The Minor Planet Center is slowly changing its systems to use a new report format. This option creates the MPC report in accordance with the 'new' format. This format is not yet operations at the MPC [Feb 2008]

NASA PDS Archive files

The historical file of observations is annually archived in NASA's Planetary Data System - small bodies node. This option creates the files needed for the archive. The files are created in the directory Occult 4\Asteroids\Results\PDS Files

The PDS system also archives the graphic representations of events. these can be generated using the Generate Graphics files for Many events option. The files are created in a user-specified directory.

The archive can accessed at PDS Occultation archive

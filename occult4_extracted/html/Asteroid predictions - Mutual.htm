Mutual asteroid transits and occultations

Overview

This form provides predictions of mutual asteroid events. That is, an occultation of one asteroid by another, or a transit of one asteroid in front of another.

These events are very rare. On average, about 1 per year involving asteroids brighter than mag 16.

Despite their rarity, it would seem that such events are observable. However before this functionality, predictions of such events have never been made by anyone.

There are two aspects to generating predictions of mutual events. The first is to identify a close approach (appulse) of two asteroids, such that a mutual event might occur. The second is to analyse that appulse, and to generate a prediction of the occultation path should it cross the Earth.

Occult does not undertake the process of identifying close appulses. That is beyond the scope of Occult. However lists of close appulses are generated by Brian Warner, and made available https://minorplanet.info/ The lists are generated on an annual basis, at around October each year. The lists are limited to asteroids brighter than mag 16.

The first year covered by the lists was 2020.

Occult uses these lists to identify possible mutual event candidates, and then undertake a full prediction. The great majority of events analysed do not give rise to a mutual event.

The factors affecting the occurrence of a mutual event are a little complex. One might think that the separation at appulse would be a good indication. However an appulse distance of just 1" (2020 Oct 17) did not result in a mutual event, while a separation of 13" (2022 July 22) did. A major factor is the distance between the asteroids as compared to the distance of the closer asteroid to the Earth.

For each event that is found, two files are generated. They are held in the Occult subdirectory \Predictions\. One file is a .dat file that contains the path prediction in text. The other is a GoogleEarth .kmz file, to display the path. The file names commence with Mutual_, followed by the year, and then the component identifiers. For example:

Mutual_2022_(23606)_1996_AS1_transits_(743)_Eugenisis.dat

There are two types of mutual events. One is where the closer asteroid appears larger than the more distant asteroid. In this case the closer asteroid occults the more distant one. Conversely when the closer asteroid appears smaller than the more distant one, the closer asteroid transits the more distant one. Note that it is the apparent size that matters, not their actual size.

A mutual event involves two stages. At the beginning and end there is a partial phase, during which either the occulted, or the transiting, asteroid are crossing the limb of the other asteroid. During this phase there will be a decline/rise in the overall light. When the transiting asteroid is fully in front of, or the occulted asteroid is fully behind, the other asteroid, the light level is constant - being the magnitude of the asteroid that has the larger apparent diameter. In the predictions, the Total duration covers the time from the start of the initial decline in light, to the end of the end of the rise in light. The Central duration refers to the period during which the light level is constant.

The data given in the prediction is:

Line 1. The date, asteroid identifiers (closer asteroid first) and type (either Transit or Occultation)

Line 2.

  • The event type - transit or occultation
  • The total magnitude of the two asteroids
  • The magnitude drop during the event
  • the Central duration of the event
  • The Total duration of the event
  • The widths of the Central path, and the Central + partial path, in km

Line 4. The coordinates of the two asteroids (which are the same!), with the elongation from the Sun and moon, and illumination of the moon

Line 5. For the closer asteroid

  • Number and name
  • Magnitude
  • Diameter - both in km, and the apparent diameter in mas
  • Distance from the Earth, in AU
  • The uncertainty ellipse, as given in JPL-Horizons.
  • The orbit identifier. The JPL-Horizons identifier of the orbit solution number, and date of that solution

Line 6. The same as for line 5, except it is for the more distance asteroid.

Line 7. The uncertainties in the prediction. Uncertainty in the event time, and the uncertainty in the path location - in the across-path direction. The distance is as projected onto the surface of the Earth

Thereafter there is a table of path coordinates. The table is very similar to the table of path coordinates for a usual asteroidal occultation. The table lists the coordinates of the central line, the UT time, Alt/Az  of the two asteroid, the coordinates of the full occultation or transit path, and the coordinates of the outer limit of the 'partial' phase of the event.

 

Generating the predictions

The form for generating the predictions looks like:

The process involves two steps:

  • Download an Appulse file
  • Generate the events

Download the Appulse file

The drop-box under Year lists all years from 2020 to the year after the present (that is, 'next year'.) The Next Year file will not be available until the latter part of the current year. To generate predictions for a year, select the relevant year, and download it. This will result in a file called Appulses_yyyy.dat in the \Downloaded Files subdirectory, with yyyy = the relevant year. This file contains all events listed in the monthly files downloaded from minorplanet.info.

When a file has been downloaded, it will be listed in the box headed Available Years.

Listing / Generating predictions

The first step is to set the maximum separation at appulse, that will be used to eliminate appulses that cannot result in a mutual event. The default value is 20" is strongly recommended, although values from 40" to 5" are provided.

When a year is selected with a single click, the List of Possible Events will display all appulses in that year with an appulse separation less than the value displayed. If you have previously generated a prediction for an event, it will be marked with a  ✔ . Otherwise a dash indicates no more than that a prediction file does not exist.

 When a year is double-clicked:

  • The List of possible events is populated with events that meet the Separation filter limit (with a - or ✔ at the end of the line, depending on whether a prediction has been previously generated)
  • Work through that list to generate predictions. For each event there will be either 4 or 6 downloads from JPL Horizons.
  • The result for individual events is marked with an X if there is no event, or a ✔ if there is an observable event.
  • Under the List of possible events box, there are two lines giving the status of each event as they are worked through. The top line is the event identification. The second line gives the minimum distance of the path from the Earth's center (in Earth radii), the total path width for that event (in km) and either 'No event', or 'Event'

When an event is found, its path will be briefly displayed. Also the path will be shown plotted in GoogleEarth. All events found in the list will be plotted together.

Viewing predictions

Any event in the  in the List of possible events that is marked with a ✔ can be viewed by simply clicking the relevant line.

Double-clicking on any line in the List of possible events will regenerate the prediction. The path coordinates remain displayed, and plotted on GoogleEarth.

The prediction files are held in the  \Predictions subdirectory of Occult. The file name is formatted as

Mutual_yyyymmdd_ ......

where yyyy, mm, and dd is the date of the event. The file name also gives the asteroid identifications and event type - as in:

Mutual_20220722_(23606)1996AS1_transits_(743)Eugenisis.dat

Two files are generated for each event. The .dat file is a text file giving the event details and path coordinates. The .kmz file provides the event path in GoogleEarth.

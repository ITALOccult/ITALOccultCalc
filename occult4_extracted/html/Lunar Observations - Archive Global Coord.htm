Consolidating regional observations

  • Under the directory \Occult 4\Lunar Archive\ create a working directory for all archive files received over a period. At convenient intervals cease using this directory and create a new directory for new archive files. Do not add files to these directories other than Archive files
  • When an archive file is received, place it in the working directory.
  • Periodically (as often as receipt of a new file) consolidate the files into a single Archive file. Do this from the Archive Editor, using the Process files from coordinators button. On clicking this button, select the currect working directory, and select ALL files. The routine will:
    • combine each of the archive files into a merged file (in this process, it will skip packed files with extensions of .zip, .rar, .arj, .lzh, .gz and .tar.)
    • create Log file.txt , a file listing all the files that have been merged
    • run the Auto correction routine over the merged file
    • sort the file chronologically
    • rename the file to Archive Observations recent.dat, ready for uploading
    • reduce the file, creating the file RArchive Observations recent.dat, ready for uploading
    • after confirmation, email a copy of the log file to each regional coordinator.

      two additional functions will be included in the future: Re-allocate graze identification numbers, and re-generate the Graze Index file. This can't be automated until the transition from ILOC is complete, with no significant backlog of old unreported observations.

  • It is possible that the AutoCorrection routine will make a correction that should not be made. This should be checked by opening the Archive Observations recent.dat file in the Archive editor — using the Edit Events button, with the Corrected events radio button checked. This will list all events that have been corrected. Select each in turn, and make sure the correction applied is appropriate. To undo a correction:
    • select the line,
    • click the Retrieve original data button,
    • in the large group of radio buttons at the right, select OK 'as is'
    • click Write correction to file button.
    Note that the correction is written directly to the file.
These two files need to be zipped into a file called latest_observations.zip, which is then uploaded to the Occult download site, replacing the file of the same name.

Creating the Archive for VizieR

The File archived at VizieR is a single file containing both the observations and reduced data. The file is created from the Archive Editor, Combine Regions page. The time taken to create the file will be of the order of hours... (and longer if the computer is slow...). The file that is created is based on the set of Archive Observations files contained in teh user's Resource Files directory. The generated file is also contained in that directory, and is aclled Vizier Archive.txt.

When generating this file, previous copies are renamed Vizier Archive.bup, Vizier Archive.bp2, and Vizier Archive.bp3.

The Vizier Archive file can also be used to generate the corrections to the table of deltaT values - to create a table that is consistent with the ephemeris being used in Occult. This is explained here. The deltaT corrections are only made for the period 1700 - 1960. To save time with this correction process, the Vizier Archive can be limited to events prior to 1961 by checking the relevant box.

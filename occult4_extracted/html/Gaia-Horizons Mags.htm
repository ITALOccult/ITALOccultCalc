Comparing Gaia measured asteroid magnitudes with Horizon's magnitudes

Displaying a possible profile using photometric data

This form is intended to assist with the interpretation of possible satellite detections, particularly when there is no shape model, nor rotational light curve data.

Great care must be taken when using the information provided on this form. There is a real question of whether the information provided is reliable for the purpose being suggested in this form. It is provided on the possibility it may prove useful in some circumstances

This form can only be accessed from the Asteroid Observations editor. The data required to generate the displayed data cannot be manually entered.

There are two components to this form:

1.  Retrieve Gaia magnitude measurements for an asteroid, and interpret them

The purpose of this capability is to investigate whether the Gaia photometry of asteroids provides any indication of the presence of absence of large variations in the asteroid's brightness, that might indicate whether the asteroid was highly elongate.

The process retrieves the Gaia magnitudes for the particular asteroid, and compares them to the asteroid magnitudes generated by Horizons for the Gaia date of observation. The data can then be copies in .csv format for pasting into a Spreadsheet such as Excel, where it can be plotted as a graphic, and visually assessed.

The assessment may not be straight forward. To avoid various issues that may affect the brightness of the asteroid, you need to look for groups of measurements that are fairly close in time, and assess the greatest difference in that group. Why? because if the time difference is within 10's of days (not 100's of days) the primary cause of light variation will be rotational variation, which is indicative of elongation of the asteroid.

The process is :

i. Open the form from the Asteroid Observations Editor, under List...  Asteroid light curve data...  Assess Gaia mag data; Assess photometric data

ii. Step 1.  Retrieve the Gaia magnitudes. Click the button for Step 1. This will open a form in your browser that displays a page at VizieR to undertake searches. The instrictions on what to do are provided in the box below. They are:

On the Vizier Form that appears in a separate browser window:
1. Find the option half-way down the form on the Left to set Max records, and select 'all'
2. In the large box immediately following:
    i. clear any and all text in the box
    ii. place the cursor in the box, and do a 'Paste' (that is, control + V) to set the search. [The text to be pasted has been automatically set.]
3. Under that box, click Quickview to confirm you have created a valid query
4. In the box 'Query name' give an identifier for your search, and set the 'Output format' to 'csv'
5. Click Run
6. When the "phase' indicates your search is COMPLETED, click 'download' at the right of that line
7. The file will be saved in your computer's default download directory, with a file name 'result (xx).csv'

iii. Step 2  Read the magnitude data in the file you have just downloaded. Click the Read downloaded file button, then find the file that has just been downloaded. It will be in the directory where you computer saves downloads as a matter of default.

That file will usually have multiple measurements on a date. When reading the file, the program will use only one entry per dates within 0.1 days of each other.

When the file is being read, the program will query Horizons to obtain its value for the asteroid's magnitude at that date/time.

The results are displayed in the large box, as Date, Gaia G magnitude, Horizons magnitude, and the difference.  The displayed data can be copied using the Copy list as text button below the box.

iv Step 3  Copy the data to use in a spreadsheet such as Excel.  Click the button Copy list in csv format, ready to Paste into Excel 

 

Step 4. Open the spreadsheet. Paste the data that has just been copied. It should automatically display as 4 columns. Those columns are:

  • JD of the Gaia measurement
  • Gaia magnitude
  • Horizons magnitude
  • Magnitude difference

Highlight the first and fourth columns, and generate a 'scatter' plot. This should plot the mag difference against the JD. From this plot, you can assess whether there is any evidence of significant rotational variation in the light level. REMEMBER. You must focus on points that are relatively close together in time.

 

2. Analyze certain photometric data

This option uses the data provided in

 Combined spin orientation and phase function of asteroids
  B. Carry1 , J. Peloton2 , R. Le Montagner2 , M. Mahlke3 , and J. Berthier4
  A&A, 687, A38 (2024)
 https://doi.org/10.1051/0004-6361/202449789
 
 The data can be downloaded from VizieR, at ftp://cdsarc.u-strasbg.fr/0/catj/J/A+A/687/A38/catalog.dat.gz . It is a 17 MB .gz file

Occult uses this data to compute 2-color magnitudes of asteroids covered by the data. [Two-color magnitudes for asteroids has previously been extremely limited.]

Included in the data set are values for the oblateness of an asteroid (assuming it is an oblate spheroid), and the RA/Dec of the axis or rotation of the asteroid. These values have been derived from a vast number of 'sparse' asteroid magnitudes extracted from large asteroid surveys. The sparse nature of the data leads creates uncertainties in their interpretation. Nevertheless, it provides data that is otherwise not available; it just requires significant care in its interpretation and assessing reliability.

A file containing the data is present in the Occult Resources file directory, as HGsHG1G2_Magnitudes.csv. Importantly, when creating this file, the data relating to the asteroid oblateness and axis orientation is limited to those asteroids where the uncertainty in the Green magnitude was less than 0.3 mags - to exclude oblateness measurements that were inherently unreliable. Thus of the 122,369 asteroids having color magnitude data, oblateness data is only included for 52,975 asteroids.

When this form is called from the Asteroid Observations editor, the data from this file is retrieved. If oblateness data is present, the form will display the derived shape and orientation of the asteroid at the event date. If the data is not present, that part of the form will not show.

Great care must be taken regarding the reliability of the asteroid shape & orientation.

Interpretations

1. The profile plot at the right of the form. Above the plot is the RA & Dec of the asteroid. Then the RA/Dec of the axis of rotation of the asteroid, and its oblateness of the asteroid. Finally, the position angle of the axis of rotation (the minor axis of the ellipse), and the latitude of the Earth as seen from the asteroid. The plotted ellipse takes into account the effect of this latitude value.

 This ellipse happens to coincide with one of two possible solutions/orientations for the main body in that event. This give a basis to select one possible fit over the other - although it is not a strong basis. It also gives a basis to adjust the axes of the fitted ellipse to have a ratio of 2.25:1

How reliable is this? Good question. Users should try this functionality with a range of other asteroids in the observations file to build a level of confidence in the reliability of this data

2. Magnitude data.

i. there are two clusters of points towards the right, around 2457600, and 2457700. They are all at the same level. This indicates (but does not prove) that there is little rotational light variation at that time (the system might have been 'face on'.

ii. There is a wide vertical scatter of the points near 2457200. This suggests either an elongate asteroid, or the presence of a satellite than is undergoing occultations/transits. However the magnitude difference is large at almost 0.5 mags, which corresponds to a 40% drop in light. This would suggest an elongate asteroid. Is this a reasonable possibility??? Or, given the observations being spread over about 2 months, are there other issues involved?

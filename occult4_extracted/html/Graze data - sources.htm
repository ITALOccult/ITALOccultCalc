Sources of the Graze data

Occult holds the observation records for over 3,000 grazes observed since 1706. The data for grazing occultations is held in two files. The first [OCCULTGrazes] contains the RGO graze data, and the ILOC grazes prior to 2002. The second [RecentGrazes] contains the graze data from the reports collected by Mitsuru Soma.

The historical graze data has been sourced as follows:

Grazes before 1981. From reports of grazes published by the Royal Greenwich Observatory. This data was not available in machine-readable form. Rather it has been converted form paper to machine-readable form by members of IOTA/ES (whose valuable work is hereby acknowledged). This data was subjected to a comprehensive data integrity checks to identify and remove a range of character identification and formatting errors that are typically with OCR'ing. While there remains a small possibility that the reported time in minutes and seconds includes OCR errors, the frequency of such errors is believed to be less than about 1 observation in 1000.

Once the RGO data was corrected, it was converted into report forms in the ILOC format as used by OCCULT. The data did not include all the data used in the ILOC reports, and used a different reporting scheme for some parameters. The transformations made in the conversion process were:

  • The telescope details were set to 'Other type, alt-azimuth mounting, manual drive, aperture=10cm, focal length 100cm.
  • Observer names were set as “Unknown ” followed by A, B, C……
  • Event types, from the value in column 25 of the RGO report, as 1=>DD, 2=>RD, 3=>DB, 4=>RB, 7=>B, 8=>F, 9=>M, 0=>?. Eclipse events (DU, RU) were identified when the value of column 44 was 8.
  • Method of time keeping, from the value in column 46, as 0=>“ ”, 1=>P, 2=>E, 3=>X, 4=>S, 5=>S, 6=>C, 7=>E, 8=>X, 9=>T
  • Time source, from the value in column 46, as 0=> “ ”, 1=> “ ”, 2=>R, 3=>C, 4=>R, 5=>T, 6=>M, 7=>C, 8=>C, 9=>R
  • Handling of personal equation, from the value in column 47, as 1=>N, 2=>S, 3=>E
  • Accuracy, from the value in column 43, as 0=>.3 1=>.2, 2=>.4, 3=>.6, 4=>.9, 5=>.9, 7=>.9, 8=>.9. [6 and 9 were not valid values]
  • Certainty, from the value in column 43, as 0=>1, 1=>1, 2=>1, 3=>2, 4=>2, 5=>3, 6=>2, 7=>2, 8=>2, 9=>3
  • Conditions, from the value in column 44, as:
    • Stability as 0=>1, 1=>1,2=>1, 3=>2, 4=>2, 5=>1, 6=>1, 7=>1, 8=>1, 9=>1
    • Transparency as 0=>1, 1=>1, 2=>2, 3=>3, 4=>3, 5=>1, 6=>1, 7=>1, 8=>1, 9=>1
    • Remarkable circumstances as 0=> “ ”, 1=>“ ”, 2=>5, 3=>6, 4=>7, 5=>1, 6=>2, 7=>3, 8=>“ ”, 9=>“ ”

Grazes for 1976 to 2001. From the files of ILOC. The data consisted of 3 files - observations, telescopes, and observers/recorders. This data was converted back to the ILOC report format. When these grazes were reduced by OCCULT, a number of keying errors in the ILOC data were identified - which have been reported to ILOC. [These errors occurred mainly in the site coordinates, and the star number.] There is very little overlap with the RGO grazes.

In addition there are several 10's of grazes where the observations do not appear to be in the ILOC files. The data for these grazes was extracted from files supplied by Mitsuru Soma.

Grazes for 2002 to 2005. From reports held by Mitsuru Soma. These were already in the standard ILOC format. For use in OCCULT, the century component of the year was added to the report. [NOTE. Since the observation files held in OCCULT extend over almost 4 centuries of observations, it has been essential to include the century year in the data format used in Occult.]

Grazes before 1978 : Van Flandern data from US Naval Observatory. Van Flandern of the US Naval Observatory collected graze observations up to 1977. Unfortunately some critical data files have been lost. The only extant file gives accurate times, but it does not provide accurate site co-ordinates (nor observer names or telescope details). Most significantly, the file only contains the site co-ordinates to a precision of 1 arc min. Also, the specified longitude includes an amount related to the earth's rotation over some period of time (apparently for reasons associated with the program for reducing the observations), and can differ from the actual site longitude by some 10's of arc minutes. However all these observations have been identified in the RGO files, and the site coordinates manually updated to the correct coordinates.

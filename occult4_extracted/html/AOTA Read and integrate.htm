AOTA: Tab 1. Read CSV file, Integration and Binning

Open a CSV file

Initiate from either the menu bar under:
    File... open CSV file,
or from the first tab (File open, Integration & Binning) using:
    Open a CSV file
Select the CSV file to read. The file must have the format used by Limovie or Tangra.

If the CSV file contains data for comparison stars, you will need to identify the star that is the occulted star. Limovie can measure the occulted star and 2 comparison stars, while Tangra can measure the occulted star and 3 comparison stars. Users typically specify the occulted star as the first star in both Limovie and Tangra. However the occulted star could be any of the other measured stars.

When the CSV file contains data for 2 or more stars, the following form will appear to allow you to specify which star is the occulted star. Note that the options for star 3 and star 4 are only enabled when those stars are present in the csv file. Select the star that is the occulted star, and click OK.

The data read can be sourced from .csv files created by the following measurement tools.


Limovie:

  • Foreground & background for the measured star, Foreground for up to two comparison stars, and the Limovie frame identification.

PyOTE:

  • Foreground less the background for the measured star and up to 3 comparison stars, and the PyOTE frame identification

R-OTE

  • superseded by PyOTE, it is limited to one comparison star.

Tangra unbinned measurements:

  • Foreground less the background as well as the background for the first star, foreground less the background for up to three comparison stars, and the Tangra frame identification.
Tangra Binned measurements:
  • data for the measured star and up to three comparison stars, and the Tangra frame ID.
Scanalyzer Essential: using the ...._Export.csv file
  • data for the measured star, a pixel number, and the associated UTC. No comparison stars. Background set using 5 readings before and after the start/end of the light curve.
Scanalyzer is used when the process for recording an occultation is to use a stationary telescope, with the stars being recorded for a period of time as they drift across the field. Image measurement tool is Maxim-DL. Scanalyzer is used to measure the data generated by Maxim. The scanalyzer software and instructions is available at: Drift Scan

Unistellar (Electronic):

  • Star less background. No comparison stars
The CSV data as read is maintained in arrays, and remains unchanged by any processing.

Integration, Binning, expected Mag drop

  • This step occurs automatically whenever a CSV file is read.
  • This step can be called at any stage during subsequent analysis - by going to the first tab and clicking the Integration/Binning details button.
When searching a light curve for an event, it can assist if the expected magnitude drop is set before any real analysis occurs. This is provided for in this form with the blue box 2. Expected Mag drop. A default value of 1.0 is set. Changes to the binning and normalisation occur without having to re-read the original data. However all subsequent processing steps will need to be re-done.
  • For a signal from an integrating camera, this allows the first frame of an integration block to be specified, and for setting the number of frames in an integration block
  • Can be used with a non-integrated signal, to bin the observed frames - again specifying start, plus number of frames to bin.

  • The plot displays the first section of the CSV file. The number of points displayed can be varied between 50 and 4000 using the 'Display xx data points' item at the bottom left of the form. ·
  • The set of data points that are used for the first binned data point is indicated with a yellow rectangle. There are two ways of setting the location of the rectangle:
    • Set the left side using the cursor and a Left mouse click; set the width using the cursor and a Right mouse click; or
    • Use the controls at the bottom of the form to set the First frame, and the Number of frames.
    For no integration/no binning, use a rectangle aligned with a single frame (that is, the Number of Frames set to 1). For a signal from an integrating camera, match the rectangle to a set of data points corresponding to a single block of identical output frames. This will set the frame ID to the first frame used in the integration block.
  • The plot has faint vertical lines repeating at the integration interval - so that the correct positioning of the integration can be assessed along the light curve. {When the integration interval is correctly set, significant changes in the height of the signal should correspond to the location of the vertical lines - with the signal being largely constant in-between.}

Normalisation

Variable atmospheric transparency (such as light clouds) can have the effect of varying the light levels along the light curve. Such variations are usually present in all the stars in the field of view. Details on Normalisation are given here.

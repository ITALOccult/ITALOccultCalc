Verify event times

This functionality is available when AOTA is accessed from a direct call from Tangra.

When a video suffers from irregularities - such as dropped anr duplicated frames - it is desirable to confirm the correctness of the time associated with the frame at the time of the event.

When Verify is called, the form that is displayed is like:

The form is specific to the event being analysed. It contains two regions - one for the D event, and one for the R event. To verify the times, click the Display frame button for the relevant event. This will display the corresponding video frame in Tangra. Below the Display Frame button, the time AOTA believes corresponds to this frame is displayed.

Below that is an 'up/down' control to apply a correction to the displayed AOTA time. Below that the time that should be displayed on the video frame is displayed. If that does not correspond to the time stamp on the video frame, adjust the value of the correction until it does correspond.

Check the times for both the D and R event. Each event is treated separately - as dropped or duplicated frames can occur anywhere.

Once done, exit the form using Exit with Corrections to have the corrections applied to those two events. Click Cancel to leave with any previous corrections for those particular events remaining unchanged. To remove previously-determined corrections, you need to set the corrections to zero.

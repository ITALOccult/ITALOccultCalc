Occelmnt files

These files maintain the occultation elements for occultations by asteroids and planets. Two formats are supported. An XML format that allows for variable field length, and a fixed field format. The fixed field format will eventually be removed, as certain data elements are not supported in this format.

XML format

The structure of the xml file is:

<Occultations>
  <Event>
    <Elements> source, duration, year, month, day, UT at closest, x,y,dX, dY, d2X, d2Y, d3X, d3Y</Elements>
    <Earth> SubstellarLong, SubstellarLat, SubsolarLong, SubsolarLat,JWST</Earth>
    <Star> Identifier, RA, Dec, Mb, Mv, Mr, dia, double star code, K2 flag, Apparent RA, Apparent Dec,MagDropsAdjusted_NearbyStars,BrightNearbyCount, TotalNearbyCount</Star>
    <Object> number, name, magnitude, diameter, distance, #rings, #moons, dRA, dDec, Taxonomy, DiameterUncertainty, PlanetMoonInPlanetShadow, MagV_Asteroid, MagR_Asteroid</Object>
    <Moons>
        <Moon>Name, diameter, distance, PA</Moon>
         ...... iterated for all moons around the asteroid
    </Moons>
    <Orbit>equinox, MA, yr, month, day, peri, node, i, e, a, q, H0, Coeff LogR, G</Orbit>
    <PlanetPhysical> [Planet ID, Moon ID,] % illumination, PAlimb, phase angle, planetocentric dec of Earth, PA of planet's pole</PlanetPhysical>
    <Errors>in Pathwidths, major axis, minor axis, PA, 1 sigma in star/asteroid posn</Errors>
    <ID>ID</ID>
  </Event>
</Occultations>

The tag <Occultations> delimits the file

The tag <Event> delimits the data for an event

Within an event, the tags <Moons>, <Orbit> and <PlanetPhysical> will only be present when appropriate

Within a data element, individual items are stored as comma-separated-values

Tag

Data items
Elements

The Besselian elements of the occultation event

  • Source - of orbit for the object, and date the prediction was generated
  • Duration - maximum, in secs
  • year - of event
  • month - of event
  • day - of event
  • UT of closest approach - to centre of the Earth
  • X, Y - the X and Y coordinates at the time of closest approach (Earth radii)
  • dX, dY - the rate of change in X and Y at the time of closest approach (Earth radii per hour)
  • d2X, d2Y- the 2nd order rate of change in X and Y at the time of closest approach
  • d3X, d3Y- the 3rd order rate of change in X and Y at the time of closest approach
Earth

Information about the Earth's orientation

  • Substellar Longitude, at UT of closest approach (deg)
  • Substellar Latitude (of date) at UT of closest approach  (deg)
  • Subsolar Longitude at UT of closest approach  (deg)
  • Subsolar Latitude at UT of closest approach  (deg)
  • For JWST. 1=true, 0=false. If true, prediction is for the location of the James Webb Space Telescope.
Star

Information about the star

  • Identifier - a star catalogue identifier
  • RA - Right Ascension, BCRS position (that is, ICRS reference frame 2000, epoch of event, no parallax applied), decimal hours
  • Dec - Declination - BCRS position, decimal degrees
  • Mb - Blue magnitude
  • Mv - Visual magnitude
  • Mr - Red magnitude
  • dia - stellar diameter in mas
  • double star code.  0 = none, 1 = WDS, 2 = double in other sources (4th Interferometric, or star catalogue flag), 4 = Variable star (listed in AAVSO Index file). Values are cumulative
  • K2 flag. K if it is a K2 star, blank otherwise
  • RA - Apparent Right Ascension of date
  • Dec - Apparent declination of date
  • MdropV - Magnitude drop in Mv at occultation
  • MdropR - Magnitude drop in Mr at occultation
  • Magnitude drops adjusted for nearby stars. 0=false, 1 = true. Has regard to all stars within 4"
  • Number of bright nearby stars. Stars within 15" of the target star which are more than 2.0 magnitudes brighter. Set to -1 if a check for nearby stars has not occurred.
  • Number of all nearby stars. Stars within 15" which are brighter than [target star mag +1.5]. Set to -1 if a check for nearby stars has not occurred.
Object

Information about the occulting object

  • number - for asteroid, the sequential asteroid number. For the major planets of their moons, this field has the form PxMyy, where x is the planet number, and y is the moon number
  • name
  • magnitude
  • diameter - in km - augmented by the star's diameter. The asteroid's actual diameter is:
      asteroidDiameterAugmentedByStarDia - StellarDiameter_mas / 3,600,000 / Radian * AsteroidDistance * 1.496e8;
  • distance - in AU
  • #rings - the number of rings around an asteroid [not set for planets]
  • moons - the number of moons around an asteroid [not set for planets]
  • dRA - hourly rate of change in RA - s/hr (used only in header of event plot)
  • dDec - hourly rate of change in Dec- "/hr (used only in header of event plot)
  • Taxonomic class.  Details of possible values are given here
  • Diameter uncertainty - in km
  • Planet Moon in Planet shadow. 0 = false, 1 = true. Set for the main moons of Jupiter and Saturn, for when the occultation event occurs while the satellite is eclipsed, but not occulted by, or transitting, the planet. [Events when the satellite is behind or in front of the planet are automatically excluded.]
  • MagV_Asteroid. The V magnitude of an asteroid computed using either the HG, or sHG1G2 systems using values for the g and r bands
  • MagR_Asteroid. The R magnitude of an asteroid computed using either the HG, or sHG1G2 systems using values for the g and r bands
Moons

Information about asteroidal moons

This group contains a series of tags <Moon>   </Moon> for each of the known moons of the asteroid. This entry/group is only given if moons are present. The reference plane for the orbital elements is the J2000 ecliptic. The data in each tag line is:
  • Name of the moon
  • diameter of the moon, in km
  • semimajor axis of the moon's orbit (in km). For a large number of moons, this distance is based on single-epoch observations, and is not equivalent to the radius of the orbit
  • orbital period (days)
  • eccentricity of the orbit
  • inclination of the orbit
  • epoch (JD)
  • Mean anomaly at epoch
  • longitude of pericenter
  • longitude of ascending node
Orbit

Information about the orbit of an asteroid

Note - the orbital elements given here are low-precision, used to plot the motion of the object on star charts.
  • Equinox of the orbit
  • MA - the Mean Anomaly at the epoch
  • yr - epoch year
  • month - epoch month
  • day - epoch day
  • perihelion - longitude of (deg)
  • node - longitude of (deg)
  • i - longitude of (deg)
  • e - orbital eccentricity
  • a - semimajor axis (AU)
  • q - perihelion distance (AU)
  • H0 - magnitude constant H0
  • Coeff LogR - coefficient of Log (R) in standard magnitude formula (used for comets)
  • G - slope coefficient in standard magnitude formula for asteroids.
PlanetPhysical

Information about a Planet (not present for an asteroid or moon)

  • % illumination - the percent illumination of the planetary disk
  • PA limb - the PA of the mid-point of the planet's bright limb
  • phase angle - the phase angle of the planet
  • Planetocentric declination of Earth - the latitude of the earth as seen from the planet. Used to correct contact times for the apparent oblateness of planets
  • PA of planet's pole - used to correct contact times for the apparent oblateness of planets
Errors

Information about prediction uncertainties

  • in Path widths - uncertainty of path location expressed as a fraction of the path width
  • major axis - the major axis of an uncertainty ellipse, in arcsec
  • minor axis- the minor axis of an uncertainty ellipse, in arcsec
  • PA - the PA of the major axis of the uncertainty ellipse
  • 1 sigma in star/asteroid position - in arcsec
  • Error basis - string that sets out the basis of the error value. Typical strings are
    * Star+Assumed (error for asteroid/planet assumed)
    * Star+PEU (Star uncertainty together with Peak Ephemeris Uncertainty for asteriod)
    * Known errors (errors for star and asteroid known)
    * Assumed (the total uncertainty is an assumed value)
  • Reliability value (Usually the RUWE. Other values that can occur are
      -1 = not set
      -2 = Hip star position definitely unreliable, from UBSC
      -3 = Hip star is in UBSC)
      -4 = Position is from Hip2 catalogue; because of limitations with matching to Gaia, the star may be a low-precision duplicate of another star in the Gaia catalog. 
  • Duplicate Source (0 = no, 1 = yes, -1 = not set)
  • Non-GAIA proper motion (0 = no, 1 = yes, -1 = not set)
  • Proper motion added to Gaia using UCAC4 position at 2000. (0 = No, 1= yes, -1 = not set
ID

Identifiers

  • A unique identifier for the event, used to eliminate duplicates when displaying. For asteroids, the ID is formed as yyyymmdd_xxxxxx, made up as the date of the event, plus end section of star ID. For planets, the ID is formed as yyyymmddPPPnnnn, made up as the date of the event, the first 3 characters of the planet or moon name, and the last 4 characters of the star identifier. Where the star is a TYC star, the third number group is ignored for this purpose.
  • The Modified Julian Date of the date of the prediction calculation.

 

Fixed field format

This format is now (2021) only used for OccultWatcher feeds.

Each entry involves 10 lines - plus an empty line at the end. The length of an entry (including the CR LF sequence at the end of each line) is 540 bytes. The data on each line is as follows, in ASCII:

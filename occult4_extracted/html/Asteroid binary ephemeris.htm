Binary asteroid ephemerides
- from IMCCE Miriade

Several hundred asterods are known to have satellites. However only a relatively small number of these binary asteroids have observations sufficient to establish a reliable orbit.
[Furthermore, as the gravitational binding between the asteroid and its satellites is relatively weak, the orbit is more susceptible to the gravitational influence of the sun and major planets]

The ephemerides of several binary asteroids have been provided on the Miriade server of IMCCE [Paris Observatory]. Theses orbits are performed by Frederic Vachier and Jerome Berthier, using the Genoid System. (F. Vachier et al. , Determination of binary asteroid orbits with a genetic-based algorithm, 2012, Astronomy & Astrophysic, Volume 543, id.A68, 07/2012) - who may be contacted at genoide@imcce.fr

Occult can obtain ephemerides of these satellites from the Miriade server. The form for retrieving the ephemerides can be accessed from two different locations, with slightly different functionality.
Called fromFunctionality
Asteroid search formA button is provided to transfer the ephemeris data to the search form - for each generated ephemeris. Also the ephemneris can be written to a 'control file' for use with asteroid predictions
Ephemerides tab, Asteroid satellitesTransfer to the Search form is not possible, but the data can be written to a control file.
In both cases an ephemeris for up to 99 dates can be generated. However only the first three points can be transferred to the Search form, or Saved in a Control file.

The form for obtaining the Miriade ephemeris is:

The Mirade system provides two lists of binary asteroid orbits. One list includes the 'best' orbits. The other list includes all orbits, including multiple entries for single satellites. Specify the number of the relevant asteroid at the top of the form, and the check boxes for that asteroid will be checked in both lists. You can select or deselect orbits using the check boxes.

The form has the following parameters which need to be set:

  • asteroid number;
  • the mass ratio of the satellite (or primary) compared to ther total mass of the system. The data in this field is transfered for this purpose to the Search form, or saved in the Control file.
  • the initial date; that is, the specified year, month, day and hour. [NOTE: the initial date requested from the Miriade server is actually 1 step interval prior to the specified year/month/day/hour - so that the 3-point ephemeris for use with asteroidal occultation predictions is centered on the specified time of the occultation.]
  • the step interval in the ephemeris - as specified on the form
  • the number of dates for the output. This is fixed at 3 for asteroid searches, or the # dates as specified on the form.
The name and diameter of the satellite, and the mass ratio, are transferred to the Search form, or saved in the Control file when either the 'Transfer 1-3...' or Save 1-3..' buttons are clicked.

Up to 12 orbits can be displayed. The data displayed are:

  • The asteroid identifier as given in Miriade:
  • for each date line in the ephemeris
    • the ephemeris date as given in Miriade.The format is yyyy-mm-ddThh:mm
    • the offset of the satellite in RA - in arc secs (as given in Miriade)
    • the offset of the satellite in Dec - in arc secs (as given in Miriade)
    • the PA of the satellite; and
    • the separation of the satellite - in mas.
  • The position of the satellite relative to the moon is plotted to the right of each ephemeris block. The first position of the satelllite is plotted in Red, with the remainder being plotted in Blue.The number at the bottom of each plot gives the width of the plot, in arcsec. The directions of North and East are indicated. You can enlarge the plot size by changing the form size.
    [Note: the configuration of the asteroid + satellite as projected on the Earth's surface is a left/right mirror image of the sky plot.]

When called from the Asteroid Search form:

  • set the Year, Month, Day and Hour to be close to the time of the occultation;
  • set the time interval to be 2 hours (unless the satellite motion is very large over a 4-hour time span - in which case a smaller interval may be used. HOWEVER avoid using a time interval of less than 1 hour.
  • The Miriade system generally contains more than one ephemeris for a given satellite. Occult will display up to 12 ephemerides. The most reliable ephemerides are those in the list of 'Best orbits'. Generally speaking, when there is more than one orbit for a particular satellite, the first listed is expected to be the most reliable.
  • For some asteroids, ephemerides are available for more than one satellite. You will need to look closely at the size and motion of the orbits to associate an orbit with a particular satellite.

    Using the results

    The results of a Miriade query can be used in two ways:
    • when the query is called from the Search form for asteroidal occultations, clicking the Transfer 1-3 to Search button will transfer the data from the relevant ephemeris to the search form, ready for a search to be conducted.
    • in all instances, clicking the Save 1-3 to Control file button will save the data from the relevant ephemeris in a tagged file ready for use as the '-A' control file for external use .
    In both instances you need to set a value for the mass ratio, as this will set the offset from the mean position on which the normal asteroid ephemeris has been generated. If the moon is small relative to the primary, setting a mass ratio of 0.05 will effectively provide an offset of the satellite from the ephemeris. However when the satellite is a significant fraction of the size of the primary, you will need to allow for an offset of both the primary and the satellite. If the mass ratio was (for example) 0.4, set 0.4 on the Search form for the satellite and generate the prediction. Then set the mass ratio to 0.6 [= 1-0.4] to generate the prediction for the primary. Note that when the mass ratio is set to 0.5 or greater the background color will change from light-green to pale-violet-red;.

    Note that in both cases only the first three data items are transferred or saved. However the ephemeris can be generated for up to 99 points - so that the orbit can be visualised.

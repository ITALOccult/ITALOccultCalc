VSOP87a
Mercury.bin, Venus.bin, Earth.bin, Mars.bin, Jupiter.bin, Saturn.bin, Uranus.bin, Neptune.bin, Pluto.bin

Occult includes the VSOP87A series as a default basis for computing planetary ephemerides - of Sun, and Mercury to Neptune. VSOP87 is a fit to the JPL-DE200 ephemeris.

The files Mercury.bin to Neptune.bin contain coefficients for computing the ecliptic X,Y,Z co-ordinates of the planets according to the VSOP expressions. The series has been truncated to terms with coefficients >5e-8 AU in any of the X,Y,Z co-ordinates - except for the Earth/Sun. The record length in each file is 26, with the following structure:

BytesData
0Co-ordinate: 1 = X, 2 = Y, 3 = Z
1P (0, 1, 2, 3, 4, or 5)
2 - 9A, stored as double precision
10 - 17B, stored as double precision
18 - 26C, stored as double precision

The co-ordinates are computed for a given Julian date JD as follows:

t=(jd-2451545)/365250
term= A * sin(B + C*t) * tP

The co-ordinates are computed by summing the terms for each co-ordinate. The equatorial XYZ co-ordinates are obtained through a rotation to the equator, and a small equinox correction. The conversion made is:

Xe = X + 0.00000044036 * Y - 0.000000190919 * Z
Ye = -0.000000479966 * X + 0.917482137087 * Y - 0.397776982902 * Z
Ze = 0.397776982902 * Y + 0.917482137087 * Z

PLUTO

The file Pluto.bin contains a series by Chapront that represents the position of Pluto over the period 1700 - 2100 to 0.03” (compared to DE200). The file is a binary file, with data stored as follows:

  • 106 binary records with the following field definitions:
      FQ As Double, cX As Double, sX As Double, cY As Double, sY As Double, cZ As Double, sZ As Double

  • 3 binary records with field definitions:
      a0 As Double, a1 As Double, a2 As Double, a3 As Double

    The co-ordinates for a given date are computed with:

      Tp = 2 * (jd - 2341972.5) / 146120# - 1 (Tp ranges from -1 to +1)
      Fx = Tp * 73060
      Arg = FQ*Fx

    To obtain the X coord, sum the following values:

  • cX*cos(Arg) + sX*sin(Arg) [for records 1 to 82]
  • Tp*(cX*cos(Arg) + sX*sin(Arg)) [for records 83 to 101]
  • Tp*Tp*(cX*cos(Arg) + sX*sin(Arg)) [for records 102 to 106]
  • a0 + a1 * Tp + a2 * Tp * Tp + a3 * Tp * Tp * Tp [using the first record for a)

    • Y is obtained using cY, sY, and the second record for 'a'; Z using cZ and sZ and the third record for 'a'

    The following is from the text file accompanying the series:

    • The tables of Pluto have been constructed by J. Chapront (BDL) with a new method of approximation using frequency analysis as described in the paper :
      Representation of planetary ephemerides by frequency analysis. Application to the five outer planets. Astron. & Astrophys.Suppl. Ser., 109, 191 (1995).


    • This representation uses the result of numerical integration DE200 of Jet Propulsion Laboratory as a source :
      Standish E. M., 1990, The observational basis for JPL'DE200, the planetary ephemerides of the Astronomical Almanac. Astron. Astrophys., 233, 252.


    • The interval of validity is 146120 days.
      Start : Jan 01 1700 0h JD2341972.5
      End : Jan 24 2100 0h JD2488092.5


    • The reference frame is defined with dynamical equinox and equator J2000 (DE200).


    • The numerical accuracy can be evaluated with the largest discrepancy obtained in a comparison with the ephemerides DE200 on the period 1700-2100 : for the positions : 5 E-7 au (0.003")

Data verification - Lunar observations

The File... Conduct 5 data validity checks function does a comprehensive check of data in the report against permitted ranges of data. The checks undertaken are detailed following.

The last validity check [Verify report content] is is performed whenever a file is opened, imported or pasted - with results only being displayed if problems are identified.

1. Unidentified stars

This check goes through the reprt looking for unidentified stars - indicated by a number of zero, or a caltalogue that is not ' ', X, S, P or A. As it goes through the list, if an unidentified star is located the form for identifying a star is opened. You can escape this check by clicking Abort on the identification form. Details

2. List possible doubles

This merely lists those events where the coding indicates a possible double star, giving the event, and any WDS code that has been allocated in the report. This listing provides a prompt to review whether a WDS component code should be added.

3. Verify Accuracy values

This checks events for Accuracy values greater than 1.0 seconds - and for any such events asks if the value should be divided by 10. [The need for this code comes from old ILOC report forms, where an incorrectly positioned value could give an Accuracy 10-times larger than intended.]

4. Verify Temperature

This checks events for Temperatures greater than 60 deg C - and for any such events asks if 100 deg should be subtracted from the temperature. [The need for this code comes from old ILOC report forms, where temperatures less than -9 could not be entered. Some observers added 100 deg to the temperature in these circumstances.]

5. Verify Report Content

This checks a range of data items in the report for consistency with the value ranges. These checks are only needed when the report has been prepared outside of Occult. [If the data is entered in Occult, these data checks are part of the data entry process]

    Report structure

    The data check occurs in the following order:
  • Sites
  • Events

    All errors found on a report line are presented in a single line. The first part of the line indicates the relevant item in the report in the following manner:

  • Tel X = Site X
  • Line nn = event line nn

    The errors on a line are presented in a comma-separated list. There is a short text identifier to indicate the relevant field, followed by brackets that contain the incorrect value. Note that the data check occurs in the order of the data in the line being checked (in the Iota 2008 format).

    IMPORTANT. Where a data item is displayed in a drop-down box, and the data item has an invalid value – when the relevant line is selected the drop-down box will display a default value. You should ensure that the drop-down box is set to the correct value for the observation.

    Items checked - Verify report Content

    Telescope line

    • telescope code is one of " ", R,, N, C and O
    • mount code is one of " ", E and A
    • drive code is noe of " ", D and M
    • aperture less than zero, or greater than 190 cm
    • focal length less than zero, or greater than 800 cm

    • Longitude - deg in range -180 to +180, minutes 0 to 59, seconds 0 to 59.99
    • Latitude - deg in range -180 to +180, minutes 0 to 59, seconds 0 to 59.99
    • Horizontal datum code <1
    • Altitude in range -200 to +5000m
    • Vertical date code - either E or M

    Event - date and type

    • Year in range 1623 to 2100
    • Month in rang 1 to 12
    • Day in range 1 to 31
    • hour in range 0 to 23
    • Minute in range 0 to 59
    • Second in range 0 to 59.999
    • The event type is one of D, R, B, F, M, S, E, O
    • The limb code is one of D, B, U
    • The graze code is one of " " and G

    Event - star number

    • catalogue is one of R, S, X, A, P, U
    • if catalogue is R, star number in range 0 to 3539
    • if catalogue is S, star is in the range 57982 to 210562
    • if catalogue is X, star is in the range 0 to 244437
    • if a WDS code is specifed, it is an alpha character in the ranges A - Z, a - z

    Timing Methods, Circumstances

    • PE less than 0 or > 2 secs
    • PE application code - one of S, A, B, U, E, X
    • Method of Recording 1 - one of " " G, V, M, S, T, E, P, K, X, C
    • Method of Recording 2 - one of " " G, V, M, S, T, E, P, K, X, C, A
    • if Method of Recording 1 is blank, and Method of Recording 2 is specified.
    • Method of Timing - one of " ", G, R, N, C, T, M, O
    • Accuracy - in range 0 to 2.0 secs
    • Certainty - 1, 2 or 3
    • Signal-to-Noise - in range 0 to 9.9
    • Double star component - one of " ", W, E, N, S, B, F, U
    • Duration - range 0 to 2.0 secs
    • Light level - one of " ", T, F
    • Stability - 0, 1, 2 or 3
    • Transparency - 0, 1, 2 or 3
    • Circumstances - integer in range 0 to 9
    • Temperature - in range -30 to + 50 C.

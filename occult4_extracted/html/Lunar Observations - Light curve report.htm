Light curve report - General issues

Reporting light curves

There are two modes for reporting light curves.
  • Report multiple light curves on a periodic basis. This is the preferred way of reporting the light curves. The user generates and saves the light curves. Then periodically the user selects those light curves and emails them to the collecting point.
  • Report light curves on an individual basis. That is, having generated the light curve, the user saves and emails the light curve at the same time.
The default location for storing light curves is the {Occult 4}/Observations/LightCurves directory. [The user can save the light curves in a different directory - however the automated process for emailing multiple light curves will not function properly.] When the user has successfully emailed the light curves, they are moved to the directory {Occult 4}/Observations/LightCurves/Reported.

The frequency of automated reporting of light curves is set in the User Settings, 2nd group (Check for availability of updates...) under Check for light curves to upload:. Available options are Never, Daily, Weekly, Monthly, or 3-Monthly - with the default setting being monthly. Those time frames are set on the basis of the date of the last upload of multiple light curves. A 'Save and upload' of a single light curve does not re-set the date for the automated uploads.

The automated upload conducts a check at the start-up of Occult. Provided the user has not set the frequency to 'Never', Occult will check for whether there are any light curve files that have not been uploaded - and if so, whether the specified time frame has passed. If so, the user will get a message asking if they want to upload light curve files. Select yes, and a dialogue box will appear to select the files to upload. Generally, select all files - then click OK. This will result in those files being emailed - and if that is successful, the files will be moved into the Reported subdirectory, and the date of the last upload will be set to today's date. If you cancel, or the email is unsuccessful, the files will not be moved, and the last upload date will not be updated.

Email addresses for reporting

The email address for reporting light curves has been added to the file of Addresses used in Occult. The address used is a temporary address, for use while the functionality is being established. If your file of addresses has not been updated to include the required address, you will get a message to that effect when you try to upload a light curve. In this situation, go to the downloads page and download the Reporting addresses for occultations (which is the 10th download on the form) - irrespective of whether or not it is tagged for download.

You can specify multiple cc email addresses - so that you can copy your reports to colleagues. From the report form, click Set CC addresses, or select the menu item File... set CC mail addresses for Light Curves. This will open a box where you can enter multiple email addresses. Points to note when entering email addresses are:

  • separate email addresses with a semicolon
  • do not force any line-breaks, or try to place addresses on separate lines. You must let the box line-wrap in its own way;
  • to indicate the message recipient, an email address also may have an associated display name for the recipient, which is followed by the address specification surrounded by angled brackets, for example: John Smith <john.smith@example.org>
  • The list of addresses is saved in the system user settings, by clicking the Save button. [Occult retrieves the addresses from the system user settings when an email is sent.] If you click the close button,or the Exit button, and there are changes that have not been saved, you will get a warning. If you make changes that are in error, and want to return to the saved version, click the Re-load button. However once you click the Save button, the previous version is lost.

When you send a Light Curve Email, the currently saved list of addresses will be added as CC addresses. That is, the addresses are those that exist at the time the email is sent, not when the light curve file was created.

Processing the light curve

Data for light curves has a wide range of variability in value ranges and length of data. The light Curve reporting functionality conducts the following processes:
  • The user can (and should!) bin the data to match any integration periods if the measurements have come from an integrating video camera;

  • When displayed, the user selects the section of the light curve to be archived. This is done by clicking on the light curve plot. If the Start radio button below the plot is checked, the start point will be set to the locaton of the mouse click, and the End radio button will now be checked. Clicking while the End radio button is checked will set the end of the region, and set the Start radio button.

    The portion of the light curve that is currently selected is displayed with a light aqua background.

    For a lunar occultation, the user should aim at setting the light curve to about 1 second before the event to 1 second after. Where a companion start is observed, that period should include that companion to a similar extent. Where a companion star was expected but not seen, the light curve should include the time interval during which the companion event was expected.

  • The height values of the light curve are scaled whenever the selected region is changed. The scaling is based on the values within the selected region. The result of this scaling is that all light curves have approximately the same height range. The maximum value of a light curve will always be about 9500. The minimum value should never be less than zero. However negative values do not indicate invalidity as such; rather it is an indication of poor observing conditions (usually strong background) such that the background subtraction is less than ideal.

Light curve report - Lunar occultations

Summary

  • Select the relevant event in the Observations Editor
  • Open the Light Curve report from the Observations Editor menu
  • Paste the light curve measurements from LiMovie
  • Select the start and end points of the light curve
  • Save the report.
  • Periodically, email multiple reports.
 The reporting of a light curve for a lunar occultation is accessed from the observations editor. The report can be of any event that has been entered into the editor. Typically, you would create the report at the time of entering a new observation (when you will usually have LiMovie open and displaying the event.) However you can report an old observation by selecting the relevant observation in the current file of observations, or a previous file of observations. In all instances, the only data you will need to manually enter/adjust is the light curve.

To initiate the Light Curve report, select the relevant event in the Observations Editor. Then click the menu item Light curve report.... Create light curve report.. This will open the Light Curve report as a lunar occultation, and prefill the report with a range of details.

Having measured the video in Limovie, there are two ways of entering the light curve into the Light Curve report:

  • In Limovie, click the button Copy CSV. This button is new to version 0.9.98.2 of Limovie, and is located between the Open AVI button and the Exit button. This will copy the measurements to the Clipboard. Having done this, in the Light Curve report select the menu item File... Paste CSV data set. [You can download version 0.9.98.2 of Limovie from here.]
  • In Limovie, click the button Save to CSV file. Having done this, in the Light Curve report select the menu item File... Open CSV file. Locate the file just saved, and open it.

If you have measured the video with Tangra, export the measurements to a csv file. Then open that file using the File... Open CSV file menu item. Similarly if you have had the light curve processed in R-OTE.

At this point, you will be taken through three standard AOTA screens which allow the time scale to be set, checks for duplicated or skipped frames, and allows for binning (which would be most unusual for a lunar occultation).

The Light Curve report displays a plot of the light curve. The region highlighted in Aqua is the part of the light curve that will be reported. For a lunar occultation, aim at reporting from 1.0 secs before the start of the event to 1.0 secs after the end of the event - allowing for the effecst of non-instantaneous transitions, and step events from double (or multiple) stars.

Once satisfied with the selection, the usual approach would be to Save the report - with a bulk-emailing of saved reports occurring at a later date. [A typical report will have a size of less than 2KB.] To save the report, simply click the button Save this report, or select the menu item File...Save _this_ report. .

However if you want to send the report immediately, click the button Save & Email this report, or select the menu item File...Save & Email _this_ report . This will save the report, email the report - and if the email is successful, move the report to the Reported subdirectory. The date for checking whether there are other reports waiting to be emailed is not affected when a single report is sent immediately).

Light curve report - Asteroidal occultations

Summary

  • Open the Light Curve report from the Asteroid Observations page in Occult
  • Paste, or read a csv file containing, the light curve measurements from LiMovie, Tangra, or R-OTE
  • Manually specify event and observer details (date, location, observer, and star)
  • Select the start and end points of the light curve
  • Save the report.
  • Periodically, email multiple reports.
 The reporting of a light curve for an asteroidal occultation is accessed from the main menu for asteroidal occultation observations. Unlike lunar occultations, the user will need to manually enter details of the date, asteroid, star, site location and observer. There are no restrictions on events that can be entered.

The Light Curve data

The first step on opening the Light curve report will frequently be to enter the light curve. The required data is the csv data generated by either Limovie, Tangra, or R-OTE. If the measurement data has been saved, simply read the csv data using the menu item File... Open CSV file. If the video has been measured with LiMovie, there is the additoional option of copying the measurements in Limovie to the clipboard, and using the menu item File... Paste CSV data set.

You will be taken through three standard AOTA screens which allow the time scale to be set, checks for duplicated or skipped frames, and allows for binning. If an integrating camera has been used, it is important to bin the measurements consistent with the video camera signal. That is, the bin size should correspond to the number of frames integrated (or be a multiple of the number of frames integrated), and the start of the bins need to align with the start of each new integration interval in the video signal.

With the light curve displayed you should select the start and end points of the light curve to be reported. Typically this will be 10's of seconds before the start, and after the end, of an occultation.

You can check the quality of the light curve by selectively plotting any comparison stars. However comparison star data is not reported. Where the comparison star data shows significant changes in transparancy (light level), it may be appropriate to normalise the light curve using a comparison star - using the button for 'Integration, binning, normalisation'. You can also change the binning level. The tools for doing this are those used in AOTA.

Bibliographic data

Unlike lunar occultations, bibliographic data for an asteroidal occultation light curve needs to be manually entered (at this time). Also, at this time the data items that are captured do not include the relative rate of motion, and the direction relative to the normal to the surface of the asteroid.

To enter the bibliographic information, click the Specify event and observer details button in the blue box area. This will open a dialogue box where you specify:

  • Date of the event (but not the time - that comes from the csv data or its processing)
  • Site location - East Longitude, Latitude, and altitude (metres)
  • Name of the observer(s)
  • Asteroid number (the name will be retrieved from the file of asteroid elements)
  • The star number. Select the star catalogue from the drop-down list, and complete the number. (Note that the star number retained is that displayed when the form is left.)
  • Equivalent star numbers in Hipparcos, SAO, Tycho2, and the Kepler 2 EPIC catalogue.

Default settings for site and observer

The currently displayed values for the site coordinates and observer name can be retained as default values by clicking the Set as Defaults button. The default values will be inserted in the fields on the form for entering bibliograhic data, where the corresponding fields in the Light Curve report are blank. You can change the displayed values to the default values by clicking the Use defaults button. Both these options require confirmation that you want to proceed.

Star numbers equivalents

The retrieval of the equivalent star numbers depends upon what catalogues are present on your computer, and having internet access. The variuos possibilities are:
  • The user has UCAC2, UCAC3 and UCAC4 on the computer, has downloaded the Kepler2 catalogue from the Occult downloads page, and has internet access. On clicking Get coords from Star, Occult will retrieve the star's coordinates from the relevant catalogue on the computer, then query Vizier to locate the identifiers of equivalent stars.
  • Option (a) - but no internet access. The identifiers of equivalent stars cannot be retrieved.
  • The user does not have the relevant catalogue on their computer. Retrive the J2000 coordinates of the star from the asteroidal occultation prediction,and manually enter those into the boxes for the star's coordinates. Then click Get IDs from coordinates - which will result in a query to Vizier to identify the equivalent stars.
  • The user does not have the relevant catalogue on their and no internet connection. The user will be unable to identify the equivalent stars.

Transferring the data to the Light Curve report

When finished, click Transfer data and Exit to transfer the data to the relevant locations in theLight Curve report, and close the Bibliographic form. If you need to change any data items, simply click the Specify event and observer details button, and change the data as required. If you click the form close button at the top right of the form, or the Cancel button at bottom right, the form will be closed with none of the data being transferred to the Light Curve report.

Sending the report

Once satisfied with the selection, the usual approach would be to Save the report - with a bulk-emailing of saved reports occurring at a later date. [A typical report will have a size of a few tens of KB.] To save the report, simply click the button Save this report, or select the menu item File...Save _this_ report. .

However if you want to send the report immediately, click the button Save & Email this report, or select the menu item File...Save & Email _this_ report . This will save the report, email the report - and if the email is successful, move the report to the Reported subdirectory. The date for checking whether there are other reports waiting to be emailed is not affected when a single report is sent immediately).

If you want the report(s) to be copied to friends/colleagues, use the Set CC Addresses button to specify the cc addresses to be included on all emails of light curve reports.

A copy of the email will be sent as a BCC to your default email address as specified in the User Settings, box 4 (Email settings...).

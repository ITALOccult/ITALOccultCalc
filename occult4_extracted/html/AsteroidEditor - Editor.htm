Asteroid observations editor

This form is used to enter and edit observations of asteroid occultations.

The form as three logical components:

  • Fields which are specific to an event - namely, the date, star and asteroid
  • Fields which are specific to an observer - namely the observer details, times, and conditions
  • Fields for managing the data, including the display of historical events.

    Fields specific to an event

    These fields, which are in groups with an orange background, need to be completed when a record is created for a new event.

    The event date

    Set here the Date of the event, and the approximate mid-time of the observations.,

    The mid-time is used to generate a 3rd-order expression for the motion of the asteroid's shadow past the Earth. The time is set to 0.1 hrs - and should be for a time close to the middle of all the observations. A time of 0.0 hours is not allowed - use 0.1 instead.

    Note 1: If observations extend across a day boundary, ensure the date corresponds to the day of the earliest observation and that all times for the following day use hours 24 and greater.

    Note 2: If the mid-time field is changed, you should click the button Get details in the Asteroid box to make sure that the internal expression for the asteroid's motion is updated. If the dates is changed by more than a few hours, you should also click the button Get details in the Star box to make sure the star position is correct.


    The star number

    To specify the star, you must first select the catalogue from the drop-down box. The catalogues catered for are Hipparcos, Tycho2, UCAC4, UCAC3, UCAC2, USNO-B1, NOMAD, URAT1, and Gaia stars in the Occult Gaia catalogues that have not been matched to UCAC4, Tycho2 or Hipparcos - identified by Ghhmmss.s±ddmmss. Use the catalogue first in this order. Provided the Gaia catalogue is installed, Occult will endeavor to retrieve a Gaia position for the star - and will preferentially use Hipparcos, Tycho2 and UCAC4 star identifiers. If a Gaia position has been retrieved, that fact will appear at the bottom-right of this group.

    The data fields for the star number depend on the star catalogue. Hipparcos, UCAC2, and the Gaia coordinate identifier, have a single field; Tycho2 has three fields (region, number, and component), while the remainder have two fields (zone and number)

    When the number has been entered, click the Get details button to retrieve the star coordinates. [If star coordinates are not retrieved, you cannot plot the observations.] Also displayed on a separate form will be all double star entries in the Washington Double Star Catalogue and 4th Interferometric Cataogue. [For this to be functional, these catalogues will need to be downloaded from the Downloads option on the Maintenance tab.]

    In appropriate circumstances, an extra button, and 2 extra data items are displayed.

    The Identify G* button checks whether a star identified by the G coordinates has a traditional identifier in the Occult Gaia catalogue. If so, this identifier should be used.

    If the star position in the Occult Gaia catalogue has used the UCAC4 position to derive a proper motion, this fact is noted. The fact thta this has occurred is an indicator that there are problems with the position of this star

    If the star has an apparent diameter that could lead to events being gradual, the estimated duration of such fades, as seen on the center line, is given.

    Note 1: to retrieve USNO-B1 stars, you need to have the NOMAD catalogue. To retrieve a 2UCAC star that is fainter than mag 12, you will need to have either the full 2UCAC catalogue, or the NOMAD catalogue.

    Note 2: Manual entry of star coordinates is not available in this version of Occult - a deliberate decision to avoid erroneous data entry.

    Note 3: The Gaia number by coordinates matches stars by the firsts star that is within 0.1sec of the RA and 1" of the Dec - with the coordinates being specified for the version of Gaia (almost inevitably DR2 - 2015.5) When there are two stars close together, the possibility of the wrong star being identified in the first instance is real. For example, an occultation on 2019 Jan 2 by Alven occulted a star identified as G100729.6+131605. The star's coordinates (2015.5) are:

    10 07 29.5956  +13 16 05.09

    However the previous entry in the file is:

    10 07 29.4260  + 13 16 06.44

    The specified G coordinates retrieves the previous entry star, as it is the first star to match the coordinates at the 0.1s/1" precision. The correct star is retrieved by adjusting the identifier to G100729.7+131605. The point to remember here is to ensure you have retrieved the correct star by comparing the retrieved coordinates with those in the prediction, and adjusting the identifier as required to retrieve the correct star.


    The asteroid or planet

    To specify an asteroid

  • select the Asteroids radio button
  • enter the asteroid's number.
  • click Get details. The field for the Name will be completed, as well as the information items of the shadow's motion on the fundamental plane (in km/sec), the parallax, visual magnitude, and diameter in km. If the asteroid is in either the DAMIT or ISAM databases of shape models, the number of models available in the relevant database will be given in yellow text on a blue background. The blue background can be clicked to download the shape models.

    Instead of the asteroid's number, you can enter the name - but it has to be the full name (case insensitive). Usually you should enter the number.

    For unnumbered asteroids, you need to enter its name - making sure that it is identical to the MPC formatted name.

    Note. If you are entering an observation of a relatively small asteroid, you will need to make sure the asteroid is in the file of asteroid elements maintained by Occult. For this purpose, it is usually best to include the asteroid number in the list of 'mandatory' minor planets maintained on the form for Converting Astorb etc.

    Note 2. When an asteroid is not included in the current file of asteroid elements, you can update that file from the Main Occult form from either the Asteroid Predictions tab, or the Maintenance tab, by clicking the Convert ASTORB and MPCOrb button, without closing the editor. Once the asteroid has been added and a new file created, you can go back to the editor and click the Get Details button to retrieve the asteroid details.

    To specify a planet or satellite

    • Check the radio button Planets
    • select the planet and satellite using the drop-down boxes
    • click Get details.

    The information displayed at the bottom of the box is

    • the motion of the shadow on the fundamental plane, in km/sec
    • the parallax of the object, and its hourly rate of change
    • the visual magnitude of the object
    • the objects diameter and uncertainty in km; its apparent diameter in mas. If the asteroid's diameter has been derived from its absolute magnitude, the diameter is indicated as an estimated diameter (est.Dia.....)


    Fields which are specific to an observer

    These fields are in groups with a light green background, and specify details of an observation. Each observation consists of a Disappearance and Reappearance event pair. In the case of a double star or double asteroid, there needs to be an observation line for each component.


    Observer details

    Observer name The form provides for two observer names to be expressly specified, with a box to check if there are more than two observers. [When there are 3 or more observers, only 2 names can be specifically stated.]

    The observer names are used when reporting astrometry to the Minor Planet Center. To avoid any problems when reporting the astrometry, it is critical that the name fields are filled on as follows:

     To facilitate extraction of name information for reporting results to (for example) the Minor Planet Center (MPC), the following formatting is essential:

    • Names are given as an Initial, followed by the family name - e.g. 'D Herald'
    • One or more full given names can be specified. However it will be automatically reduced to a single initial when reporting to the MPC.
    • Name qualifiers like Mc (as in McArthur), Van der...., and so forth, should be included.
    • Qualifiers like Mrs/Mr etc must not be included. Similarly educational qualifications (eg PhD), or membership of organisations, must not be included.
    • Do not qualify the name with anything intended to indicate the observer's location. The field must contain the observer's name, and absolutely nothing else!
    • There MUST be an initial. A family name with no initial will not be accepted by the MPC
    • The name must be of a person, not an organisation.
    • The field is processed in the order of given name - family name. For cultures where the family name normally comes first, it is important that the names be captured in the given name - family name order.

    Located near

    This box provides for a textual description of the approximate location of he observer. It is provided essentially for assistance with validation the site coordinates, in instances where there is cause for concern about where the observer was located. As a free text box, users should try to make sure that what is specified has meaning to 3rd parties, potentially years after an event. In circumstances where an observer sets up a multi-station deployment with all having the same approximate location, it is appropriate to label the sites by appending characters like '#1'   '#2' etc

    Country or State

    • This field provides for an indication of the country or state in which the observer was located - again, solely for general indication of where the observer was located. The filed is to be completed by specifying the relevant International Standard 2- or 3- letter country codes or (for Australia, Canada, and the USA) state or province codes.
    • for a list of the codes, click the Help button just above this box.
    • Commas cannot (& must not) be used in this field

    Telescope aperture - specify to the nearest cm.

    Telescope type - select the most appropriate from the drop-down box. Available options are

      _ = unstated
      1 = Refractor
      2 = Newtonian
      3 = SCT
      4 = Dobsonian
      5 = Binoculars
      6 = Other
      7 = None
      8 = Electronic

    When Electronic is selected, the Method, and Time fields are automatically set for that telescope type/p>

    Site coordinates - select the relevant format for the data entry (dms, dm.m or d.dd), then enter the coordinates. Remember to include a '-' for west longitudes (i.e. the Americas, most of England, etc.] and for southern latitudes.

      Note. Once an observation line has been entered, you can plot all observation sites in GoogleEarth under the menu item with Observations... Plot sites in GoogleEarth. This will highlight any significant data entry error!

    Datum. Indicate the datum used for the latitude/longitude, using the drop-down box. Almost all modern datums can be treated asWGS84

    Altitude. Select meters or feet (the data is stored as meters). Note. Altitudes are assumed to be referenced to mean sea level (msl) - which is the international standard for referencing altitudes. Some GPS units have the ability to output an elevation referenced to the WGS spheroid. That elevation MUST NOT BE SPECIFIED - always use altitude above mean sea level (msl).

    The word Altitude is actually a 'button'. Click on the word Altitude,  and you will activate the Site Altitude validation function that is described below under the with Observations... description. When an event is read from a file, or is pasted, the word Altitude will be shown in red - until the site validation has been undertaken.

     


    Event times

    Enter the observed times of the events here. Particular issues are:

    • Ensure each field of h, m and s is completed. If any of the fields for a time is blank, the event will be treated as a 'no event'.
    • For a Miss event, insert the expected time of the occultation - for BOTH the D and R
    • '+/-' is the 3-sigma uncertainty in the reported time. For a visual observation, it is usually smaller than the PEq.
    • PEq is the personal equation applied to visual observations. Preferably, the reported time should be corrected for PEq. In any case, the value should be specified here.
    • Wt. A value of 0 to 5. This field should normally be left blank - in which case visual observations are weighted as 1 and video observations are weighted as 5. However during analyses stages it may be appropriate to apply different weightings (within the range 0 to 5). A weighting of 0 results in an observation being excluded from the solution but can be included in the plot of observations (with the observation number being underlined). The default values (which are used when the setting is blank) are set as follows:
        5 - for analogue or digital (non-SLR) video
        4 - for digital SLR-camera video, and Photometer
        3 - for sequential images and drift scan
        1 - for visual and anything else.
        0 - event is ignored in any solution, but can still be plotted, depending on obne of the options on the Plot form.
    • Time Shift (secs) This is an adjustment applied in analysis to bring a discordant observation into alignment with other observations. It assumes the existence of an error in the time base used by the observer. A time shift should not be applied without good reason.
    • Not seen. This should rarely be used. This allows a coordinator to enter the coordinates of an observer to demonstrate what they missed out on as a result of (for example) cloud, or failing to go out and observe. Data entries for this event type are not added to the file of Historical observations.

    You also need to indicate the Event qualifier using the check boxes. Check boxes in the left column can be checked by themselves, or (when appropriate) together with check boxes in the right column. The significance of the check items is:

    • a 'normal' occultation with regular D and R events has no check boxes marked.
    • No occultation detected. Check this to indicate no occultation was detected. A nominal D and R time (hh mm ss) must be specified for both (a time consistent with the event analysis will be generated during the event analysis). These events most commonly identify observations where the asteroid does not cover the target star (a so-called Miss event). However there may be other explanations for such an observation.
    • Prediction. It is very useful to include an entry representing the prediction, as a means for verifying the data entry and accuracy of the observation. The preferred source of the prediction entry is from an Occult generated path prediction using the Horizons system with the paste from that prediction including the JPL solution number and date.
    • unseen Satellite parent. This is used when the observations for an event are of the satellite of an asteroid, and not the main body of the asteroid. Such an event is 2010 June 14 (617) Patroclus. To process such an observation, Occult needs a nominal 'observation' of the main body. While the box can be checked manually, the strongly preferred approach to such a situation is:
      • generate a prediction for the asteroid (which will usually relate to the main body)
      • generate path coordinates for that event, and select a line near the location of the observation - and click the menu item with prediction... Copy prediction line for Observations report... Occult.
      • In the observations editor, click the menu item Paste... Prediction for the primary body of a binary asteroid (which has not been observed

      This will create two lines in the observation editor. One is a Prediction line, with event code P. The other is an event line for the unseen primary, with event codes e and f and an observer name of Unseen-Primary-Event.

      If you have a prediction line for the satellite, you can also paste that into the editor in the usual manner.

    • 2nd star. Where the occultation involves a binary star, this is used to indicate that the reported times are associated with the fainter star. Note:
      • frequently the 'double star' event will only be noticed at one of the D and R phases. In that situation it would be expected that the report line for the 2nd star will contain just one time.
      • It is often difficult to correctly associate the event times with each of the components. Frequently it is only at the analysis stage that the correct allocation of times to components can be determined.
    • Ring - an occultation by a ring around an asteroid or planet. D and R times are for the middle of the events on opposite arms of the ring. The duration of the occultation by a ring arm should be noted in the Free text.
    • Satellite. This box is checked when an occultation is by a satellite of an asteroid. This may not be known until the analysis of other observations of that event have been included.
    IMPORTANT: Where multiple events are observed from the same location, place them in chronological order, to facilitate plots that show the star when visible.


    Conditions

    In this section you select various items from the drop-down boxes. [Note - attempts to key changes into the drop down boxes will be ineffective). The items are:

    Method - select the most appropriate description of the method used to make the observation (and for visual observers, the basis of the Personal equation). Available options are:

      _  unspecified
      a  Analogue or digital video
      b  Digital SLR-camera video
      c  Photometer
      d  Sequential images
      e  Drift scan
      f   Visual
      g  Other

    Time - select the most appropriate description for the source of time used to make the observation. The available options are:

      _ unspecified
      a GPS
      b NTP
      c Telephone time (fixed or mobile)
      d Radio time signal
      e Internal clock of recorder [such as by calibrating the recorder well before and after the event]
      f  Stopwatch
      g Other

    Signal-to-Noise ratio- set the most appropriate description of the stability of the atmosphere. The control has variable increments, as follows:
    • < 0.20       0.02
    • 0.20 to 2.0  0.1
    • 2 to 4       0.5
    • >5           1

    Stability - select the most appropriate description of the stability of the atmosphere. Options are:
      _ = unstated
      1 = Steady
      2 = Slight flickering
      3 = Strong flickering

    Transparency     - select the most appropriate description of the transparency of the atmosphere. Options are:
      _ = unstated
      1 = Clear
      2 = Fog
      3 = Thin cloud <2 [mag loss <2 mag.]
      4 = Thick cloud >2 [mag loss >2 mag.
      5 = Broken opaque cloud [that is, observed thru gaps in the cloud]
      6 = Star faint
      7 = By averted vision

    Free text - this is a location where you can add comments to highlight special issues. Comments on routine matters should not be included. You cannot use Enter or Tab. The character  |, as well as non-ASCII characters, the character ", and control characters, will not be accepted. The number of characters in the free text is indicated above the top-right of the field
    When entering text into this field, the text box will be enlarged, with a button underneath to return it to its small size. Clicking another data entry control will also reduce it back to its small size. it is important that it be meaningful for external readers, and relates to an issue that is relevant to the reliability or credibility of the observation. Comments about the tool used to measure the video, or analyze the light curve, are generally not appropriate.

    Include in solution This field identifies observations that are excluded from the solution (including the graphic display of the event. The choices available are:
    OptionComments
    Include observationBoth the D and R events are included in the solution, and plotted
    Include D onlyOnly the D event is  included in the solution and plotted. The R event is ignored
    Include R onlyOnly the R event is included in the solution and plotted. The D event is ignored
    Exclude observationThe complete observation is excluded from the solution, and is not plotted

    The setting can be used to:

  • to exclude an apparently erroneous observations; or
  • exclude extremely distant Not detected events, so that the plot of the events is properly concentrated on the observed chords and close not-detected events.

    Manage observers

    At the bottom-right of the form are buttons for managing the data contained in memory for the particular event displayed:

    • Add the data in the edit fields as a new observation record
    • Replace the currently selected observation record with the data in the edit fields
    • move the currently selected observation either up or down in the list of observations
    • delete the currently selected observation; and
    • sequentially renumber the observations in the list

    In addition, controls are provided in the menu item with Observations, where you can sort the list of observations by any of

    • longitude
    • latitude
    • the observer name field
    • the observer name field ignoring the first two characters (that is, the family name of the observer - if the name has been entered in accordance with the guidelines above)
    • path distance, or
    • sequence number.
    Note. Sorting by distance (followed by a Re-number) is particularly useful, as it ensures the observer numbers displayed on the plot of the event progress uniformly across the plot. This makes it much easier to find the chord of a particular observer.

    Manage Historical file

    This button requires administrator access; It is used to maintain the Historical Observations file with the edited data, and is located below the buttons for managing observers. The text on the button varies between Add event to Historical file, and Update Historical file with revised event, depending on

    • whether the data in the form has been 'entered' or imported using the File... open, or Paste options, or
    • whether the data has originated from the Historical observations file.
       

    Open-save check box

    This only appears when Administration access is enabled. When checked, a button will appear with the label '+1'. If that button is clicked, the next event listed in the event drop-down box will be opened, and then immediately saved. This functionality is provided for updating the event solution (particularly the astrometric solution) as a result of code changes. It should not be used generally.

    • in the case of single-chord observations, provide a basis for assessing whether there are any large errors in the observation - either in the reported data, or in the data as entered.

    Displayed information

    The following information is displayed in the bottom part of the form

    • A list box listing all the observations entered for the event
    • At the top left of the list box, the expected magnitude drop at occultation
    • At top-middle of the list box - a summary of the current solution for the ellipse and any double star component

    Viewing Historical observations

    The viewing of a past asteroid occultation is governed by the controls in the Historical Observations box.

    Most of the controls in the box are used to set the list of events that are listed in the drop-down box at the bottom of the area. The event data displayed in the drop-down box are:

    • Year, month and day of the event
    • # - when shape models are available, and all have been fitted
    • † - when shape models are available that have not been fitted
    • Asteroid number and name
    • number of observations (records) - excluding prediction lines
    • number of observed chords (limited to the primary star)

    Use the radio buttons to set whether to select the event on the basis of date, by asteroid name, asteroid number, but the number of observation lines (Records), or events involving a planet or a moon of a planet.

    • When Date is selected, select the year range with the drop-down box. The main drop down box will list events in that date range, sorted by date.
    • When Name is selected, select the relevant letter for the first letter of the asteroid's name from the drop-down box. The main drop down box will list all events involving asteroids commencing with that letter (or number), sorted alphanumerically, then by date.
    • When Number is selected, select the number range with the drop-down box. The main drop down box will list asteroids in that number range, sorted by number then date.
    • Shape Models limits the displayed events to those having shape models available.
    • Unfitted Models limits the displayed events to those where shape models are available that have not been fitted.
    • When #Records is selected, the main drop down box will list all events sorted by the number of records.
    • When #Chords is selected, the main drop down box will list all events sorted by the number of observed chords.
    • When Planets only is selected, the main drop down box will list events involving planets or their moons, sorted by planet number, then moon number.
    With any of these selections, the text immediately above the main drop-down box gives the number of items in the main drop-down box, and the total number of events.

     To display an event, merely select the event in the drop-down box. the observations will be displayed on the form, and the events plotted.

    Shape Models  
    List current-event shape-model solution(s) Lists all the shape model solutions that have been fitted to the event
    Delete current-event shape-model solution(s) This allows the user to delete one or more shape model solutions that have been entered. Used for when a shape model has been fitted, but would prefer to not have been made.
    Download shape models Retrieves all available shape models from the Database of Asteroid Models from Inversion Techniques, and from the Interactive Service for Asteroid Models. . These are databases of three-dimensional models of asteroids computed using inversion techniques. DAMIT contains models for more asteroids than ISAM, whereas ISAM contains more models than DAMIT for many individual asteroids.

    This functionality is also provided by clicking the blue text boxes in the Asteroid group on the Editor. Note that those text boxes only appear when a shape model is available.

    Administrator functions

    When an event is displayed from the file of Historical Observations, and you make changes to the data, you cannot update the entry in the file of Historical observations unless Administrator status is enabled in the User Settings. Similarly, you cannot add new observations to the file of Historical observations.

    Additionally, Administrator status enables a menu option to import observations or events from the Euraster site, and from the site of Japanese observations. These function as follows:

      Euraster
    • To add a complete event report from Euraster - highlight all lines of the report, Copy, then select Paste..New event [date + observation] from Euraster. This will copy all the events into the editor, and complete all the star/asteroid details where possible.
    • To add multiple event reports from Euraster - highlight the lines from the start of the first report to the end of the last. Copy, then select Paste..Multiple events [date + observation] from Euraster. This will process all events where a positive observation was reported, one-by-one
    • To add a single observation from Euraster to an existing report - highlight the additional observation line, Copy, then select Paste..New observation line [one only] from Euraster. This will add the event to those in the editor.
      Note 1. To Paste a new event, you will need to click File... New, to clear any events from the current display
    • Note 2. From Occult 4.6, new event lines can be added to a displayed Historical observation using the Paste functionality.

      Japan

    • To add a complete event report from Japan - highlight all lines of the observation [starting at the line that identifies the star]. Copy, then select Paste..Multiple events [date + observation] from JP. This will process all events where a positive observation was reported, one-by-one.
    • To add a single observation from Japan to an existing report - highlight the additional observation line, Copy, then select Paste..New observation line [one only] from JP. This will add the event to those in the editor.
      Note 1. To Paste a new event, you will need to click File... New, to clear any events from the current display
    • Note 2. From Occult 4.6, new event lines can be added to a displayed Historical observation using the Paste functionality.
     

          OBS format  [This is an old format that is no longer used.]

    There are two processes for importing observations that are in the native format used in Occult:

     Individual files can be opened/edited/saved using the menu item File... Save. Multiple files can be opened, edited and (typically) added to the Historical file of observations, using the menu item File... Open Multiple files.

    • Similarly, individual events can be pasted using the menu item FileTo add a complete event report from a report that is already in the OBS format used by Occult - highlight all lines of the observation. Copy, then select Paste..Ne w event - OBS format. This will copy all the events into the editor, and complete all the star/asteroid details where possible.
    • To add one or more observation lines in the OBS format - highlight the one or several observation lines, Copy, then select Paste..New observation line [or Lines] - OBS format. This will add the events to those in the editor.
      Note 1 - unlike pasting from Euraster and JP, you can add more than one observation in a single Paste.
      Note 2 - if adding observations to an event in the Historical observations - you must first do a File ... Save as with the event. Then add the observations and save the updated file. Incorporation of the updated event into the observations file will require a manual edit of the file.

      Prediction line

    • To add a prediction line - either from the path coordinate list in Steve Preston's site, or from the Occult path prediction. Highlight (select) a line in the prediction that relates to a location near to the observers. From the menu line, select with Prediction....  Create Prediction line for Observations Report... Occult Then in the Observations Editor, click Paste... Prediction line.

    IMPORTANT Always check the data for both completeness and correctness before adding the event to the Historical observations file.
